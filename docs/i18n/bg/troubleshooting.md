# Отстраняване на проблеми

Използвайте тази страница, когато изпълнението на превод приключи неочаквано успешно, провали се по време на конфигурация или генерира резултат, който се нуждае от преглед.

## Започнете тук

1. Първо изпълнете целенасочена команда, например `translate -l "ko" -md`.
2. Добавете `-d` за конзолни регистри за отстраняване на грешки.
3. Добавете `-s`, за да запазите регистрите за отстраняване на грешки под `<root-dir>/logs/`.
4. Изпълнете `co-op-review` след превода, за да проверите актуалността, структурата и локалните връзки.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## Грешки в конфигурацията

### Няма доставчик на езиков модел

Грешка:

```text
No language model configuration found.
```

Решение:

- Конфигурирайте Azure OpenAI или OpenAI.
- Уверете се, че променливите са в средата, в която се изпълнява командата.
- За локална употреба, поставете ги в `.env` в корена на проекта.

Вижте [Конфигурация](configuration.md).

### Превод на изображения без Azure AI Vision

Грешка:

```text
Image translation requested but Azure AI Service is not configured.
```

Решение:

- Добавете `AZURE_AI_SERVICE_API_KEY`.
- Добавете `AZURE_AI_SERVICE_ENDPOINT`.
- Или стартирайте само текстова команда, например `translate -l "ko" -md`.

### Невалиден ключ или крайна точка

Симптомите могат да включват `401`, скрити съобщения за грешки относно разрешения, или грешки при достъп до крайна точка.

Решение:

- Потвърдете, че ключът принадлежи на същия Azure ресурс като крайната точка.
- Потвърдете, че ресурсът поддържа Vision, когато използвате `-img`.
- Потвърдете, че името на разгръщането на Azure OpenAI и версията на API съвпадат с вашето разгръщане.
- Стартирайте с регистри за отстраняване на грешки: `translate -l "ko" -md -d -s`.

## Няма преведени файлове

Чести причини:

- Избраните флагове не съответстват на вашите файлове.
- Вече има съществуващи преведени файлове.
- Изходните файлове са под изключени директории.
- Командата се изпълнява от неправилен корен на проекта.

Проверки:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

Използвайте `--root-dir`, когато командата се изпълнява извън корена на проекта.

## Неочаквано поведение на връзките

Пренаписването на връзки зависи от избраните типове съдържание:

- `-nb` включен: връзките към тетрадки могат да сочат към преведени тетрадки.
- `-nb` изключен: връзките към тетрадки могат да останат насочени към изходните тетрадки.
- `-img` включен: връзките към изображения могат да сочат към преведени изображения.
- `-img` изключен: връзките към изображения могат да останат насочени към изходните изображения.

Извършете пълен превод на съдържанието, когато всички вътрешни връзки трябва да предпочитат преведените резултати:

```bash
translate -l "ko" -md -nb -img
```

Извършете преглед на връзките след превода:

```bash
co-op-review -l "ko"
```

## Проблеми с визуализирането на Markdown

Ако преведеният Markdown се визуализира неправилно:

- Проверете дали frontmatter започва и завършва с `---`.
- Проверете дали броят на ограждащите маркери за код съвпада между изходните и преведените файлове.
- Стартирайте `co-op-review`, за да откриете чести структурни проблеми.
- Преведете отново конкретния файл, ако изходът е бил повреден.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action се изпълни, но не беше създаден pull request

Ако `peter-evans/create-pull-request` отчита, че клонът не е пред базата, работният процес не е намерил файлове за commit.

Вероятни причини:

- Преводното изпълнение не е произвело промени.
- `.gitignore` изключва `translations/`, `translated_images/` или преведените тетрадки.
- `add-paths` не съвпада с генерираните изходни директории.
- Стъпката за превод е приключила преждевременно.

Поправки:

1. Потвърдете, че генерираните файлове съществуват в `translations/` или `translated_images/`.
2. Потвърдете, че `.gitignore` не игнорира генерираните изходи.
3. Използвайте съвпадащи `add-paths`:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. Временно добавете debug флагове към командата за превод:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. Потвърдете, че разрешенията на работния процес включват:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## Качество на превода

Машинните преводи може да изискват човешка проверка. Използвайте `evaluate` само когато желаете експериментално оценяване на качеството и работни процеси за поправка при ниско доверие.

!!! warning "Experimental"
    `evaluate` може да използва проверки, базирани на правила и LLM, а неговият модел за оценяване и поведението на метаданните могат да се променят. Дръжте го извън изискваните CI проверки, освен ако работният ви процес не е подготвен за промени.

За детерминирани CI проверки, използвайте вместо това `co-op-review`.
