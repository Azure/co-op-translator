# সমস্যা সমাধান

এই পৃষ্ঠা ব্যবহার করুন যখন অনুবাদ চালানো হঠাৎ সফল হয়, কনফিগারেশনের সময় ব্যর্থ হয়, বা এমন আউটপুট তৈরি করে যা পর্যালোচনা প্রয়োজন।

## এখানে শুরু করুন

1. প্রথমে একটি সীমিত কমান্ড চালান, যেমন `translate -l "ko" -md`.
2. কনসোল ডিবাগ লগের জন্য `-d` যোগ করুন।
3. ডিবাগ লগগুলো সংরক্ষণ করতে `-s` যোগ করুন `<root-dir>/logs/`-এর মধ্যে।
4. অনুবাদের পরে তাজা থাকা, কাঠামো, এবং লোকাল লিঙ্কগুলো পরীক্ষা করতে `co-op-review` চালান।

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## কনফিগারেশন ত্রুটি

### কোনো ভাষা মডেল প্রদানকারী নেই

ত্রুটি:

```text
No language model configuration found.
```

সমাধান:

- Azure OpenAI বা OpenAI কনফিগার করুন।
- নিশ্চিত করুন যে ভেরিয়েবলগুলো সেই পরিবেশে আছে যেখানে কমান্ডটি চালানো হয়।
- লোকাল ব্যবহারের জন্য, সেগুলো প্রকল্প রুটে `.env`-এ রাখুন।

দেখুন [কনফিগারেশন](configuration.md).

### Azure AI Vision ছাড়া ইমেজ অনুবাদ

ত্রুটি:

```text
Image translation requested but Azure AI Service is not configured.
```

সমাধান:

- `AZURE_AI_SERVICE_API_KEY` যোগ করুন।
- `AZURE_AI_SERVICE_ENDPOINT` যোগ করুন।
- অথবা একটি কেবল টেক্সট কমান্ড চালান, যেমন `translate -l "ko" -md`।

### অবৈধ কী বা এন্ডপয়েন্ট

লক্ষণগুলোতে থাকতে পারে `401`, রেড্যাক্টেড অনুমতি ত্রুটি, অথবা এন্ডপয়েন্ট অ্যাকসেস ত্রুটি।

সমাধান:

- নিশ্চিত করুন কীটি এন্ডপয়েন্টের সঙ্গে একই Azure রিসোর্সের অন্তর্গত।
- `-img` ব্যবহার করার সময় নিশ্চিত করুন রিসোর্সটি Vision সমর্থন করে।
- নিশ্চিত করুন Azure OpenAI ডিপ্লয়মেন্ট নাম ও API সংস্করণ আপনার ডিপ্লয়মেন্টের সাথে মেলে।
- ডিবাগ লগসহ চালান: `translate -l "ko" -md -d -s`.

## কোনো ফাইল অনুবাদ হয়নি

সাধারণ কারণসমূহ:

- নির্বাচিত ফ্ল্যাগগুলো আপনার ফাইলগুলোর সাথে মেলে না।
- অনুবাদকৃত ফাইলগুলো ইতিমধ্যেই উপস্থিত আছে।
- সোর্স ফাইলগুলো বাদ দেওয়া ডিরেক্টরির অধীনে আছে।
- কমান্ডটি ভুল প্রকল্প রুট থেকে চালানো হচ্ছে।

পরীক্ষা:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

যখন কমান্ডটি প্রকল্প রুটের বাইরেই চালানো হয় তখন `--root-dir` ব্যবহার করুন।

## অপ্রত্যাশিত লিঙ্ক আচরণ

লিঙ্ক পুনঃলিখন নির্বাচিত কন্টেন্ট টাইপগুলোর ওপর নির্ভর করে:

- `-nb` অন্তর্ভুক্ত: নোটবুক লিঙ্কগুলো অনুবাদকৃত নোটবুকের দিকে নির্দেশ করতে পারে।
- `-nb` বাদ দেওয়া হলে: নোটবুক লিঙ্কগুলো সোর্স নোটবুকের দিকে থাকতে পারে।
- `-img` অন্তর্ভুক্ত: ইমেজ লিঙ্কগুলো অনুবাদকৃত ইমেজের দিকে নির্দেশ করতে পারে।
- `-img` বাদ দেওয়া হলে: ইমেজ লিঙ্কগুলো সোর্স ইমেজের দিকে থাকতে পারে।

সমস্ত অভ্যন্তরীণ লিঙ্ক অনুবাদিত আউটপুটগুলোকেই অগ্রাধিকার দেবে এমন ক্ষেত্রে পূর্ণ কন্টেন্ট অনুবাদ চালান:

```bash
translate -l "ko" -md -nb -img
```

অনুবাদের পরে লিঙ্ক রিভিউ চালান:

```bash
co-op-review -l "ko"
```

## মার্কডাউন রেন্ডারিং সমস্যা

যদি অনুবাদকৃত মার্কডাউন ভুলভাবে রেন্ডার করে:

- যাচাই করুন যে ফ্রন্টম্যাটার `---` দিয়ে শুরু এবং শেষ হচ্ছে।
- যাচাই করুন যে সোর্স ও অনুবাদকৃত ফাইলগুলোর মধ্যে কোড ফেন্সের সংখ্যা মেলে।
- সাধারণ কাঠামোগত ত্রুটি ধরতে `co-op-review` চালান।
- যদি আউটপুট ক্ষতিগ্রস্ত হয়ে থাকে, নির্দিষ্ট ফাইলটি পুনরায় অনুবাদ করুন।

```bash
co-op-review -l "ko" --format github
```

## GitHub Action চালিত হয়েছে কিন্তু কোন পুল রিকুয়েস্ট তৈরি হয়নি

যদি `peter-evans/create-pull-request` রিপোর্ট করে যে ব্রাঞ্চটি বেসের চেয়ে এগিয়ে নেই, তবে ওয়ার্কফ্লো কোন ফাইল কমিট করতে পায়নি।

সম্ভাব্য কারণসমূহ:

- অনুবাদ চালানো কোনো পরিবর্তন তৈরি করেনি।
- `.gitignore` `translations/`, `translated_images/`, বা অনুবাদকৃত নোটবুকগুলোকে বাদ দিয়েছে।
- `add-paths` জেনারেটেড আউটপুট ডিরেক্টরিগুলোর সাথে মেলে না।
- অনুবাদ ধাপটি আগে থেকেই শেষ হয়ে গেছে।

সমাধান:

1. নিশ্চিত করুন জেনারেটেড ফাইলগুলো `translations/` অথবা `translated_images/`-এ রয়েছে।
2. নিশ্চিত করুন `.gitignore` জেনারেটেড আউটপুটগুলোকে উপেক্ষা করে না।
3. ম্যাচিং `add-paths` ব্যবহার করুন:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. অস্থায়ীভাবে translate কমান্ডে ডিবাগ ফ্ল্যাগ যোগ করুন:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. নিশ্চিত করুন ওয়ার্কফ্লো অনুমতিসমূহ অন্তর্ভুক্ত:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## অনুবাদের গুণমান

মেশিন অনুবাদে মানব পর্যালোচনা প্রয়োজন হতে পারে। কেবল তখনই `evaluate` ব্যবহার করুন যখন আপনি পরীক্ষামূলক গুণগত স্কোরিং এবং কম-আত্মবিশ্বাসের মেরামত ওয়ার্কফ্লো চান।

!!! warning "Experimental"
    `evaluate` নিয়ম-ভিত্তিক এবং LLM-ভিত্তিক পরীক্ষা ব্যবহার করতে পারে, এবং এর স্কোরিং মডেল ও মেটাডেটা আচরণ পরিবর্তনশীল হতে পারে। এটি বাধ্যতামূলক CI গেটগুলোর বাইরে রাখুন যদি না আপনার ওয়ার্কফ্লো পরিবর্তনগুলোর জন্য প্রস্তুত।

For deterministic CI checks, use `co-op-review` instead.
