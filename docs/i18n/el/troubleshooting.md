# Αντιμετώπιση προβλημάτων

Χρησιμοποιήστε αυτή τη σελίδα όταν μια εκτέλεση μετάφρασης ολοκληρώνεται απροσδόκητα με επιτυχία, αποτυγχάνει κατά τη διαμόρφωση, ή παράγει έξοδο που χρειάζεται έλεγχο.

## Ξεκινήστε εδώ

1. Εκτελέστε πρώτα μια εστιασμένη εντολή, όπως `translate -l "ko" -md`.
2. Προσθέστε `-d` για καταγραφές αποσφαλμάτωσης στην κονσόλα.
3. Προσθέστε `-s` για να αποθηκεύσετε τις καταγραφές αποσφαλμάτωσης στο `<root-dir>/logs/`.
4. Εκτελέστε `co-op-review` μετά τη μετάφραση για να ελέγξετε την επικαιρότητα, τη δομή και τους τοπικούς συνδέσμους.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## Σφάλματα Διαμόρφωσης

### Δεν βρέθηκε πάροχος μοντέλου γλώσσας

Σφάλμα:

```text
No language model configuration found.
```

Διόρθωση:

- Ρυθμίστε το Azure OpenAI ή το OpenAI.
- Επιβεβαιώστε ότι οι μεταβλητές βρίσκονται στο περιβάλλον όπου εκτελείται η εντολή.
- Για τοπική χρήση, τοποθετήστε τις στο `.env` στη ρίζα του έργου.

Δείτε [Διαμόρφωση](configuration.md).

### Μετάφραση εικόνων χωρίς Azure AI Vision

Σφάλμα:

```text
Image translation requested but Azure AI Service is not configured.
```

Διόρθωση:

- Προσθέστε `AZURE_AI_SERVICE_API_KEY`.
- Προσθέστε `AZURE_AI_SERVICE_ENDPOINT`.
- Ή εκτελέστε μια εντολή μόνο για κείμενο, όπως `translate -l "ko" -md`.

### Μη έγκυρο κλειδί ή endpoint

Τα συμπτώματα μπορεί να περιλαμβάνουν `401`, σφάλματα δικαιωμάτων με απόκρυψη ή σφάλματα πρόσβασης στο endpoint.

Διόρθωση:

- Επιβεβαιώστε ότι το κλειδί ανήκει στον ίδιο πόρο Azure με το endpoint.
- Επιβεβαιώστε ότι ο πόρος υποστηρίζει Vision όταν χρησιμοποιείτε `-img`.
- Επιβεβαιώστε ότι το όνομα deployment του Azure OpenAI και η έκδοση API ταιριάζουν με την ανάπτυξή σας.
- Εκτελέστε με καταγραφές αποσφαλμάτωσης: `translate -l "ko" -md -d -s`.

## Δεν Μεταφράστηκαν Αρχεία

Συνήθεις αιτίες:

- Οι επιλεγμένες σημαίες (flags) δεν ταιριάζουν με τα αρχεία σας.
- Υπάρχουν ήδη υπάρχοντα μεταφρασμένα αρχεία.
- Τα αρχεία προέλευσης βρίσκονται σε αποκλεισμένους καταλόγους.
- Η εντολή εκτελείται από λανθασμένη ρίζα έργου.

Έλεγχοι:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

Χρησιμοποιήστε `--root-dir` όταν η εντολή εκτελείται εκτός της ρίζας του έργου.

## Απροσδόκητη συμπεριφορά συνδέσμων

Η αναγραφή/επαναγραφή συνδέσμων εξαρτάται από τους επιλεγμένους τύπους περιεχομένου:

- `-nb` συμπεριλαμβάνεται: οι σύνδεσμοι προς notebook μπορούν να δείχνουν σε μεταφρασμένα notebooks.
- `-nb` εξαιρείται: οι σύνδεσμοι προς notebook μπορούν να παραμείνουν να δείχνουν στα πρωτότυπα notebooks.
- `-img` συμπεριλαμβάνεται: οι σύνδεσμοι εικόνων μπορούν να δείχνουν σε μεταφρασμένες εικόνες.
- `-img` εξαιρείται: οι σύνδεσμοι εικόνων μπορούν να παραμείνουν να δείχνουν σε εικόνες προέλευσης.

Εκτελέστε πλήρη μετάφραση περιεχομένου όταν όλοι οι εσωτερικοί σύνδεσμοι πρέπει να προτιμούν μεταφρασμένες εξόδους:

```bash
translate -l "ko" -md -nb -img
```

Εκτελέστε έλεγχο συνδέσμων μετά τη μετάφραση:

```bash
co-op-review -l "ko"
```

## Προβλήματα Απόδοσης Markdown

Εάν το μεταφρασμένο Markdown εμφανίζεται εσφαλμένα:

- Ελέγξτε ότι το frontmatter αρχίζει και τελειώνει με `---`.
- Ελέγξτε ότι ο αριθμός των code fences ταιριάζει μεταξύ των αρχείων προέλευσης και των μεταφρασμένων αρχείων.
- Εκτελέστε `co-op-review` για να εντοπίσετε κοινά προβλήματα δομής.
- Μεταφράστε ξανά το συγκεκριμένο αρχείο εάν η έξοδος είχε αλλοιωθεί.

```bash
co-op-review -l "ko" --format github
```

## Το GitHub Action Εκτελέστηκε αλλά Δεν Δημιουργήθηκε Pull Request

Εάν το `peter-evans/create-pull-request` αναφέρει ότι το branch δεν είναι μπροστά από τη βάση, το workflow δεν βρήκε αρχεία για commit.

Πιθανά αίτια:

- Η εκτέλεση μετάφρασης δεν παρήγαγε αλλαγές.
- Το `.gitignore` αποκλείει τους φακέλους `translations/`, `translated_images/` ή τα μεταφρασμένα notebooks.
- Το `add-paths` δεν ταιριάζει με τους παραγόμενους καταλόγους εξόδου.
- Το βήμα μετάφρασης τερματίστηκε πρόωρα.

Διορθώσεις:

1. Επιβεβαιώστε ότι τα παραγόμενα αρχεία υπάρχουν στο `translations/` ή στο `translated_images/`.
2. Επιβεβαιώστε ότι το `.gitignore` δεν αγνοεί τα παραγόμενα αρχεία εξόδου.
3. Χρησιμοποιήστε αντίστοιχα `add-paths`:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. Προσωρινά προσθέστε σημαίες αποσφαλμάτωσης στην εντολή translate:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. Επιβεβαιώστε ότι τα δικαιώματα του workflow περιλαμβάνουν:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## Ποιότητα Μετάφρασης

Οι μηχανικές μεταφράσεις ενδέχεται να χρειάζονται ανθρώπινη αναθεώρηση. Χρησιμοποιήστε το `evaluate` μόνο όταν θέλετε πειραματική βαθμολόγηση ποιότητας και ροές εργασίας επιδιόρθωσης χαμηλής εμπιστοσύνης.

!!! warning "Experimental"
    `evaluate` μπορεί να χρησιμοποιεί ελέγχους βασισμένους σε κανόνες και σε LLM, και το μοντέλο βαθμολόγησης και η συμπεριφορά των μεταδεδομένων του ενδέχεται να αλλάξουν. Μην το εντάσσετε σε απαιτούμενες πύλες CI εκτός αν το workflow σας είναι προετοιμασμένο για αλλαγές.

Για ντετερμινιστικούς ελέγχους CI, χρησιμοποιήστε `co-op-review`.
