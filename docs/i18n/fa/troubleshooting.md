# رفع اشکال

از این صفحه زمانی استفاده کنید که اجرای ترجمه به‌طور غیرمنتظره موفق شود، در مرحله پیکربندی خطا بدهد، یا خروجی‌ای تولید کند که نیاز به بازبینی داشته باشد.

## از اینجا شروع کنید

1. ابتدا یک فرمان متمرکز اجرا کنید، مانند `translate -l "ko" -md`.
2. برای لاگ‌های دیباگ کنسول، `-d` را اضافه کنید.
3. برای ذخیرهٔ لاگ‌های دیباگ زیر `<root-dir>/logs/`، `-s` را اضافه کنید.
4. بعد از ترجمه `co-op-review` را اجرا کنید تا تازگی، ساختار و لینک‌های محلی بررسی شوند.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## خطاهای پیکربندی

### عدم وجود ارائه‌دهنده مدل زبانی

خطا:

```text
No language model configuration found.
```

رفع:

- Azure OpenAI یا OpenAI را پیکربندی کنید.
- بررسی کنید متغیرها در محیطی که فرمان اجرا می‌شود وجود داشته باشند.
- برای استفاده محلی، آن‌ها را در `.env` در ریشهٔ پروژه قرار دهید.

برای جزئیات به [پیکربندی](configuration.md) مراجعه کنید.

### ترجمهٔ تصویر بدون Azure AI Vision

خطا:

```text
Image translation requested but Azure AI Service is not configured.
```

رفع:

- مقدار `AZURE_AI_SERVICE_API_KEY` را اضافه کنید.
- مقدار `AZURE_AI_SERVICE_ENDPOINT` را اضافه کنید.
- یا یک فرمان فقط متنی اجرا کنید مانند `translate -l "ko" -md`.

### کلید یا نقطهٔ پایانی نامعتبر

علائم می‌تواند شامل `401`، خطاهای مربوط به مجوز که مخفی شده‌اند، یا خطاهای دسترسی به نقطهٔ پایانی باشد.

رفع:

- تأیید کنید که کلید متعلق به همان منبع Azure است که نقطهٔ پایانی به آن تعلق دارد.
- تأیید کنید منبع هنگام استفاده از `-img` از Vision پشتیبانی می‌کند.
- تأیید کنید نام استقرار Azure OpenAI و نسخهٔ API با استقرار شما مطابقت دارند.
- با لاگ‌های دیباگ اجرا کنید: `translate -l "ko" -md -d -s`.

## هیچ فایلی ترجمه نشد

علل متداول:

- گزینه‌های انتخاب‌شده با فایل‌های شما مطابقت ندارند.
- فایل‌های ترجمه‌شده موجود هستند.
- فایل‌های منبع در دایرکتوری‌های مستثنی قرار دارند.
- فرمان از ریشهٔ پروژهٔ اشتباه اجرا می‌شود.

بررسی‌ها:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

از `--root-dir` زمانی استفاده کنید که فرمان خارج از ریشهٔ پروژه اجرا می‌شود.

## رفتار غیرمنتظرهٔ لینک‌ها

بازنویسی لینک بسته به انواع محتوای انتخاب‌شده است:

- اگر `-nb` شامل شده باشد: لینک‌های نوت‌بوک می‌توانند به نوت‌بوک‌های ترجمه‌شده اشاره کنند.
- اگر `-nb` مستثنی شده باشد: لینک‌های نوت‌بوک می‌توانند همچنان به نوت‌بوک‌های منبع اشاره کنند.
- اگر `-img` شامل شده باشد: لینک‌های تصویر می‌توانند به تصاویر ترجمه‌شده اشاره کنند.
- اگر `-img` مستثنی شده باشد: لینک‌های تصویر می‌توانند همچنان به تصاویر منبع اشاره کنند.

وقتی همه لینک‌های داخلی باید خروجی‌های ترجمه‌شده را ترجیح دهند، یک ترجمهٔ کامل محتوا اجرا کنید:

```bash
translate -l "ko" -md -nb -img
```

بعد از ترجمه بازبینی لینک‌ها را اجرا کنید:

```bash
co-op-review -l "ko"
```

## مشکلات رندر Markdown

اگر Markdown ترجمه‌شده به‌درستی نمایش داده نمی‌شود:

- بررسی کنید که frontmatter با `---` شروع و پایان می‌یابد.
- بررسی کنید شمار code fenceها بین فایل منبع و فایل ترجمه‌شده یکسان باشد.
- برای یافتن مشکلات ساختاری رایج، `co-op-review` را اجرا کنید.
- اگر خروجی خراب شده بود، آن فایل خاص را دوباره ترجمه کنید.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action اجرا شد اما هیچ Pull Request‌ای ایجاد نشد

اگر `peter-evans/create-pull-request` گزارش دهد که شاخه جلوتر از base نیست، workflow هیچ فایلی برای کامیت پیدا نکرده است.

دلایل احتمالی:

- اجرای ترجمه هیچ تغییری تولید نکرد.
- در `.gitignore`، `translations/`، `translated_images/` یا نوت‌بوک‌های ترجمه‌شده مستثنی شده‌اند.
- `add-paths` با دایرکتوری‌های خروجی تولیدشده مطابقت ندارد.
- مرحلهٔ ترجمه زودتر خاتمه یافته است.

راه‌حل‌ها:

1. تأیید کنید فایل‌های تولیدشده در `translations/` یا `translated_images/` وجود دارند.
2. تأیید کنید `.gitignore` خروجی‌های تولیدشده را نادیده نمی‌گیرد.
3. از `add-paths` هماهنگ استفاده کنید:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. موقتاً flagهای دیباگ را به فرمان translate اضافه کنید:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. تأیید کنید مجوزهای workflow شامل موارد زیر باشد:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## کیفیت ترجمه

ترجمه‌های ماشینی ممکن است نیاز به بازبینی انسانی داشته باشند. از `evaluate` تنها زمانی استفاده کنید که بخواهید ارزیابی کیفیت تجربی و گردش‌کارهای تعمیر با اعتماد پایین را اجرا کنید.

!!! warning "Experimental"
    `evaluate` می‌تواند از چک‌های مبتنی بر قانون و مبتنی بر LLM استفاده کند، و مدل امتیازدهی و رفتار متادیتای آن ممکن است تغییر کند. آن را از دروازه‌های ضروری CI خارج نگه دارید مگر اینکه گردش‌کار شما برای تغییرات آماده باشد.

برای بررسی‌های قطعی در CI، از `co-op-review` به‌جای آن استفاده کنید.
