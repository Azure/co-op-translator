# समस्या निवारण

इस पृष्ठ का उपयोग तब करें जब अनुवाद रन अप्रत्याशित रूप से सफल हो, कॉन्फ़िगरेशन के दौरान असफल हो, या आउटपुट की समीक्षा की आवश्यकता हो।

## यहाँ शुरू करें

1. सबसे पहले एक लक्षित कमांड चलाएँ, जैसे `translate -l "ko" -md`.
2. कंसोल डिबग लॉग्स के लिए `-d` जोड़ें।
3. डिबग लॉग्स को `<root-dir>/logs/` के अंतर्गत संग्रहीत करने के लिए `-s` जोड़ें।
4. ताज़गी, संरचना और स्थानीय लिंक जांचने के लिए अनुवाद के बाद `co-op-review` चलाएँ।

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## कॉन्फ़िगरेशन त्रुटियाँ

### कोई भाषा मॉडल प्रदाता नहीं

त्रुटि:

```text
No language model configuration found.
```

समाधान:

- Azure OpenAI या OpenAI कॉन्फ़िगर करें।
- सुनिश्चित करें कि वेरिएबल उस वातावरण में हैं जहाँ कमांड चल रही है।
- स्थानीय उपयोग के लिए, इन्हें प्रोजेक्ट रूट में `.env` में रखें।

See [कॉन्फ़िगरेशन](configuration.md).

### Azure AI Vision के बिना छवि अनुवाद

त्रुटि:

```text
Image translation requested but Azure AI Service is not configured.
```

समाधान:

- `AZURE_AI_SERVICE_API_KEY` जोड़ें।
- `AZURE_AI_SERVICE_ENDPOINT` जोड़ें।
- या `translate -l "ko" -md` जैसे केवल टेक्स्ट कमांड चलाएँ।

### अमान्य कुंजी या एंडपॉइंट

लक्षणों में `401`, रेडैक्टेड अनुमति त्रुटियाँ, या एंडपॉइंट एक्सेस त्रुटियाँ शामिल हो सकती हैं।

समाधान:

- पुष्टि करें कि कुंजी उसी Azure संसाधन से संबंधित है जो एंडपॉइंट से जुड़ा है।
- पुष्टि करें कि `-img` उपयोग करते समय संसाधन Vision का समर्थन करता है।
- पुष्टि करें कि Azure OpenAI डिप्लॉयमेंट नाम और API संस्करण आपकी डिप्लॉयमेंट से मेल खाते हैं।
- डिबग लॉग्स के साथ चलाएँ: `translate -l "ko" -md -d -s`.

## कोई फ़ाइल अनुवादित नहीं हुईं

आम कारण:

- चुने गए फ्लैग आपकी फ़ाइलों से मेल नहीं खाते।
- अनुवादित फ़ाइलें पहले से मौजूद हैं।
- स्रोत फ़ाइलें बहिष्कृत निर्देशिकाओं के अंदर हैं।
- कमांड गलत प्रोजेक्ट रूट से चल रही है।

जाँचें:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

जब कमांड प्रोजेक्ट रूट के बाहर चलाया जाता है तो `--root-dir` का उपयोग करें।

## अनपेक्षित लिंक व्यवहार

लिंक री-राइटिंग चयनित सामग्री प्रकारों पर निर्भर करती है:

- `-nb` शामिल: नोटबुक लिंक अनुवादित नोटबुक्स की ओर इशारा कर सकते हैं।
- `-nb` बाहर रखा गया: नोटबुक लिंक स्रोत नोटबुक की ओर बने रह सकते हैं।
- `-img` शामिल: इमेज लिंक अनुवादित चित्रों की ओर इशारा कर सकते हैं।
- `-img` बाहर रखा गया: इमेज लिंक स्रोत चित्रों की ओर बने रह सकते हैं।

जब सभी आंतरिक लिंक अनुवादित आउटपुट को प्राथमिकता दें, तो पूर्ण सामग्री अनुवाद चलाएँ:

```bash
translate -l "ko" -md -nb -img
```

अनुवाद के बाद लिंक समीक्षा चलाएँ:

```bash
co-op-review -l "ko"
```

## Markdown रेंडरिंग समस्याएँ

यदि अनूदित Markdown गलत तरीके से रेंडर होता है:

- जाँचें कि फ्रंटमैटर `---` से शुरू और समाप्त हो रहा है।
- जाँचें कि कोड फ़ेन्स की गिनती स्रोत और अनुवादित फ़ाइलों के बीच मेल खाती है।
- सामान्य संरचना समस्याओं को पकड़ने के लिए `co-op-review` चलाएँ।
- अगर आउटपुट भ्रष्ट हुआ है तो विशिष्ट फ़ाइल को पुनः-अनुवाद करें।

```bash
co-op-review -l "ko" --format github
```

## GitHub Action चला लेकिन कोई पुल रिक्वेस्ट नहीं बनी

यदि `peter-evans/create-pull-request` रिपोर्ट करता है कि ब्रांच बेस से आगे नहीं है, तो वर्कफ़्लो को कमिट करने के लिए कोई फ़ाइलें नहीं मिलीं।

संभावित कारण:

- अनुवाद रन ने कोई परिवर्तन उत्पन्न नहीं किया।
- `.gitignore` `translations/`, `translated_images/`, या अनूदित नोटबुक्स को बहिष्कृत करता है।
- `add-paths` जेनरेट किए गए आउटपुट निर्देशिकाओं से मेल नहीं खाता।
- अनुवाद चरण समय से पहले समाप्त हुआ।

समाधान:

1. पुष्टि करें कि जेनरेट की गई फ़ाइलें `translations/` या `translated_images/` में मौजूद हैं।
2. पुष्टि करें कि `.gitignore` जेनरेट किए गए आउटपुट को इग्नोर नहीं कर रहा है।
3. Use matching `add-paths`:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. Temporarily add debug flags to the translate command:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. Confirm workflow permissions include:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## अनुवाद गुणवत्ता

मशीन अनुवादों को मानव समीक्षा की आवश्यकता हो सकती है। केवल तब `evaluate` का उपयोग करें जब आप प्रायोगिक गुणवत्ता स्कोरिंग और कम-विश्वास मरम्मत वर्कफ़्लो चाहते हों।

!!! warning "Experimental"
    `evaluate` नियम-आधारित और LLM-आधारित जाँचों का उपयोग कर सकता है, और इसका स्कोरिंग मॉडल और मेटाडेटा व्यवहार बदल सकता है। आवश्यक CI गेट्स से इसे बाहर रखें जब तक कि आपका वर्कफ़्लो परिवर्तनों के लिए तैयार न हो।

निश्चित CI जाँचों के लिए, इसके बजाय `co-op-review` का उपयोग करें।
