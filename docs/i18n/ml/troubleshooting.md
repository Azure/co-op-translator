# പ്രശ്നപരിഹാരം

വിവർത്തന റൺ അനുപേക്ഷിതമായി വിജയിക്കുമ്പോൾ, ക്രമീകരണത്തിൽ പരാജയപ്പെട്ടപ്പോൾ, അല്ലെങ്കിൽ അവലോകനത്തിന് ആവശ്യമായ ഔട്ട്പുട്ട് ഉണ്ടാകുമ്പോൾ ഈ പേജ് ഉപയോഗിക്കു.

## ഇവിടെ ആരംഭിക്കുക

1. ആദ്യം ഒരു കേന്ദ്രീകൃത കമാൻഡ് ഓടിക്കുക, ഉദാഹരണത്തിന് `translate -l "ko" -md`.
2. കോൺസോൾ ഡീബ്ഗ് ലോഗുകൾക്കായി `-d` ചേർക്കുക.
3. ഡീബഗ് ലോഗുകൾ `<root-dir>/logs/`-ലിടെ സംരക്ഷിക്കാൻ `-s` ചേർക്കുക.
4. വിവർത്തനം കഴിഞ്ഞ് അപ്‌ടേറ്റ്നെസ്സ്, ഘടന, എന്നിവയും ലൊക്കൽ ലിങ്കുകളും പരിശോധിക്കാൻ `co-op-review` ഓടിക്കുക.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## ക്രമീകരണ പിശകുകൾ

### ഭാഷാ മോഡൽ പ്രൊവൈഡർ ഇല്ല

പിശക്:

```text
No language model configuration found.
```

പരിഹാരം:

- Azure OpenAI അല്ലെങ്കിൽ OpenAI കോൺഫിഗർ ചെയ്യുക.
- കമാൻഡ് ഓടുന്ന പരിസരങ്ങളിലും ആവശ്യമായ മൂല്യങ്ങൾ (variables) സജ്ജമായുണ്ടെന്ന് സ്ഥിരീകരിക്കുക.
- പ്രാദേശിക ഉപയോഗത്തിനായി, അവ പ്രോജക്ട് റൂട്ടിലുള്ള `.env` ഫയലിലേക്ക് ചേർക്കുക.

കാണുക [Configuration](configuration.md).

### Azure AI Vision ഇല്ലാതെ ഇമേജ് വിവർത്തനം

പിശക്:

```text
Image translation requested but Azure AI Service is not configured.
```

പരിഹാരം:

- `AZURE_AI_SERVICE_API_KEY` ചേർക്കുക.
- `AZURE_AI_SERVICE_ENDPOINT` ചേർക്കുക.
- അല്ലെങ്കിൽ `translate -l "ko" -md` പോലുള്ള ടെക്സ്റ്റ്-മാത്രം കമാൻഡ് ഓടിക്കുക.

### അസാധുവായ കീ അല്ലെങ്കിൽ എന്റ്പോയിന്റ്

ലക്ഷണങ്ങളിൽ `401`, റീഡാക്ടഡ് അനുമതി പിശകുകൾ, അല്ലെങ്കിൽ എന്റ്പോയിന്റ് ആക്‌സസ് പിശകുകൾ ഉൾപ്പെടാം.

പരിഹാരം:

- കീ എന്റ്പോയിന്റുമായി ബന്ധപ്പെട്ടിരിക്കുന്ന അതേ Azure റിസോഴ്‌സിന്റെതാണ് എന്ന് സ്ഥിരീകരിക്കുക.
- `-img` ഉപയോഗിക്കുമ്പോൾ റിസോഴ്‌സ് Vision പിന്തുണയ്ക്കുന്നുണ്ടെന്ന് സ്ഥിരീകരിക്കുക.
- Azure OpenAI ഡിപ്പ്ലോയ്മെന്റ് നാമവും API പതിപ്പും നിങ്ങളുടെ ഡിപ്പ്ലോയ്മെൻറുമായി പൊരുത്തപ്പെടുന്നതാണെന്ന് സ്ഥിരീകരിക്കുക.
- ഡീബഗ് ലോഗുകളോടുകൂടെ ഓടിക്കുക: `translate -l "ko" -md -d -s`.

## ഫയലുകൾ ഒന്നും വിവർത്തനം ചെയ്തില്ല

സാധാരണ原因ങ്ങൾ:

- തിരഞ്ഞെടുക്കപ്പെട്ട ഫ്‌ലാഗുകൾ നിങ്ങളുടെ ഫയലുകളുമായി പൊരുത്തപ്പെടുന്നില്ല.
- ഇതിനകം വിവർത്തനം ചെയ്ത ഫയലുകൾ നിലവിലുണ്ട്.
- സോഴ്‌സ് ഫയലുകൾ ഒഴിവാക്കിയ ഡയറക്ടറികളിൽ ഉണ്ടാകും.
- കമാൻഡ് തെറ്റായ പ്രോജക്ട് റൂട്ടിൽ നിന്ന് ഓടുന്നവയാണ്.

പരിശോധനകൾ:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

കമാൻഡ് പ്രോജക്ട് റൂട്ടിന് പുറത്താണ് ഓടിക്കുന്നത് എങ്കിൽ `--root-dir` ഉപയോഗിക്കുക.

## അപ്രതീക്ഷിത ലിങ്ക് പെരുമാറ്റം

ലിങ്ക് പുനഃരചന തിരഞ്ഞെടുക്കപ്പെട്ട ഉള്ളടക്ക തരംകളിൽ ആശ്രയിച്ചിരിക്കുന്നു:

- `-nb` ഉൾപ്പെടുത്തിയെങ്കിൽ: നോട്ട്‌ബുക്ക് ലിങ്കുകൾ വിവർത്തനം ചെയ്ത നോട്ട്‌ബുക്കുകളിലേക്ക് സൂചിപ്പിക്കാവുന്നതാണ്.
- `-nb` ഒഴിവാക്കിയാൽ: നോട്ട്‌ബുക്ക് ലിങ്കുകൾ സോഴ്‌സ് നോട്ട്‌ബുക്കുകളിലേക്ക് തന്നെയാണ് സൂചിപ്പിച്ചു നിലക്കുക.
- `-img` ഉൾപ്പെടുത്തിയെങ്കിൽ: ഇമേജ് ലിങ്കുകൾ വിവർത്തനം ചെയ്ത ചിത്രങ്ങളിലേക്കാണ് സൂചിപ്പിക്കാവുന്നതെന്ന് കാണാം.
- `-img` ഒഴിവാക്കിയാൽ: ഇമേജ് ലിങ്കുകൾ സോഴ്‌സ് ചിത്രങ്ങളിലേക്കാണ് സൂചിപ്പിച്ചു നിലക്കുക.

എല്ലാ ആന്തരിക ലിങ്കുകളും വിവർത്തന ഔട്ട്പുട്ടുകൾക്കു മുൻഗണന നൽകേണ്ടതുണ്ടെങ്കിൽ പൂർണ്ണ ഉള്ളടക്ക വിവർത്തനം നടത്തുക:

```bash
translate -l "ko" -md -nb -img
```

വിവർത്തനം കഴിഞ്ഞ് ലിങ്ക് റിവ്യൂ നടത്തുക:

```bash
co-op-review -l "ko"
```

## മാർക്ക്ഡൗൺ റെൻഡറിങ് പ്രശ്നങ്ങൾ

വിവർത്തിച്ച മാർക്ക്ഡൗൺ തെറ്റായി പ്രകടമാകുകയാണെങ്കിൽ:

- ഫ്രോൺറ്റ്മാറ്റർ `---` കൊണ്ടാരംഭിക്കുകയും അവസാനിക്കുകയും ചെയ്തിട്ടുണ്ടെന്ന് പരിശോധിക്കുക.
- കോഡ് ഫെൻസ് എണ്ണങ്ങൾ സോഴ്‌സ് ഫയലുടെയും വിവർത്തിത ഫയലിന്റെയും തമ്മിൽ പൊരുത്തപ്പെടുന്നുണ്ടെന്ന് പരിശോധിക്കുക.
- സാധാരണ ഘടനാ പ്രശ്നങ്ങൾ പിടികൂടാൻ `co-op-review` ഓടിക്കുക.
- ഔട്ട്പുട്ട് കേടായിരുന്നുവെങ്കിൽ പ്രത്യേക ഫയൽ വീണ്ടും വിവർത്തനം ചെയ്യുക.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action ഓടിച്ചെങ്കിലും ഒരു പുൾ റിക്വസ്റ്റ് സൃഷ്ടിച്ചില്ല

`peter-evans/create-pull-request` റിപ്പോർട്ട് ചെയ്യുന്നതനുസരിച്ച് ബ്രാഞ്ച് ബേസ് ന്റെ മുന്നിലാണ് ഇല്ലെന്ന് കാണിച്ചാൽ, വേർക്ക്ഫ്ലോയ്ക്ക് commit ചെയ്യാനുള്ള ഫയലുകൾ കണ്ടെത്തിയില്ല.

സാദ്ധ്യതയുള്ള കാരണങ്ങൾ:

- വിവർത്തന റൺ മാറ്റങ്ങൾ ഉണ്ടാക്കിയില്ല.
- `.gitignore` `translations/`, `translated_images/`, അല്ലെങ്കിൽ വിവർത്തിച്ച നോട്ട്‌ബുക്കുകൾ ഒഴിവാക്കുന്നുണ്ടാകാം.
- `add-paths` സൃഷ്ടിച്ച ഔട്ട്പുട്ട് ഡയറക്ടറികളുമായി പൊരുത്തപ്പെടുന്നില്ല.
- വിവർത്തന ഘട്ടം മടങ്ങി വാങ്ങാതെ വിട്ടുപോയി.

പരിഹാരങ്ങൾ:

1. സൃഷ്ടിച്ച ഫയലുകൾ `translations/` അല്ലെങ്കിൽ `translated_images/` ൽ നിലവിലുണ്ടെന്ന് സ്ഥിരീകരിക്കുക.
2. `.gitignore` സൃഷ്ടിച്ച ഔട്ട്പുട്ടുകൾ അവഗണിക്കാത്തതായി ഉറപ്പാക്കുക.
3. പൊരുത്തപ്പെടുന്ന `add-paths` ഉപയോഗിക്കുക:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. താൽക്കാലികമായി `translate` കമാൻഡിൽ ഡീബഗ് ഫ്‌ലാഗുകൾ ചേർക്കുക:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. വേർക്ക്ഫ്ലോ അനുമതികളിൽ ഇവ ഉൾപ്പെട്ടിട്ടുണ്ടെന്ന് സ്ഥിരീകരിക്കുക:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## വിവർത്തന ഗുണമേൻമ

മെഷീൻ വിവർത്തനങ്ങൾക്ക് മനുഷ്യ നിരീക്ഷണം ആവശ്യമാകാം. പരീക്ഷണാത്മക ഗുണനിലവാര സ്കോറിംഗിനും കുറഞ്ഞ-വിശ്വാസപരമായ പരിഹാര പ്രവാഹങ്ങൾക്കുമാത്രമായി `evaluate` ഉപയോഗിക്കുക.

!!! warning "Experimental"
    `evaluate` നിയമാധിഷ്ഠിതവും LLM-ആധാരിതവുമായ പരിശോധനകൾ ഉപയോഗിക്കാവുന്നതാണ്, അതിന്റെ സ്കോറിംഗ് മോഡലും മെറ്റാഡേറ്റാ പെരുമാറ്റവും മാറാൻ സാധ്യതയുണ്ട്. നിങ്ങളുടെ വേർക്ക്ഫ്ലോ മാറ്റങ്ങൾക്ക് തയ്യാറല്ലാത്തവയിൽ ഇത് നിർബന്ധമായ CI ഗേറ്റുകളിൽ ഉൾപ്പെടുത്തരുത്.

    നിർണായക CI പരിശോധനകൾക്കായി പകരം `co-op-review` ഉപയോഗിക്കുക.
