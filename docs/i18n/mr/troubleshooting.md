# समस्या निवारण

हा पृष्ठ वापरा जेव्हा अनुवाद चालवताना अनपेक्षितरीत्या यश मिळते, कॉन्फिगरेशन दरम्यान अयशस्वी होते किंवा आऊटपुट पुनरावलोकनासाठी आवश्यक असते.

## येथे प्रारंभ करा

1. प्रथम एक लक्ष केंद्रित कमांड चालवा, उदा. `translate -l "ko" -md`.
2. कन्सोल डीबग लॉगसाठी `-d` जोडा.
3. डीबग लॉग <root-dir>/logs/ अंतर्गत जतन करण्यासाठी `-s` जोडा.
4. ताजगी, संरचना आणि स्थानिक दुव्यांची तपासणी करण्यासाठी अनुवादानंतर `co-op-review` चालवा.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## कॉन्फिगरेशन त्रुटी

### भाषा मॉडेल प्रदाता नाही

त्रुटी:

```text
No language model configuration found.
```

उपाय:

- Azure OpenAI किंवा OpenAI कॉन्फिगर करा.
- कमांड ज्या पर्यावरणात चालते त्या भागात व्हेरिएबल्स आहेत याची पडताळणी करा.
- स्थानिक वापरासाठी, त्यांना प्रकल्प रूटमधील `.env` मध्ये ठेवा.

पाहा [कॉन्फिगरेशन](configuration.md).

### Azure AI Vision शिवाय प्रतिमा अनुवाद

त्रुटी:

```text
Image translation requested but Azure AI Service is not configured.
```

उपाय:

- `AZURE_AI_SERVICE_API_KEY` जोडा.
- `AZURE_AI_SERVICE_ENDPOINT` जोडा.
- किंवा `translate -l "ko" -md` सारखा फक्त मजकूर आदेश चालवा.

### अवैध की किंवा एन्डपॉइंट

लक्षणांमध्ये `401`, परवानगीसंबंधी त्रुटी (रेडक्टेड), किंवा एन्डपॉइंट प्रवेश त्रुटी यांचा समावेश असू शकतो.

उपाय:

- की एन्डपॉइंटसारख्याच Azure रिसोर्सला संबंधित आहे याची खात्री करा.
- `-img` वापरताना रिसोर्स Vision ला समर्थन करते का ते तपासा.
- Azure OpenAI डिप्लॉयमेंट नाव आणि API आवृत्ती तुमच्या डिप्लॉयमेंटशी जुळतात का ते तपासा.
- डीबग लॉगसह चालवा: `translate -l "ko" -md -d -s`.

## कोणतेही फाइल अनुवादित झाल्या नाहीत

सामान्य कारणे:

- निवडलेले फ्लॅग तुमच्या फाइलशी जुळत नाहीत.
- आधीपासून अनुवादित फाइल्स विद्यमान आहेत.
- स्त्रोत फाइल्स वगळलेल्या निर्देशिकांमध्ये आहेत.
- कमांड चुकीच्या प्रकल्प रूटमधून चालत आहे.

तपासणी:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

कमांड प्रकल्प रूटच्या बाहेर चालवताना `--root-dir` वापरा.

## अनपेक्षित दुवा वर्तन

दुवा पुनर्लेखन निवडलेल्या सामग्री प्रकारांवर अवलंबून असते:

- `-nb` समाविष्ट: नोटबुक दुवे अनुवादित नोटबुककडे निर्देश करू शकतात.
- `-nb` वगळलेले: नोटबुक दुवे स्रोत नोटबुककडेच राहू शकतात.
- `-img` समाविष्ट: प्रतिमा दुवे अनुवादित प्रतिमांकडे निर्देश करू शकतात.
- `-img` वगळलेले: प्रतिमा दुवे स्रोत प्रतिमांकडेच राहू शकतात.

जेव्हा सर्व अंतर्गत दुवे अनुवादित आउटपुटला प्राधान्य द्यायला हवेत तेव्हा संपूर्ण सामग्री अनुवाद चालवा:

```bash
translate -l "ko" -md -nb -img
```

अनुवादानंतर दुवा पुनरावलोकन चालवा:

```bash
co-op-review -l "ko"
```

## Markdown रेंडरिंग समस्या

जर अनुवादित Markdown चुकीने रेंडर होत असेल तर:

- तपासा की फ्रंटमॅटर `---` ने सुरू आणि समाप्त होते का.
- तपासा की स्त्रोत आणि अनुवादित फाइल्समधील कोड फेन्सची संख्या जुळते का.
- सामान्य संरचना समस्या पकडण्यासाठी `co-op-review` चालवा.
- जर आउटपुट भ्रष्ट झाले असेल तर संबंधित फाइल पुन्हा अनुवादित करा.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action चालला परंतु कोणताही Pull Request तयार झाला नाही

जर `peter-evans/create-pull-request` असे अहवाल देत असेल की ब्रांच बेसच्या पुढे नाही, तर वर्कफ्लोने कमिट करण्यास कोणतीही फाइल्स सापडली नाहीत.

संभाव्य कारणे:

- अनुवाद धावण्यानं कोणतेही बदल निर्माण केले नाहीत.
- `.gitignore` मध्ये `translations/`, `translated_images/`, किंवा अनुवादित नोटबुक्स वगळलेले आहेत.
- `add-paths` जनरेट केलेल्या आउटपुट निर्देशिकांशी जुळत नाही.
- अनुवाद पायरी लवकर थांबली.

उपाय:

1. `translations/` किंवा `translated_images/` मध्ये जनरेट झालेल्या फाइल्स अस्तित्वात आहेत याची पुष्टता करा.
2. `.gitignore` जनरेट केलेल्या आउटपुटना इग्नोर करत नाही याची पुष्टी करा.
3. जुळणारे `add-paths` वापरा:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. तात्पुरते `translate` कमांडमध्ये डीबग फ्लॅग जोडा:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. खात्री करा की वर्कफ्लो परवानग्यांमध्ये खालील गोष्टी समाविष्ट आहेत:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## अनुवाद गुणवत्ता

मशीन अनुवादांना मानवी पुनरावलोकनाची आवश्यकता असू शकते. `evaluate` फक्त तेव्हा वापरा जेव्हा तुम्हाला प्रयोगात्मक गुणवत्ता स्कोअरिंग आणि कमी आत्मविश्वासाच्या दुरुस्ती वर्कफ्लो हवे असतील.

!!! warning "Experimental"
    `evaluate` नियम-आधारित आणि LLM-आधारित तपासण्या करू शकते, आणि त्याचे स्कोअरिंग मॉडेल व मेटाडेटा वर्तन बदलू शकते. तुमचा वर्कफ्लो बदलांसाठी तयार नसेल तर ते आवश्यक CI गेटमधून दूर ठेवा.

For deterministic CI checks, use `co-op-review` instead.
