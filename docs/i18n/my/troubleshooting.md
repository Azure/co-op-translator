# ပြဿနာဖြေရှင်းခြင်း

ဘာသာပြန်လည်ပြုလုပ်မှုတစ်ခု မျှော်လင့်ထားသလို အောင်မြင်သွားသည်၊ ဖွဲ့စည်းမှုအတွင်း မအောင်မြင်သွားသည် သို့မဟုတ် ပြန်လည်စစ်ဆေးရန် လိုအပ်သော အထွက်များ ထုတ်ပေးသွားပါက ဤစာမျက်နှာကို အသုံးပြုပါ။

## စတင်ရန်

1. ကျစပ်သော အမိန့်ကို ပထမဦးစွာ ပြေးပါ၊ ဥပမာ `translate -l "ko" -md`။
2. ကွန်ဆိုး debug မှတ်တမ်းများအတွက် `-d` ကို ထည့်ပါ။
3. Debug မှတ်တမ်းများကို `<root-dir>/logs/` အောက်သို့ သိမ်းရန် `-s` ကို ထည့်ပါ။
4. ဘာသာပြန်ပြီးနောက် `co-op-review` ကို ပြေး၍ နောက်ဆုံးအခြေအနေ၊ ဖွဲ့စည်းပုံနှင့် ဒေသীয় လင့်ခ်များကို စစ်ဆေးပါ။

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## ဖွဲ့စည်းမှု အမှားများ

### ဘာသာစကား မော်ဒယ် ပံ့ပိုးသူ မရှိခြင်း

အမှား:

```text
No language model configuration found.
```

ဖြေရှင်းချက်:

- Azure OpenAI သို့မဟုတ် OpenAI ကို ဖော်ပြပါ။
- အမိန့်ကို ပြေးရာ ပတ်ဝန်းကျင်တွင် variables များ သတ်မှတ်ထားကြောင်း စစ်ဆေးပါ။
- ဒေသတွင်း အသုံးပြုမှုအတွက်၊ project root တွင် ရှိသော `.env` ထဲသို့ ထည့်ပါ။

အသေးစိတ်အချက်အလက်များအတွက် [Configuration](configuration.md) ကို ကြည့်ပါ။

### Azure AI Vision မပါတဲ့ အခြေအနေတွင် ရုပ်ပုံဘာသာပြန်ခြင်း

အမှား:

```text
Image translation requested but Azure AI Service is not configured.
```

ဖြေရှင်းချက်:

- `AZURE_AI_SERVICE_API_KEY` ကို ထည့်ပါ။
- `AZURE_AI_SERVICE_ENDPOINT` ကို ထည့်ပါ။
- သို့မဟုတ် စာသားပဲ ပါသော အမိန့် တစ်ခုကို ပြေးပါ၊ ဥပမာ `translate -l "ko" -md`။

### မှားယွင်းသည့် Key သို့မဟုတ် Endpoint

လက္ခဏာများတွင် `401`၊ ခွင့်ပြဿနာများ ပြန်လည်ဖျက်ထားခြင်း သို့မဟုတ် endpoint အသုံးပြုခွင့် အမှားများ ပါဝင်နိုင်သည်။

ဖြေရှင်းချက်:

- key သည် endpoint နှင့်တူညီသော Azure resource ကိုပဲ ပိုင်ဆိုင်ကြောင်း အတည်ပြုပါ။
- `-img` ကို အသုံးပြုသောအခါ resource သည် Vision ကို ထောက်ပံ့ကြောင်း အတည်ပြုပါ။
- Azure OpenAI deployment နာမည်နှင့် API version သည် သင့် deployment နှင့် ကိုက်ညီကြောင်း စစ်ဆေးပါ။
- debug မှတ်တမ်းများဖြင့် ပြေးပါ: `translate -l "ko" -md -d -s`။

## ဘာသာပြန်မပြုလုပ်သည့် ဖိုင်များ

ပုံမှန် ဖြစ်စေသော အကြောင်းရင်းများ:

- ရွေးချယ်ထားသော flags များသည် သင့်ဖိုင်များနှင့် ကိုက်ညီမှုမရှိပါ။
- ဘာသာပြန်ပြီးသား ဖိုင်များ လက်ရှိ ရှိနေပါသည်။
- မူရင်းဖိုင်များသည် ပယ်ဖျက်ထားသော ဒါရိုက်ထရီများအောက်တွင် ရှိနေပါသည်။
- အမိန့်ကို မမှန်သော project root မှ ပြေးဆွဲနေပါသည်။

စစ်ဆေးရန်:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

အမိန့်ကို project root ပြင်ပတွင် ပြေးလျှင် `--root-dir` ကို သုံးပါ။

## မမျှော်လင့်ထားသည့် လင့်ခ် လုပ်ဆောင်မှု

လင့်ခ် ပြန်ရေးခြင်းသည် ရွေးချယ်ထားသော အကြောင်းအရာ အမျိုးအစားများပေါ် မူတည်သည်။

- `-nb` ပါဝင်ပါက: notebook လင့်ခ်များသည် ဘာသာပြန်ထားသော notebook များသို့ ကိုးကားနိုင်သည်။
- `-nb` မပါလျှင်: notebook လင့်ခ်များသည် မူလ notebook များကို ဆက်လက် ကိုးကားထားနိုင်သည်။
- `-img` ပါဝင်ပါက: image လင့်ခ်များသည် ဘာသာပြန်ထားသော ပုံများကို ကိုးကားနိုင်သည်။
- `-img` မပါလျှင်: image လင့်ခ်များသည် မူရင်း ပုံများကို ဆက်လက် ကိုးကားထားနိုင်သည်။

အတွင်းလင့်ခ်အားလုံးကို ဘာသာပြန်ထားသည့် အထွက်များကို ဦးစားပေးစေရန် ပြည့်စုံသော အကြောင်းအရာ ဘာသာပြန်မှုတစ်ခုလုံးကို ပြေးပါ။

```bash
translate -l "ko" -md -nb -img
```

ဘာသာပြန်ပြီးနောက် လင့်ခ် ပြန်လည်စစ်ဆေးပါ။

```bash
co-op-review -l "ko"
```

## Markdown ပြသမှုဆိုင်ရာ ပြဿနာများ

ဘာသာပြန်ထားသော Markdown ကို မမှန်ကန်စွာ ပြသပါက:

- frontmatter သည် `---` ဖြင့် စပြီး `---` ဖြင့် အဆုံးသတ်ထားကြောင်း စစ်ဆေးပါ။
- မူရင်းဖိုင်နှင့် ဘာသာပြန်ဖိုင်အကြား code fence ရေတွက်မှုများ ကိုက်ညီကြောင်း စစ်ဆေးပါ။
- သာမန် ဖွဲ့စည်းပုံ ပြဿနာများကိုဖမ်းရန် `co-op-review` ကို ပြေးပါ။
- အထွက် ပျက်စီးနေခဲ့ပါက သတ်မှတ်ထားသည့် ဖိုင်ကို ပြန်လည်ဘာသာပြန်ပါ။

```bash
co-op-review -l "ko" --format github
```

## GitHub Action ပြေးခဲ့ပေမယ့် Pull Request မဖန်တီးခဲ့ပါ

`peter-evans/create-pull-request` သည် branch သည် base ထက် ရှေ့မမီကြောင်းရပ်တန်ဆာပြခဲ့လျှင် workflow သည် commit သည့် ဖိုင် မရှိကြောင်း တွေ့ရှိခဲ့သည်။

ဖြစ်နိုင်သော အကြောင်းရင်းများ:

- ဘာသာပြန် run သည် ပြောင်းလဲမှု မထုတ်ပေးခဲ့ပါ။
- `.gitignore` သည် `translations/`, `translated_images/`, သို့မဟုတ် translated notebooks များကို ပယ်ဖျက်ထားနိုင်သည်။
- `add-paths` သည် ဖန်တီးထားသော output ဒါရိုက်ထရီများနှင့် ကိုက်ညီမှု မရှိနိုင်သည်။
- ဘာသာပြန် အဆင့်သည် သတ်မှတ်ချိန်မတိုင်မှီ ပိတ်ထားသွားခဲ့သည်။

ဖြေရှင်းနည်းများ:

1. ဖန်တီးထားသော ဖိုင်များသည် `translations/` သို့မဟုတ် `translated_images/` တွင် ရှိကြောင်း အတည်ပြုပါ။
2. `.gitignore` သည် ဖန်တီးထားသော အထွက်များကို ပယ်ဖျက်ထားခြင်း မရှိကြောင်း အတည်ပြုပါ။
3. ကိုက်ညီသော `add-paths` ကို သုံးပါ။

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. ယာယီအားဖြင့် translate အမိန့်တွင် debug flags များ ထည့်ပါ။

   ```bash
   translate -l "ko" -md -d -s
   ```

5. workflow permissions တွင် အောက်ပါ အချက်များ ပါဝင်ကြောင်း အတည်ပြုပါ။

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## ဘာသာပြန် အရည်အသွေး

စက်ရုပ်ဘာသာပြန်ချက်များသည် လူ့စစ်ဆေးမှုကို လိုအပ်နိုင်သည်။ စမ်းလေ့လာရန် အရည်အသွေး ရမှတ်ပေးခြင်းနှင့် ယုံကြည်မှုနည်းသော ပြင်ဆင်မှု workflows များ လိုချင်ပါကသာ `evaluate` ကို အသုံးပြုပါ။

!!! warning "Experimental"
    `evaluate` သည် rule-based နှင့် LLM-based စစ်ဆေးမှုများကို အသုံးပြုနိုင်ပြီး၊ ၎င်း၏ ရမှတ်မော်ဒယ်နှင့် metadata အပြုအမူများ ပြောင်းလဲနိုင်သည်။ သင့် workflow မှ ပြောင်းလဲမှုများကို ကိုင်တွယ်နိုင်ရန်ပြင်ဆင်ထားမထားပါက required CI gates များတွင် ထည့်သွင်းထားရန် မသင့်ပါ။

သတ်မှတ်နိုင်သော CI စစ်ဆေးမှုများအတွက် `co-op-review` ကို အစားထိုးအသုံးပြုပါ။
