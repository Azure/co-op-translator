# समस्या निवारण

यो पृष्ठ अनुवाद चलाउँदा अनपेक्षित रूपमा सफल भएमा, कन्फिगरेसनको क्रममा असफल भएमा, वा समीक्षा आवश्यक आउटपुट उत्पन्न भएमा प्रयोग गर्नुहोस्।

## यहाँबाट सुरु गर्नुहोस्

1. पहिले एक लक्षित आदेश चलाउनुहोस्, जस्तै `translate -l "ko" -md`।
2. कन्सोल डिबग लगहरूको लागि `-d` थप्नुहोस्।
3. डिबग लगहरू `<root-dir>/logs/` भित्र सुरक्षित गर्न `-s` थप्नुहोस्।
4. अनुवादपछि ताजगी, संरचना, र स्थानीय लिङ्कहरू जाँच गर्न `co-op-review` चलाउनुहोस्।

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## कन्फिगरेसन त्रुटिहरू

### भाषा मोडेल प्रदायक छैन

त्रुटि:

```text
No language model configuration found.
```

समाधान:

- Azure OpenAI वा OpenAI कन्फिगर गर्नुहोस्।
- आदेश चल्ने वातावरणमा भेरिएबलहरू छन् कि छैनन् जाँच गर्नुहोस्।
- स्थानीय प्रयोगको लागि, तिनीहरूलाई प्रोजेक्ट रुटमा `.env` मा राख्नुहोस्।

हेर्नुहोस् [कन्फिगरेसन](configuration.md).

### Azure AI Vision बिना छवि अनुवाद

त्रुटि:

```text
Image translation requested but Azure AI Service is not configured.
```

समाधान:

- `AZURE_AI_SERVICE_API_KEY` थप्नुहोस्।
- `AZURE_AI_SERVICE_ENDPOINT` थप्नुहोस्।
- वा `translate -l "ko" -md` जस्तो केवल टेक्स्ट आदेश चलाउनुहोस्।

### अमान्य कुञ्जी वा अन्तबिन्दु

लक्षणहरूमा `401`, छुटाइएको अनुमति त्रुटिहरू, वा अन्तबिन्दु पहुँच त्रुटिहरू समावेश हुन सक्छन्।

समाधान:

- कुञ्जी त्यो अन्तबिन्दुसँग सम्बन्धित एउटै Azure स्रोतको हो कि होईन पुष्टि गर्नुहोस्।
- `-img` प्रयोग गर्दा स्रोतले Vision समर्थन गर्छ कि गर्दैन पुष्टि गर्नुहोस्।
- Azure OpenAI डिप्लोयमेन्ट नाम र API संस्करण तपाइँको डिप्लोयमेन्टसँग मेल खान्छ कि हेर्नुहोस्।
- डिबग लगहरूसहित चलाउनुहोस्: `translate -l "ko" -md -d -s`।

## कुनै फाइल अनुवाद गरिएको छैन

सामान्य कारणहरू:

- छानिएका फ्ल्यागहरू तपाइँका फाइलहरूसँग मेल खाँदैनन्।
- अनुवाद गरिएका फाइलहरू पहिले नै अवस्थित छन्।
- स्रोत फाइलहरू बहिष्कृत निर्देशिकाहरू भित्र छन्।
- आदेश गलत प्रोजेक्ट रुटबाट चलिरहेको छ।

जाँचहरू:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

`--root-dir` प्रयोग गर्नुहोस् जब आदेश प्रोजेक्ट रुट बाहिरबाट चलाइँदैछ।

## अनपेक्षित लिङ्क व्यवहार

लिङ्क पुनर्लेखन छानिएका सामग्री प्रकारहरूमा निर्भर हुन्छ:

- `-nb` समावेश: नोटबुक लिङ्कहरू अनुवादित नोटबुकहरूतर्फ संकेत गर्न सक्छन्।
- `-nb` बहिष्कृत: नोटबुक लिङ्कहरू स्रोत नोटबुकहरूमा संकेत गरिरहन सक्छन्।
- `-img` समावेश: छवि लिङ्कहरू अनुवादित छविहरूमा संकेत गर्न सक्छन्।
- `-img` बहिष्कृत: छवि लिङ्कहरू स्रोत छविहरूमा संकेत गरिरहन सक्छन्।

जब सबै आन्तरिक लिङ्कहरूले अनुवादित आउटपुटहरूलाई प्राथमिकता दिनु पर्छ भने पूर्ण सामग्री अनुवाद चलाउनुहोस्:

```bash
translate -l "ko" -md -nb -img
```

अनुवादपछि लिङ्क समीक्षा चलाउनुहोस्:

```bash
co-op-review -l "ko"
```

## Markdown रेंडरिङ समस्या

यदि अनुवादित Markdown गलत रूपमा रेंडर हुन्छ भने:

- फ्रन्टम्याटर `---` बाट सुरु र समाप्त हुन्छ कि गर्दैन जाँच गर्नुहोस्।
- स्रोत र अनुवादित फाइलबीच कोड फेन्सको गणना मिल्छ कि गर्दैन जाँच गर्नुहोस्।
- `co-op-review` चलाएर सामान्य संरचना समस्याहरू पत्ता लगाउनुहोस्।
- यदि आउटपुट बिग्रिएको छ भने सो विशेष फाइल पुनः अनुवाद गर्नुहोस्।

```bash
co-op-review -l "ko" --format github
```

## GitHub Action चल्यो तर कुनै Pull Request सिर्जना भएन

यदि `peter-evans/create-pull-request` ले रिपोर्ट गर्छ कि शाखा बेस भन्दा अगाडि छैन भने, वर्कफ्लोले कमिट गर्ने कुनै फाइल फेला पारेन।

संभावित कारणहरू:

- अनुवाद रनले कुनै परिवर्तन उत्पादन गरेन।
- `.gitignore` ले `translations/`, `translated_images/`, वा अनुवादित नोटबुकहरूलाई बहिष्कृत गरेको छ।
- `add-paths` ले उत्पन्न आउटपुट निर्देशिकाहरूसँग मेल खाँदैन।
- अनुवाद चरणले छिट्टै समाप्त भयो।

समाधानहरू:

1. पुष्टि गर्नुहोस् कि उत्पन्न फाइलहरू `translations/` वा `translated_images/` मा अवस्थित छन्।
2. पुष्टि गर्नुहोस् कि `.gitignore` ले उत्पन्न आउटपुटहरूलाई इग्नोर गरेको छैन।
3. मेल खाने `add-paths` प्रयोग गर्नुहोस्:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. अस्थायी रूपमा translate आदेशमा डिबग फ्ल्यागहरू थप्नुहोस्:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. पुष्टि गर्नुहोस् कि वर्कफ्लो अनुमतिहरूमा समावेश छन्:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## अनुवाद गुणस्तर

मेसिन अनुवादहरूमा मानव समीक्षा आवश्यक पर्न सक्छ। प्रयोग गर्नुहोस् `evaluate` केवल जब तपाईं प्रायोगिक गुणस्तर स्कोरिङ र कम-विश्वास मर्मत वर्कफ्लोहरू चाहनुहुन्छ।

!!! warning "Experimental"
    `evaluate` ले नियम-आधारित र LLM-आधारित जाँचहरू प्रयोग गर्न सक्छ, र यसको स्कोरिङ मोडेल र मेटाडाटा व्यवहार परिवर्तन हुन सक्छ। तपाईंको वर्कफ्लो परिवर्तनका लागि तयार नभएसम्म यसलाई आवश्यक CI गेटहरूबाट बाहिर राख्नुहोस्।

For deterministic CI checks, use `co-op-review` instead.
