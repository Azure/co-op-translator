# ਸਮੱਸਿਆ ਨਿਵਾਰਣ

ਇਸ ਪੇਜ਼ ਨੂੰ ਉਸ ਵੇਲੇ ਵਰਤੋ ਜਦੋਂ ਇੱਕ ਅਨੁਵਾਦ ਰਨ ਅਚਾਨਕ ਸਫਲ ਹੋ ਜਾਵੇ, ਕਨਫਿਗਰੇਸ਼ਨ ਦੌਰਾਨ ਫੇਲ ਹੋਵੇ, ਜਾਂ ਐਸਾ ਆউਟਪੁੱਟ ਨਿਕਲੇ ਜੋ ਸਮੀਖਿਆ ਦੀ ਲੋੜ ਰੱਖਦਾ ਹੋਵੇ।

## ਇੱਥੋਂ ਸ਼ੁਰੂ ਕਰੋ

1. ਪਹਿਲਾਂ ਇੱਕ ਫੋਕਸਡ ਕਮਾਂਡ ਚਲਾਓ, ਜਿਵੇਂ `translate -l "ko" -md`.
2. ਕੰਸੋਲ ਡੀਬੱਗ ਲੌਗ ਲਈ `-d` ਜੋੜੋ।
3. `-s` ਨੂੰ ਡੀਬੱਗ ਲੌਗਾਂ ਨੂੰ `<root-dir>/logs/` ਵਿੱਚ ਸੇਵ ਕਰਨ ਲਈ ਜੋੜੋ।
4. ਅਨੁਵਾਦ ਤੋਂ ਬਾਅਦ ਤਾਜ਼ਗੀ, ਬਣਤਰ, ਅਤੇ ਲੋਕਲ ਲਿੰਕ ਜਾਂਚਣ ਲਈ `co-op-review` ਚਲਾਓ।

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## ਕੰਫਿਗਰੇਸ਼ਨ ਦੀਆਂ ਗਲਤੀਆਂ

### ਭਾਸ਼ਾ ਮਾਡਲ ਪ੍ਰਦਾਤਾ ਨਹੀਂ

ਗਲਤੀ:

```text
No language model configuration found.
```

ਹੱਲ:

- Azure OpenAI ਜਾਂ OpenAI ਨੂੰ ਕਨਫਿਗਰ ਕਰੋ।
- ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਵੈਰੀਏਬਲ ਉਹਨਾਂ ਐਨਵਾਇਰਨਮੈਂਟ ਵਿੱਚ ਹਨ ਜਿੱਥੇ ਕਮਾਂਡ ਚਲਦੀ ਹੈ।
- ਲੋਕਲ ਵਰਤੋਂ ਲਈ, ਪ੍ਰੋਜੈਕਟ ਰੂਟ ਵਿੱਚ `.env` ਵਿੱਚ ਇਨ੍ਹਾਂ ਨੂੰ ਰੱਖੋ।

ਵੇਖੋ [ਕੰਫਿਗਰੇਸ਼ਨ](configuration.md).

### Azure AI Vision ਦੇ ਬਿਨਾਂ ਚਿੱਤਰ ਅਨੁਵਾਦ

ਗਲਤੀ:

```text
Image translation requested but Azure AI Service is not configured.
```

ਹੱਲ:

- `AZURE_AI_SERVICE_API_KEY` ਜੋੜੋ।
- `AZURE_AI_SERVICE_ENDPOINT` ਜੋੜੋ।
- ਜਾਂ `translate -l "ko" -md` ਵਰਗੇ ਸਿਰਫ਼ ਟੈਕਸਟ ਕਮਾਂਡ ਚਲਾਓ।

### ਅਵੈਧ ਕੁੰਜੀ ਜਾਂ ਐਂਡਪੌਇੰਟ

ਲਛਣਾਂ ਵਿੱਚ `401`, ਰੀਡੈਕਟ ਕੀਤੀਆਂ ਪਰਮਿਸ਼ਨ ਐਰਰ, ਜਾਂ ਐਂਡਪੋਇੰਟ ਪਹੁੰਚ ਐਰਰ ਸ਼ਾਮਲ ਹੋ ਸਕਦੇ ਹਨ।

ਹੱਲ:

- ਪੱਕਾ ਕਰੋ ਕਿ ਕੁੰਜੀ ਉਸੇ Azure ਰਿਸੋਰਸ ਨਾਲ ਸਬੰਧਿਤ ਹੈ ਜੋ ਐਂਡਪੋਇੰਟ ਲਈ ਵਰਤਿਆ ਜਾ ਰਿਹਾ ਹੈ।
- ਪੱਕਾ ਕਰੋ ਕਿ ਰਿਸੋਰਸ `-img` ਵਰਤਣ ਸਮੇਂ Vision ਨੂੰ ਸਮਰਥਨ ਦਿੰਦਾ ਹੈ।
- ਪੱਕਾ ਕਰੋ ਕਿ Azure OpenAI ਡਿਪਲਾਇਮੈਂਟ ਨਾਮ ਅਤੇ API ਵਰਜਨ ਤੁਹਾਡੇ ਡਿਪਲਾਇਮੈਂਟ ਨਾਲ ਮਿਲਦੇ ਹਨ।
- ਡੀਬੱਗ ਲੌਗਾਂ ਦੇ ਨਾਲ ਚਲਾਓ: `translate -l "ko" -md -d -s`.

## ਕਿਸੇ ਫਾਇਲ ਦਾ ਅਨੁਵਾਦ ਨਹੀਂ ਹੋਇਆ

ਆਮ ਕਾਰਨ:

- ਚੁਣੇ ਹੋਏ ਫਲੈਗ ਤੁਹਾਡੀਆਂ ਫਾਇਲਾਂ ਨਾਲ ਮਿਲਦੇ ਨਹੀਂ।
- ਮੌਜੂਦਾ ਅਨੁਵਾਦ ਕੀਤੀਆਂ ਫਾਇਲਾਂ ਪਹਿਲਾਂ ਹੀ ਮੌਜੂਦ ਹਨ।
- ਸਰੋਤ ਫਾਇਲਾਂ ਐਸੇ ਡਾਇਰੈਕਟਰੀਆਂ ਹੇਠਾਂ ਹਨ ਜੋ ਬਾਹਰ ਰੱਖੀਆਂ ਗਈਆਂ ਹਨ।
- ਕਮਾਂਡ ਗਲਤ ਪ੍ਰੋਜੈਕਟ ਰੂਟ ਤੋਂ ਚਲ ਰਹੀ ਹੈ।

ਜਾਂਚਾਂ:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

ਜਦੋਂ ਕਮਾਂਡ ਪ੍ਰੋਜੈਕਟ ਰੂਟ ਦੇ ਬਾਹਰ ਚਲਾਈ ਜਾਂਦੀ ਹੋਵੇ ਤਾਂ `--root-dir` ਵਰਤੋ।

## ਅਣਉਮੀਦਤ ਲਿੰਕ ਵਰਤਾਰਾ

ਲਿੰਕ ਰੀਰਾਈਟ ਕਰਨਾ ਚੁਣੀਆਂ ਸਮੱਗਰੀ ਕਿਸਮਾਂ 'ਤੇ ਨਿਰਭਰ ਕਰਦਾ ਹੈ:

- `-nb` ਸ਼ਾਮਿਲ: ਨੋਟਬੁੱਕ ਲਿੰਕ ਅਨੁਵਾਦ ਕੀਤੀਆਂ ਨੋਟਬੁੱਕ ਵੱਲ ਸੰਕੇਤ ਕਰ ਸਕਦੇ ਹਨ।
- `-nb` ਬਾਹਰ ਰਹੇ: ਨੋਟਬੁੱਕ ਲਿੰਕ ਸੋਰਸ ਨੋਟਬੁੱਕ ਵੱਲ ਹੀ ਰਹਿ ਸਕਦੇ ਹਨ।
- `-img` ਸ਼ਾਮਿਲ: ਚਿੱਤਰ ਲਿੰਕ ਅਨੁਵਾਦ ਕੀਤੀਆਂ ਚਿੱਤਰਾਂ ਵੱਲ ਸੰਕੇਤ ਕਰ ਸਕਦੇ ਹਨ।
- `-img` ਬਾਹਰ ਰਹੇ: ਚਿੱਤਰ ਲਿੰਕ ਸੋਰਸ ਚਿੱਤਰਾਂ ਵੱਲ ਹੀ ਰਹਿ ਸਕਦੇ ਹਨ।

ਜਦੋਂ ਸਾਰੇ ਅੰਦਰੂਨੀ ਲਿੰਕ ਤਰਜمه ਕੀਤੇ ਨਤੀਜਿਆਂ ਨੂੰ ਤਰਜੀਹ ਦੇਣੇ ਚਾਹੀਦੇ ਹਨ ਤਾਂ ਪੂਰਾ ਸਮੱਗਰੀ ਅਨੁਵਾਦ ਚਲਾਓ:

```bash
translate -l "ko" -md -nb -img
```

ਅਨੁਵਾਦ ਤੋਂ ਬਾਅਦ ਲਿੰਕ ਸਮીਖਿਆ ਚਲਾਓ:

```bash
co-op-review -l "ko"
```

## Markdown ਰੇਂਡਰਿੰਗ ਸਮੱਸਿਆਵਾਂ

ਜੇ ਅਨੁਵਾਦ ਕੀਤਾ Markdown ਗਲਤ ਤਰ੍ਹਾਂ ਰੇਂਡਰ ਹੁੰਦਾ ਹੈ:

- ਚੈੱਕ ਕਰੋ ਕਿ ਫਰੰਟਮੇਟਰ `---` ਨਾਲ ਸ਼ੁਰੂ ਅਤੇ ਖਤਮ ਹੁੰਦਾ ਹੈ।
- ਚੈੱਕ ਕਰੋ ਕਿ ਕੋਡ ਫੈਂਸ ਦੀਆਂ ਗਿਣਤੀਆਂ ਸੋਰਸ ਅਤੇ ਅਨੁਵਾਦ ਕੀਤੀਆਂ ਫਾਇਲਾਂ ਵਿੱਚ ਮਿਲਦੀਆਂ ਹਨ।
- ਆਮ ਬਣਤਰ ਸਮੱਸਿਆਵਾਂ ਫੜਣ ਲਈ `co-op-review` ਚਲਾਓ।
- ਜੇ ਆਊਟਪੁੱਟ ਖਰਾਬ ਹੋਇਆ ਸੀ ਤਾਂ ਉਸ ਖਾਸ ਫਾਇਲ ਨੂੰ ਦੁਬਾਰਾ ਅਨੁਵਾਦ ਕਰੋ।

```bash
co-op-review -l "ko" --format github
```

## GitHub Action ਚਲਿਆ ਪਰ ਕੋਈ Pull Request ਨਹੀਂ ਬਣੀ

ਜੇ `peter-evans/create-pull-request` ਰਿਪੋਰਟ ਕਰਦਾ ਹੈ ਕਿ ਬ੍ਰਾਂਚ ਬੇਸ ਤੋਂ ਅੱਗੇ ਨਹੀਂ ਹੈ, ਤਾਂ ਵਰਕਫਲੋ ਨੇ commit ਕਰਨ ਲਈ ਕੋਈ ਫਾਇਲਾਂ ਨਹੀਂ ਲੱਭੀਆਂ।

ਸੰਭਾਵਤ ਕਾਰਨ:

- ਅਨੁਵਾਦ ਰਨ ਨੇ ਕੋਈ ਬਦਲਾਅ ਨਹੀਂ ਬਣਾਏ।
- `.gitignore` `translations/`, `translated_images/`, ਜਾਂ ਅਨੁਵਾਦ ਕੀਤੀਆਂ ਨੋਟਬੁੱਕਾਂ ਨੂੰ ਬਾਹਰ ਰੱਖਦਾ ਹੈ।
- `add-paths` ਜਨਰੇਟ ਕੀਤੀਆਂ ਆਉਟਪੁੱਟ ਡਾਇਰੈਕਟਰੀਆਂ ਨਾਲ ਮੇਲ ਨਹੀਂ ਖਾਂਦਾ।
- ਅਨੁਵਾਦ ਕਦਮ ਪਹਿਲਾਂ ਹੀ ਬੰਦ ਹੋ ਗਿਆ।

ਸੁਧਾਰ:

1. ਪੱਕਾ ਕਰੋ ਕਿ ਜਨਰੇਟ ਕੀਤੀਆਂ ਫਾਇਲਾਂ `translations/` ਜਾਂ `translated_images/` ਵਿੱਚ ਮੌਜੂਦ ਹਨ।
2. ਪੱਕਾ ਕਰੋ ਕਿ `.gitignore` ਜਨਰੇਟ ਕੀਤੇ ਆਉਟਪੁੱਟਾਂ ਨੂੰ ਨਜ਼ਰਅੰਦਾਜ਼ ਨਹੀਂ ਕਰ ਰਿਹਾ।
3. ਮੈਚਿੰਗ `add-paths` ਵਰਤੋਂ:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. ਅਸਥਾਈ ਤੌਰ 'ਤੇ translate ਕਮਾਂਡ ਵਿੱਚ ਡੀਬੱਗ ਫਲੈਗ ਜੋੜੋ:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. ਪੱਕਾ ਕਰੋ ਕਿ workflow ਅਧਿਕਾਰ ਸ਼ਾਮਲ ਹਨ:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## ਅਨੁਵਾਦ ਦੀ ਗੁਣਵੱਤਾ

ਮਸ਼ੀਨੀ ਅਨੁਵਾਦਾਂ ਨੂੰ ਮਨੁੱਖੀ ਸਮੀਖਿਆ ਦੀ ਲੋੜ ਹੋ ਸਕਦੀ ਹੈ। ਸਿਰਫ਼ ਉਸ ਸਮੇਂ `evaluate` ਵਰਤੋ ਜਦੋਂ ਤੁਸੀਂ ਪ੍ਰਯੋਗਾਤਮਕ ਗੁਣਵੱਤਾ ਸਕੋਰਿੰਗ ਅਤੇ ਘੱਟ ਵਿਸ਼ਵਾਸ ਵਾਲੇ ਰੀਪੇਅਰ ਵਰਕਫਲੋਜ਼ ਚਾਹੁੰਦੇ ਹੋ।

!!! warning "ਪ੍ਰਯੋਗਾਤਮਕ"
    `evaluate` ਨਿਯਮ-ਅਧਾਰਿਤ ਅਤੇ LLM-ਅਧਾਰਿਤ ਚੈਕ ਵਰਤ ਸਕਦਾ ਹੈ, ਅਤੇ ਇਸ ਦਾ ਸਕੋਰਿੰਗ ਮਾਡਲ ਅਤੇ ਮੈਟਾ ਡਾਟਾ ਵਿਹਾਰ ਬਦਲ ਸਕਦਾ ਹੈ। ਇਸਨੂੰ ਲਾਜ਼ਮੀ CI ਗੇਟਾਂ ਤੋਂ ਬਾਹਰ ਰੱਖੋ ਜਦ ਤੱਕ ਤੁਹਾਡਾ ਵਰਕਫਲੋ ਬਦਲਾਵਾਂ ਲਈ ਤਿਆਰ ਨਾ ਹੋਵੇ।

For deterministic CI checks, use `co-op-review` instead.
