# Устранение неполадок

Используйте эту страницу, когда процесс перевода завершается неожиданно успешно, прерывается на этапе конфигурации или генерирует результат, требующий проверки.

## Начало работы

1. Сначала выполните целевую команду, например `translate -l "ko" -md`.
2. Добавьте `-d` для вывода отладочных логов в консоль.
3. Добавьте `-s`, чтобы сохранить отладочные логи в каталоге `<root-dir>/logs/`.
4. Запустите `co-op-review` после перевода, чтобы проверить актуальность, структуру и локальные ссылки.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## Ошибки конфигурации

### Отсутствует поставщик языковой модели

Ошибка:

```text
No language model configuration found.
```

Исправление:

- Настройте Azure OpenAI или OpenAI.
- Убедитесь, что переменные доступны в окружении, где выполняется команда.
- Для локального использования поместите их в `.env` в корне проекта.

См. [Конфигурация](configuration.md).

### Перевод изображений без Azure AI Vision

Ошибка:

```text
Image translation requested but Azure AI Service is not configured.
```

Исправление:

- Добавьте `AZURE_AI_SERVICE_API_KEY`.
- Добавьте `AZURE_AI_SERVICE_ENDPOINT`.
- Или выполните команду только для текста, например `translate -l "ko" -md`.

### Неверный ключ или конечная точка

Симптомы могут включать `401`, замаскированные ошибки разрешений или ошибки доступа к конечной точке.

Исправление:

- Убедитесь, что ключ принадлежит тому же ресурсу Azure, что и конечная точка.
- Убедитесь, что ресурс поддерживает Vision при использовании `-img`.
- Убедитесь, что имя развертывания Azure OpenAI и версия API соответствуют вашему развертыванию.
- Запустите с отладочными логами: `translate -l "ko" -md -d -s`.

## Не были переведены файлы

Типичные причины:

- Выбранные флаги не соответствуют вашим файлам.
- Переводы уже присутствуют (существуют переведённые файлы).
- Исходные файлы находятся в исключённых каталогах.
- Команда выполняется из неверного корня проекта.

Проверки:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

Используйте `--root-dir`, когда команда запускается вне корня проекта.

## Неожиданное поведение ссылок

Переписывание ссылок зависит от выбранных типов содержимого:

- `-nb` включён: ссылки на ноутбуки могут указывать на переведённые ноутбуки.
- `-nb` исключён: ссылки на ноутбуки могут оставаться указывающими на исходные ноутбуки.
- `-img` включён: ссылки на изображения могут указывать на переведённые изображения.
- `-img` исключён: ссылки на изображения могут оставаться указывающими на исходные изображения.

Запустите полный перевод содержимого, когда все внутренние ссылки должны предпочитать переведённые версии:

```bash
translate -l "ko" -md -nb -img
```

Проведите проверку ссылок после перевода:

```bash
co-op-review -l "ko"
```

## Проблемы с рендерингом Markdown

Если переведённый Markdown отображается некорректно:

- Проверьте, что frontmatter начинается и заканчивается на `---`.
- Проверьте, что количество кодовых ограждений совпадает между исходными и переведёнными файлами.
- Запустите `co-op-review`, чтобы обнаружить распространённые структурные проблемы.
- Перевыполните перевод конкретного файла, если выходной результат был повреждён.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action запущен, но Pull Request не был создан

Если `peter-evans/create-pull-request` сообщает, что ветка не опережает базовую, то workflow не нашёл файлов для коммита.

Возможные причины:

- Запуск перевода не привёл к изменениям.
- В `.gitignore` указаны исключения для `translations/`, `translated_images/` или переведённых ноутбуков.
- `add-paths` не соответствует сгенерированным директориям вывода.
- Шаг перевода завершился преждевременно.

Исправления:

1. Убедитесь, что сгенерированные файлы существуют в `translations/` или `translated_images/`.
2. Убедитесь, что `.gitignore` не игнорирует сгенерированные результаты.
3. Используйте соответствующие `add-paths`:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. Временно добавьте флаги отладки в команду translate:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. Убедитесь, что разрешения workflow включают:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## Качество перевода

Машинные переводы могут требовать проверки человеком. Используйте `evaluate` только когда вы хотите экспериментальную оценку качества и рабочие процессы исправления с низкой уверенностью.

!!! warning "Experimental"
    `evaluate` может использовать проверки на базе правил и LLM, а его модель оценки и поведение метаданных могут измениться. Не включайте его в обязательные проверки CI, если ваш рабочий процесс не готов к изменениям.

Для детерминированных проверок в CI используйте вместо этого `co-op-review`.
