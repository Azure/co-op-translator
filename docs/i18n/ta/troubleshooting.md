# பிரச்சினை நீக்குதல்

இந்தப் பக்கத்தை மொழிபெயர்ப்பு ஓட்டம் எதிர்பாராத முறையில் வெற்றியடைந்தால், கட்டமைப்பில் தோல்வியுற்றால், அல்லது பரிசீலனை தேவைப்படும் அவுட்புட் உற்பத்தி செய்தால் பயன்படுத்தவும்.

## இங்கே தொடங்கவும்

1. முதலில் கவனம் செலுத்தப்பட்ட கட்டளையை இயக்கவும், உதாரணத்திற்கு `translate -l "ko" -md`.
2. கான்சோல் டீபக் பதிவுகளிற்கு `-d` ஐச் சேர்க்கவும்.
3. `-s` ஐ பதிவு பதிவுகளை `<root-dir>/logs/` இல் சேமிக்கச் சேர்க்கவும்.
4. மொழிபெயர்ப்புக்குப் பிறகு `co-op-review` ஐ இயக்கி புதுமை, கட்டமைப்பு, மற்றும் உள்ளூர் இணைப்புகளை சரிபார்க்கவும்.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## கட்டமைப்பு பிழைகள்

### மொழி மாதிரி வழங்குநர் இல்லை

பிழை:

```text
No language model configuration found.
```

தீர்வு:

- Azure OpenAI அல்லது OpenAI ஐ கட்டமைக்கவும்.
- கட்டளை இயங்கும் சூழலில் மாற்றி மாறிலிகள் உள்ளனவா என்பதை உறுதிசெய்யவும்.
- உள்ளூர்த் பயன்பாட்டிற்காக, அவற்றை திட்டத்தின் ரூட் இல் உள்ள `.env` கோப்பில் வைக்கவும்.

பாருங்கள் [கட்டமைப்பு](configuration.md).

### Azure AI Vision இல்லாமல் படப் மொழிபெயர்ப்பு

பிழை:

```text
Image translation requested but Azure AI Service is not configured.
```

தீர்வு:

- `AZURE_AI_SERVICE_API_KEY` ஐச் சேர்க்கவும்.
- `AZURE_AI_SERVICE_ENDPOINT` ஐச் சேர்க்கவும்.
- அல்லது `translate -l "ko" -md` போன்ற உரை மட்டமான கட்டளையை இயக்கு.

### தவறான விசை அல்லது எண்ட்பாயிண்ட்

அறிகுறிகளில் `401`, மறைக்கப்பட்ட அனுமதி பிழைகள், அல்லது எண்ட்பாயிண்ட் அணுகல் பிழைகள் ஆகியவை இருக்கும்.

தீர்வு:

- விசை எண்ட்பாயிண்டுடன் ஒரே Azure வளத்திற்கு சொந்தமா என்பதைக் உறுதிப்படுத்தவும்.
- `-img` பயன்படுத்தும்போது அந்த வளம் Vision-ஐ ஆதரிக்கிறதா என்பதை உறுதிசெய்யவும்.
- Azure OpenAI deployment பெயரும் API பதிப்பும் உங்கள் பிரசாரத்துடன் ஒத்திருக்கிறதா என்பதை உறுதிசெய்யவும்.
- டீபக் பதிவுகளுடன் இயக்கவும்: `translate -l "ko" -md -d -s`.

## எந்த கோப்புகளும் மொழிபெயர்க்கப்படவில்லை

பொதுவான காரணங்கள்:

- தேர்ந்தெடுக்கப்பட்ட கொடிகள் உங்கள் கோப்புகளுக்கு பொருந்தவில்லை.
- இருப்பு மொழிபெயர்க்கப்பட்ட கோப்புகள் ஏற்கனவே உள்ளன.
- மூலக் கோப்புகள் நீக்கப்பட்ட அடைவுகளில் உள்ளன.
- கட்டளை தவறான திட்ட ரூட்டிலிருந்து இயக்கப்படுகிறது.

சரிபார்ப்புகள்:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

கட்டளை திட்ட ரூட் வெளியே இயங்கும் போது `--root-dir` பயன்படுத்தவும்.

## எதிர்பாராத இணைப்பு நடத்தை

இணைப்புகளை மாற்றுவது தேர்ந்தெடுக்கப்பட்ட உள்ளடக்க வகைகளின் அடிப்படையிலானது:

- `-nb` சேர்க்கப்பட்டால்: நோட்புக் இணைப்புகள் மொழிபெயர்க்கப்பட்ட நோட்புக் களை நோக்கலாம்.
- `-nb` நீக்கப்பட்டால்: நோட்புக் இணைப்புகள் மூல நோட்புக் களை நோக்கியிருந்தே இருக்கலாம்.
- `-img` சேர்க்கப்பட்டால்: பட இணைப்புகள் மொழிபெயர்க்கப்பட்ட படங்களை நோக்கலாம்.
- `-img` நீக்கப்பட்டால்: பட இணைப்புகள் மூல படங்களை நோக்கியிருந்தே இருக்கலாம்.

உள்ளக அனைத்து இணைப்புகளும் மொழிபெயர்க்கப்பட்ட அவுட்புட்களை முன்னுரிமை கொடுக்கவேண்டுமானால் முழு உள்ளடக்க மொழிபெயர்ப்பை இயக்கவும்:

```bash
translate -l "ko" -md -nb -img
```

மொழிபெயர்ப்புக்குப் பிறகு இணைப்பு மதிப்பாய்வை இயக்கவும்:

```bash
co-op-review -l "ko"
```

## மார்க்டவுன் காட்சிப்படுத்தல் சிக்கல்கள்

மொழிபெயர்க்கப்பட்ட மார்க்டவுன் தவறாக காட்சியளித்தால்:

- frontmatter `---` மூலம் துவங்கி முடிவடைகிறது என்பதை சரிபார்க்கவும்.
- குறியீடு கம்பங்கள் (code fence) எண்ணிக்கை மூலமும் மொழிபெயர்க்கப்பட்ட கோப்புகளும் ஒத்திருக்கிறதா என்பதைச் சரிபார்க்கவும்.
- பொதுவான கட்டமைப்பு சிக்கல்களை பிடிக்க `co-op-review` ஐ இயக்கவும்.
- வெளியீடு கெட்டிருந்தால் குறிப்பிட்ட கோப்பை மீண்டும் மொழிபெயர்க்கவும்.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action இயக்கப்பட்டது ஆனால் எந்த Pull Request உருவாகவில்லை

`peter-evans/create-pull-request` கிளை அடித்தளத்திற்குப் முன்னிலையிலல்ல என்று റിപ്പോർட் செய்தால், வேலைநடை commit செய்ய கோப்புகளைப் பதிவு செய்யவில்லை.

சாத்தியமான காரணங்கள்:

- மொழிபெயர்ப்பு ஓட்டம் எந்த மாற்றங்களையும் உண்டாக்கவில்லை.
- `.gitignore` `translations/`, `translated_images/`, அல்லது மொழிபெயர்க்கப்பட்ட நோட்புக் களை புறக்கணிக்கிறது.
- `add-paths` உருவாக்கப்பட்ட வெளியீடு அடைவுகளிற்கு பொருந்தவில்லை.
- மொழிபெயர்ப்பு படி முன்னதாகவே நிறுத்தப்பட்டது.

தீர்வுகள்:

1. உருவாக்கப்பட்ட கோப்புகள் `translations/` அல்லது `translated_images/` இல் உள்ளனவா என்பதை உறுதிப்படுத்தவும்.
2. `.gitignore` உருவாக்கப்பட்ட வெளியீடுகளை புறக்கணிக்கவில்லை என்பதை உறுதிசெய்யவும்.
3. ஒத்த `add-paths` ஐ பயன்படுத்தவும்:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. தற்காலிகமாக டீபக் கொடிகளை translate கட்டளைக்குச் சேர்க்கவும்:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. வேலைநடை அனுமதிகளில் கீழ்காணும் உள்ளனவா என்பதை உறுதிசெய்யவும்:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## மொழிபெயர்ப்பு தரம்

யந்திர மொழிபெயர்ப்புகள் மனிதப் பரிசீலனை தேவைப்படலாம். நீங்கள் பரிசோதனை தர மதிப்பீடு மற்றும் குறைந்த நம்பகத்தன்மை சரிசெய்தல் வேலைநடைகளை வேண்டுமெனில் மட்டுமே `evaluate` பயன்படுத்துங்கள்.

!!! warning "பரிசோதனை"
    `evaluate` விதி-அடிப்படையிலான மற்றும் LLM-அடிப்படையிலான சோதனைகளைப் பயன்படுத்தலாம், அதன் மதிப்பீட்டு மாதிரி மற்றும் மெட்டாடேட்டாவின் நடத்தை மாற்றப்படலாம். உங்கள் வேலைநடை மாற்றங்களுக்கு தயார் இல்லையெனில் அதை தேவையான CI கட்டுகளில் சேர்க்காதீர்கள்.

தீர்மானிக்கக்கூடிய CI சோதனைகளுக்காக, அதற்கு பதிலாக `co-op-review` ஐப் பயன்படுத்தவும்.
