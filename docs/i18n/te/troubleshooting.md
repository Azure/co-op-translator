# సమస్య పరిష్కారం

అనువాద రన్ అనుకోకుండా విజయవంతమైందా, కాన్ఫిగరేషన్ సమయంలో విఫలమైంది లేదా సమీక్ష అవసరమైన ఔట్‌పుట్ ఉత్పత్తి అయిందా అన్న సందర్భాల్లో ఈ పేజీని ఉపయోగించండి.

## ఇక్కడ మొదలుపెట్టండి

1. ముందుగా లక్ష్యంగా ఉన్న కమాండ్‌ను నడిపి చూడండి, ఉదాహరణకు `translate -l "ko" -md`.
2. కన్సోల్ డీబగ్ లాగ్‌ల కోసం `-d` ను జోడించండి.
3. డీబగ్ లాగ్‌లను `<root-dir>/logs/` కింద సేవ్ చేయడానికి `-s` ను జోడించండి.
4. తాజాతనం, నిర్మాణం మరియు స్థానిక లింక్‌లను తనిఖీ చేయడానికి అనువాదం తర్వాత `co-op-review` ను నడపండి.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## కాన్ఫిగరేషన్ లోపాలు

### భాషా మోడల్ ప్రొవైడర్ లేదు

లోపం:

```text
No language model configuration found.
```

పరిష్కారం:

- Azure OpenAI లేదా OpenAIను కాన్ఫిగర్ చేయండి.
- ఆ వేరియబుల్స్ కమాండ్ నడిచే ఎన్విరన్మెంట్‌లో ఉన్నాయా అని ధృవీకరించండి.
-లోకల్ వినియోగానికి, ప్రాజెక్ట్ రూట్‌లో `.env` లో వాటిని ఉంచండి.

చూడండి [కాన్ఫిగరేషన్](configuration.md).

### Azure AI Vision లేకుండా చిత్రాల అనువాదం

లోపం:

```text
Image translation requested but Azure AI Service is not configured.
```

పరిష్కారం:

- `AZURE_AI_SERVICE_API_KEY` ను జోడించండి.
- `AZURE_AI_SERVICE_ENDPOINT` ను జోడించండి.
- లేదా `translate -l "ko" -md` వంటి టెక్స్ట్-మాత్రమైన కమాండ్‌ను నడపండి.

### చెల్లని కీలకము లేదా ఎండ్‌పాయింట్

లక్షణాల్లో `401`, తీయబడిన అనుమతి లోపాలు, లేదా ఎండ్‌పాయింట్ యాక్సెస్ లోపాలు ఉండవచ్చు.

పరిష్కారం:

- కీ అదే Azure వనరుకు మరియు ఎండ్‌పాయింట్‌కు చెందిందా అని నిర్ధారించండి.
- `-img` ఉపయోగించినప్పుడు ఆ వనరు Vision ను మద్దతు ఇస్తుందా అని నిర్ధారించండి.
- Azure OpenAI deployment name మరియు API version మీ deployment కి సరిపోతున్నాయా అని నిర్ధారించండి.
- డీబగ్ లాగ్స్‌తో నడపండి: `translate -l "ko" -md -d -s`.

## ఏ ఫైళ్ళూ అనువదించబడలేదు

సాధారణ కారణాలు:

- ఎంచుకున్న ఫ్లాగులు మీ ఫైళ్లకు సరిపోకవచ్చు.
- ఇప్పటికే అనువదించిన ఫైళ్లు ఉంటాయి.
- సోర్స్ ఫైళ్లు బహిష్కృత డైరెక్టరీల క్రింద ఉండవచ్చు.
- కమాండ్ తప్పు ప్రాజెక్ట్ రూట్ నుండి నడుపబడుతున్నది.

తనిఖీలు:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

కమాండ్ ప్రాజెక్ట్ రూట్ వెలుపల నడిస్తుంటే `--root-dir` ఉపయోగించండి.

## అనుకోని లింక్ వ్యవహారం

లింక్ రీరైటింగ్ ఎంచుకున్న కంటెంట్ రకాలపై ఆధారపడి ఉంటుంది:

- `-nb` చేర్చబడినప్పుడు: నోట్‌బుక్ లింక్‌లు అనువదించిన నోట్‌బుక్స్‌కు సూచించవచ్చు.
- `-nb` తొలగించబడినప్పుడు: నోట్‌బుక్ లింక్‌లు మూల నోట్‌బుక్స్‌ను సూచిస్తూనే ఉండవచ్చు.
- `-img` చేర్చబడినప్పుడు: చిత్ర లింక్‌లు అనువదించిన చిత్రాలను సూచించవచ్చు.
- `-img` తీసివేయబడినప్పుడు: చిత్ర లింక్‌లు మూల చిత్రాలను సూచిస్తూనే ఉండవచ్చు.

అంతర్గత అన్ని లింకులు అనువదించిన ఔట్‌పుట్‌లను ప్రాధాన్యం ఇవ్వాలంటే పూర్తి కంటెంట్ అనువాదాన్ని నడపండి:

```bash
translate -l "ko" -md -nb -img
```

అనువాదం తర్వాత లింక్ సమీక్ష ను నడపండి:

```bash
co-op-review -l "ko"
```

## మార్క్డౌన్ రెండరింగ్ సమస్యలు

అనువదించిన మార్క్డౌన్ తప్పుగా ప్రదర్శించబడితే:

- frontmatter `---` తో ప్రారంభమవుతూ `---` తో ముగియడాన్ని తనిఖీ చేయండి.
- కోడ్ ఫెన్స్‌ల సంఖ్య సోర్స్ మరియు అనువదించిన ఫైళ్లలో తాను మ్యాచ్ అవుతున్నదో చూడండి.
- సాధారణ నిర్మాణ సమస్యలను పట్టుకునేందుకు `co-op-review` ని నడపండి.
- ఔట్‌పుట్ దెబ్బతిన్నట్లయితే ఆ ప్రత్యేక ఫైల్‌ను మళ్లీ అనువదించండి.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action నడిచింది కానీ పుల్ రిక్వెస్ట్ సృష్టించబడలేదు

`peter-evans/create-pull-request` బ్రాంచ్ base కంటే ముందుగా లేదు అని నివేదిస్తే, వర్క్‌ఫ్లో కమిట్ చేయడానికి ఎలాంటి ఫైళ్ళను కనుగొనలేదని అర్థం.

సాధ్యమైన కారణాలు:

- అనువాద రన్ మార్పులు ఉత్పత్తి చేయలేదు.
- `.gitignore` `translations/`, `translated_images/`, లేదా అనువదించిన నోట్‌బుక్స్‌ను బహిష్కరిస్తుంది.
- `add-paths` ఉత్పత్తి అవుట్‌పుట్ డైరెక్టరీలకు సరిపోకపోవచ్చు.
- అనువాద స్టెప్ ముందుగానే ముగిసింది.

పరిష్కారాలు:

1. `translations/` లేదా `translated_images/` లో ఉత్పత్తి ఫైళ్లు ఉన్నాయా నిర్ధారించండి.
2. `.gitignore` ఉత్పత్తి అవుట్‌పుట్‌లను ఇగ్నోర్ చేయడంలేదు అనని నిర్ధారించండి.
3. Use matching `add-paths`:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. తాత్కాలికంగా అనువాద కమాండ్ కు డీబగ్ ఫ్లాగ్‌లను జోడించండి:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. వర్క్‌ఫ్లో అనుమతులు ఇవి ఉంటాయా నిర్ధారించండి:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## అనువాద నాణ్యత

యంత్ర అనువాదాలకు మానవ సమీక్ష అవసరం కావొచ్చు. ప్రయోగాత్మకమైన నాణ్యత స్కోరింగ్ మరియు తక్కువ-నమ్మకత్వ రిపేర్ వర్క్‌ఫ్లోలను మాత్రమే కోరుకున్నప్పుడు `evaluate` ను ఉపయోగించండి.

!!! warning "Experimental"
    `evaluate` షరతు-ఆధారిత మరియు LLM-ఆధారిత పరీక్షలను ఉపయోగించవచ్చు, మరియు దాని స్కోరింగ్ మోడల్ మరియు మెటాడేటా ప్రవర్తన మారవచ్చు. మీ వర్క్‌ఫ్లో మార్పులకు సిద్ధంగా లేకపోతే దీన్ని తప్పనిసరి CI గేట్లలో ఉంచకండి.

నిర్ణాయక CI తనిఖీలకు, బదులుగా `co-op-review` ను ఉపయోగించండి.
