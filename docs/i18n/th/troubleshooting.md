# การแก้ไขปัญหา

ใช้หน้านี้เมื่อการรันการแปลสำเร็จโดยไม่คาดคิด ล้มเหลวระหว่างการกำหนดค่า หรือสร้างผลลัพธ์ที่ต้องการการตรวจทาน

## เริ่มที่นี่

1. รันคำสั่งแบบมุ่งเป้าก่อน เช่น `translate -l "ko" -md`.
2. เพิ่ม `-d` เพื่อรับบันทึกดีบักบนคอนโซล
3. เพิ่ม `-s` เพื่อบันทึกบันทึกดีบักไว้ที่ `<root-dir>/logs/`
4. รัน `co-op-review` หลังการแปลเพื่อตรวจความสด โครงสร้าง และลิงก์ภายในเครื่อง

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## ข้อผิดพลาดในการกำหนดค่า

### ไม่มีผู้ให้บริการโมเดลภาษา

ข้อผิดพลาด:

```text
No language model configuration found.
```

การแก้ไข:

- กำหนดค่า Azure OpenAI หรือ OpenAI
- ตรวจสอบว่าตัวแปรต่าง ๆ อยู่ในสภาพแวดล้อมที่คำสั่งถูกรัน
- สำหรับการใช้งานในเครื่อง ให้ใส่ไว้ใน `.env` ที่ root ของโปรเจ็กต์

ดู [การกำหนดค่า](configuration.md)

### การแปลรูปภาพโดยไม่มี Azure AI Vision

ข้อผิดพลาด:

```text
Image translation requested but Azure AI Service is not configured.
```

การแก้ไข:

- เพิ่ม `AZURE_AI_SERVICE_API_KEY`
- เพิ่ม `AZURE_AI_SERVICE_ENDPOINT`
- หรือรันคำสั่งเฉพาะข้อความ เช่น `translate -l "ko" -md`

### คีย์หรือ Endpoint ไม่ถูกต้อง

อาการอาจรวมถึง `401`, ข้อผิดพลาดสิทธิ์ที่ถูกปิดบัง หรือข้อผิดพลาดการเข้าถึง endpoint

การแก้ไข:

- ยืนยันว่าคีย์เป็นของทรัพยากร Azure เดียวกับ endpoint
- ยืนยันว่าทรัพยากรรองรับ Vision เมื่อใช้ `-img`
- ยืนยันว่า deployment name ของ Azure OpenAI และเวอร์ชัน API ตรงกับการปรับใช้งานของคุณ
- รันพร้อมบันทึกดีบัก: `translate -l "ko" -md -d -s`

## ไม่มีไฟล์ใดถูกแปล

สาเหตุทั่วไป:

- แฟลกที่เลือกไม่ตรงกับไฟล์ของคุณ
- มีไฟล์แปลที่มีอยู่แล้ว
- ไฟล์ต้นทางอยู่ภายใต้ไดเรกทอรีที่ถูกยกเว้น
- คำสั่งถูกรันจาก root ของโปรเจ็กต์ที่ไม่ถูกต้อง

การตรวจสอบ:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

ใช้ `--root-dir` เมื่อคำสั่งถูกรันนอก root ของโปรเจ็กต์

## พฤติกรรมลิงก์ที่ไม่คาดคิด

การเขียนทับลิงก์ขึ้นอยู่กับประเภทเนื้อหาที่เลือก:

- `-nb` ถูกเลือก: ลิงก์โน้ตบุ๊กสามารถชี้ไปยังโน้ตบุ๊กที่แปลแล้วได้
- `-nb` ไม่ถูกเลือก: ลิงก์โน้ตบุ๊กสามารถยังคงชี้ไปยังโน้ตบุ๊กต้นฉบับได้
- `-img` ถูกเลือก: ลิงก์รูปภาพสามารถชี้ไปยังรูปภาพที่แปลแล้วได้
- `-img` ไม่ถูกเลือก: ลิงก์รูปภาพสามารถยังคงชี้ไปยังรูปภาพต้นฉบับได้

รันการแปลเนื้อหาเต็มรูปแบบเมื่อคุณต้องการให้ลิงก์ภายในทั้งหมดชี้ไปยังผลลัพธ์ที่แปลแล้ว:

```bash
translate -l "ko" -md -nb -img
```

รันการตรวจสอบลิงก์หลังการแปล:

```bash
co-op-review -l "ko"
```

## ปัญหาการเรนเดอร์ Markdown

หาก Markdown ที่แปลแล้วแสดงผลไม่ถูกต้อง:

- ตรวจสอบว่า frontmatter เริ่มและจบด้วย `---`
- ตรวจสอบว่าจำนวนรั้วโค้ด (code fence) ตรงกันระหว่างไฟล์ต้นฉบับและไฟล์ที่แปลแล้ว
- รัน `co-op-review` เพื่อตรวจจับปัญหาโครงสร้างทั่วไป
- แปลไฟล์นั้นใหม่หากผลลัพธ์ถูกเสียหาย

```bash
co-op-review -l "ko" --format github
```

## GitHub Action ถูกรัน แต่ไม่มี Pull Request ถูกสร้าง

หาก `peter-evans/create-pull-request` รายงานว่าสาขาไม่อยู่เหนือ base แปลว่า workflow ไม่พบไฟล์ที่จะ commit

สาเหตุที่เป็นไปได้:

- การรันแปลไม่ได้สร้างการเปลี่ยนแปลงใดๆ
- `.gitignore` ยกเว้น `translations/`, `translated_images/` หรือโน้ตบุ๊กที่แปลแล้ว
- `add-paths` ไม่ตรงกับไดเรกทอรีผลลัพธ์ที่สร้างขึ้น
- ขั้นตอนการแปลออกก่อนเวลา

การแก้ไข:

1. ยืนยันว่าไฟล์ที่สร้างขึ้นมีอยู่ใน `translations/` หรือ `translated_images/`
2. ยืนยันว่า `.gitignore` ไม่ได้ละเว้นผลลัพธ์ที่สร้างขึ้น
3. ใช้ `add-paths` ที่ตรงกัน:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. เพิ่มแฟลก debug ชั่วคราวกับคำสั่ง translate:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. ยืนยันว่า permissions ของ workflow รวมถึง:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## คุณภาพการแปล

การแปลด้วยเครื่องอาจต้องการการตรวจทานโดยมนุษย์ ใช้ `evaluate` ก็ต่อเมื่อคุณต้องการการให้คะแนนคุณภาพแบบทดลองและกระบวนการซ่อมแซมเมื่อความมั่นใจต่ำ

!!! warning "Experimental"
    `evaluate` สามารถใช้การตรวจสอบแบบอิงกฎและแบบอิง LLM และโมเดลการให้คะแนนรวมถึงพฤติกรรมเมตาดาต้าอาจเปลี่ยนแปลงได้ ให้งดใช้ในการบังคับเกต CI ที่จำเป็น เว้นแต่กระบวนงานของคุณเตรียมพร้อมสำหรับการเปลี่ยนแปลง

สำหรับการตรวจสอบ CI แบบกำหนดผลได้ ให้ใช้ `co-op-review` แทน
