# مسائل حل کرنا

Use this page when a translation run succeeds unexpectedly, fails during configuration, or produces output that needs review.

## یہاں شروع کریں

1. پہلے ایک مخصوص کمانڈ چلائیں، مثلاً `translate -l "ko" -md`.
2. کونسول ڈی بگ لاگز کے لیے `-d` شامل کریں.
3. ڈی بگ لاگز کو `<root-dir>/logs/` میں محفوظ کرنے کے لیے `-s` شامل کریں.
4. ترجمہ کے بعد تازگی، ساخت، اور مقامی روابط چیک کرنے کے لیے `co-op-review` چلائیں.

```bash
translate -l "ko" -md -d -s
co-op-review -l "ko"
```

## ترتیبی غلطیاں

### زبان ماڈل فراہم کنندہ موجود نہیں

خرابی:

```text
No language model configuration found.
```

حل:

- Azure OpenAI یا OpenAI ترتیب دیں.
- اس بات کی تصدیق کریں کہ متغیرات اس ماحول میں موجود ہیں جہاں کمانڈ چلتی ہے.
- مقامی استعمال کے لیے انہیں پروجیکٹ روٹ میں `.env` میں رکھیں.

دیکھیں [ترتیب](configuration.md).

### Azure AI Vision کے بغیر تصویر کا ترجمہ

خرابی:

```text
Image translation requested but Azure AI Service is not configured.
```

حل:

- `AZURE_AI_SERVICE_API_KEY` شامل کریں.
- `AZURE_AI_SERVICE_ENDPOINT` شامل کریں.
- یا `translate -l "ko" -md` جیسا صرف متن والا کمانڈ چلائیں.

### غلط کلید یا اینڈ پوائنٹ

علامات میں `401`، اجازت سے متعلقہ خامیاں، یا اینڈ پوائنٹ تک رسائی کی غلطیاں شامل ہو سکتی ہیں.

حل:

- تصدیق کریں کہ کلید اسی Azure resource کی ہے جو اینڈ پوائنٹ کے لیے ہے.
- `-img` استعمال کرتے وقت تصدیق کریں کہ وہ resource Vision کو سپورٹ کرتا ہے.
- Azure OpenAI کی deployment کا نام اور API ورژن آپ کی deployment سے میل کھاتا ہے یا نہیں، تصدیق کریں.
- ڈی بگ لاگز کے ساتھ چلائیں: `translate -l "ko" -md -d -s`.

## کوئی فائلیں ترجمہ نہیں ہوئیں

عام وجوہات:

- منتخب کیے گئے فلیگز آپ کی فائلوں سے میل نہیں کھاتے.
- پہلے سے ترجمہ شدہ فائلیں موجود ہیں.
- ماخذ فائلیں مستثنیٰ ڈائریکٹریز کے تحت ہیں.
- کمانڈ غلط پروجیکٹ روٹ سے چل رہی ہے.

چیک کریں:

```bash
translate -l "ko" -md --dry-run
translate -l "ko" -nb --dry-run
translate -l "ko" -img --dry-run
```

جب کمانڈ پروجیکٹ روٹ کے باہر چلائی جا رہی ہو تو `--root-dir` استعمال کریں.

## غیر متوقع لنک برتاؤ

لنک ری رائٹنگ منتخب کردہ مواد کی اقسام پر منحصر ہوتی ہے:

- `-nb` شامل: نوٹ بک لنکس ترجمہ شدہ نوٹ بکس کی طرف اشارہ کر سکتے ہیں.
- `-nb` خارج: نوٹ بک لنکس ماخذ نوٹ بکس کی طرف رہ سکتے ہیں.
- `-img` شامل: امیج لنکس ترجمہ شدہ تصاویر کی طرف اشارہ کر سکتے ہیں.
- `-img` خارج: امیج لنکس ماخذ تصاویر کی طرف رہ سکتے ہیں.

جب تمام داخلی لنکس کو ترجمہ شدہ آؤٹ پٹس پر ترجیح دینی ہو تو مکمل مواد کا ترجمہ چلائیں:

```bash
translate -l "ko" -md -nb -img
```

ترجمہ کے بعد لنک کا جائزہ چلائیں:

```bash
co-op-review -l "ko"
```

## مارک ڈاؤن رینڈرنگ کے مسائل

اگر ترجمہ شدہ مارک ڈاؤن غلط طریقے سے رینڈر ہو رہا ہے:

- چیک کریں کہ frontmatter `---` سے شروع اور ختم ہو.
- چیک کریں کہ کوڈ فینسز کی تعداد ماخذ اور ترجمہ شدہ فائلوں میں مطابقت رکھتی ہے.
- عمومی ساختی مسائل پکڑنے کے لیے `co-op-review` چلائیں.
- اگر آؤٹ پٹ خراب ہو گیا ہو تو مخصوص فائل کو دوبارہ ترجمہ کریں.

```bash
co-op-review -l "ko" --format github
```

## GitHub Action چلا مگر کوئی Pull Request نہیں بنی

اگر `peter-evans/create-pull-request` رپورٹ کرتا ہے کہ برانچ base کے آگے نہیں ہے، تو ورک فلو کو commit کرنے کے لیے کوئی فائلیں نہیں ملیں.

ممکنہ وجوہات:

- ترجمہ رن نے کوئی تبدیلیاں پیدا نہیں کیں.
- `.gitignore` `translations/`, `translated_images/`، یا ترجمہ شدہ نوٹ بکس کو مستثنیٰ کر رہا ہے.
- `add-paths` جنریٹڈ آؤٹ پٹ ڈائریکٹریز سے میل نہیں کھاتا.
- ترجمہ مرحلہ قبل از وقت ختم ہو گیا.

حل:

1. تصدیق کریں کہ جنریٹڈ فائلیں `translations/` یا `translated_images/` میں موجود ہیں.
2. تصدیق کریں کہ `.gitignore` جنریٹڈ آؤٹ پٹس کو نظر انداز نہیں کر رہا.
3. Use matching `add-paths`:

   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```

4. Temporarily add debug flags to the translate command:

   ```bash
   translate -l "ko" -md -d -s
   ```

5. Confirm workflow permissions include:

   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## ترجمے کا معیار

مشین ترجموں کو انسانی جائزے کی ضرورت ہو سکتی ہے۔ `evaluate` کا استعمال صرف اُس وقت کریں جب آپ تجرباتی معیار اسکورنگ اور کم اعتماد مرمت کے ورک فلو چاہتے ہوں۔

!!! warning "Experimental"
    `evaluate` قواعد کی بنیاد پر اور LLM کی بنیاد پر چیکس استعمال کر سکتا ہے، اور اس کا اسکورنگ ماڈل اور میٹا ڈیٹا رویہ تبدیل ہو سکتا ہے۔ اسے لازمی CI گیٹس سے دور رکھیں جب تک کہ آپ کا ورک فلو تبدیلیوں کے لیے تیار نہ ہو۔

مستقل CI چیکس کے لیے اس کی جگہ `co-op-review` استعمال کریں.
