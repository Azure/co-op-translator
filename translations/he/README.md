## מבט כללי

**Co-op Translator** עוזר לך למקומת את התוכן החינוכי שלך ב‑GitHub למספר שפות בקלות.
כשאתה מעדכן קבצי Markdown, תמונות או יומנים, התרגומים נשארים מסונכרנים באופן אוטומטי, מה שמבטיח שהתוכן שלך יישאר מדויק ועדכני ללומדים ברחבי העולם.

השתמש בו מה‑CLI כדי לתרגם מאגר, מה‑API של Python לאוטומציה, או דרך שרת MCP לזרמי עבודה עם סוכנים ועורכים.

## למה Co-op Translator?

תרגום קובץ אחד הוא קל. שמירה על מאגר תיעוד שלם
מתורגם, מקושר ועדכני היא החלק הקשה.

| בעיה | כיצד Co-op Translator עוזר |
| --- | --- |
| מסמכים ארוכים לא מטופלים בבקשה אחת | קבצי Markdown גדולים מפוצלים לחלקים, כך שקובץ README ארוך אינו תלוי בתשובה בודדת של המודל. אם חלק נכשל, Co-op Translator יכול לנסות שוב ולפצל מחדש רק את החלק שנכשל. |
| אין לסמן תרגומים לא שלמים כעדכניים | תרגום מקוצר לא צריך להיחתם כמעודכן. Co-op Translator בודק את שלמות התרגום לפני שמירה ויכול לזהות תרגומים קיימים הפגועים במבנה. |
| הקישורים צריכים להתאים למבנה המאגר המתורגם | תרגומים ידניים לעיתים משאירים קישורים יחסיים המפנים לעץ המקור. Co-op Translator כותב מחדש קישורי Markdown, יומנים, תמונות ו‑README כך שיתאימו למבנה `translations/<lang>/...`. |
| התרגום צריך לפעול על כל המאגר | Co-op Translator מטפל בקבצי README, במסמכים, ביומנים ובטקסט בתמונות כחלק מזרימת עבודה אחת למאגר, במקום לתרגם קבצים אחד‑אחד. |
| תחזוקת תרגומים חשובה יותר מיצירתם פעם אחת | ערכי hash של המקור ומטא‑נתוני תרגום מאפשרים ל‑Co-op Translator למצוא קבצים מיושנים, לדלג על קבצים שלא השתנו, ולשמור על סנכרון התוכן המתורגם ככל שהמאגר המקורי מתפתח. |

## כיצד מנוהל מצב התרגום

Co-op Translator מנהל תוכן מתורגם כ־**ארטיפקטים תוכנה בגרסה**,  
ולא כקבצים סטטיים.

הכלי עוקב אחר מצב ה‑Markdown, התמונות והיומנים המתורגמים
באמצעות **מטא‑נתונים בהיקף שפה**.

עיצוב זה מאפשר ל‑Co-op Translator:

- לזהות בתבונה תרגומים מיושנים
- לטפל ב‑Markdown, תמונות ויומנים באופן עקבי
- להתרחב בבטחה על פני מאגרים גדולים, מהירים ורב‑שפתיים

על‑ידי מיצוג התרגומים כארטיפקטים מנוהלים,
זרמי העבודה בתרגום מתחברים באופן טבעי לנוהלי
ניהול תלויות וארטיפקטים מודרניים.

## התחלה

ניתן להשתמש ב‑Co-op Translator מה‑CLI, מה‑API של Python, או משרת MCP. התחל במדריך זרימת העבודה אם אתה בוחר בין תרגום מקומי, אוטומציה, CI ואינטגרציה עם סוכנים/עורכים.

- [בחר את זרימת העבודה שלך](../../docs/workflows.md)
- [הגדר אישורים](../../docs/configuration.md)
- [תרגם מה‑CLI](../../docs/cli.md)
- [אוטומציה עם API של Python](../../docs/api.md)
- [התחבר עם שרת MCP](../../docs/mcp.md)
- [הרצה ב‑GitHub Actions](../../docs/github-actions.md)

דוגמת CLI מינימלית לאחר ההגדרה:

```bash
python -m venv .venv
# ווינדוס
.venv\Scripts\activate
# macOS/לינוקס
source .venv/bin/activate

pip install co-op-translator
translate -l "ko" -md
co-op-review -l "ko"
```

בהרצות ראשוניות על מאגרים גדולים, השתמש ב־`--dry-run` לפני כתיבת הקבצים המתורגמים. ראה את [מדריך ה‑CLI](../../docs/cli.md) עבור דגלי סוג תוכן, לוגים, סקירה והעברת קישורים.

הרצה מהירה של מיכל עם Bash/Zsh:

```bash
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

הרצה מהירה של מיכל עם PowerShell:

```powershell
docker run --rm -it --env-file .env -v ${PWD}:/work ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

## תכונות

- תרגום אוטומטי עבור Markdown, יומנים ותמונות
- שומר על סנכרון התרגומים עם שינויים במקור
- עובד באופן מקומי (CLI) או ב‑CI (GitHub Actions)
- מספק כלים לתרגום Markdown, יומנים, תמונות, סקירה ופרויקטים דרך MCP
- משתמש ב‑Azure OpenAI או OpenAI לתרגום מבוסס ספק
- מאפשר לשרת MCP לארח סוכנים שיתרגמו חתיכות Markdown ו‑notebook ללא האישורים של LLM של Co‑op Translator
- משתמש ב‑Azure AI Vision להוצאת טקסט מתמונות ולתרגומו
- בוחן את מבנה התרגום ורעננותו באמצעות בדיקות דטרמיניסטיות
- שומר על עיצוב ומבנה Markdown

[Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) מספק שכבת הגנה עצמאית היכולה לזהות תוכן מזיק שמיוצר על ידי משתמשים או על ידי AI באפליקציות ובשירותים. Azure AI Content Safety כולל ממשקי API לטקסט ותמונה המאפשרים לזהות חומרים מזיקים. כמו כן קיימת לנו Content Safety Studio אינטראקטיבית המאפשרת לצפות, לחקור ולנסות דוגמאות קוד לזיהוי תוכן מזיק במודאליות שונות. מדריך ההתחלה המהירה הבא ([quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest)) ינחה אתכם כיצד לבצע בקשות לשירות.

היבט נוסף שיש לקחת בחשבון הוא ביצועי היישום הכוללים. באפליקציות רב‑מודאליות ורב‑מודליות, ביצועים משמעותם שהמערכת פועלת כפי שאתם והמשתמשים שלכם מצפים, כולל אי‑ייצור פלטים מזיקים. חשוב להעריך את ביצועי היישום הכולל שלכם באמצעות [מטריקות איכות יצירה, סיכון ובטיחות](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in).

ניתן להעריך את אפליקציית ה‑AI שלכם בסביבת הפיתוח בעזרת ה־[prompt flow SDK](https://microsoft.github.io/promptflow/index.html). בהינתן אוסף נתונים לבחינה או מטרת יעד, יצירות ה‑generative AI שלכם נמדדות באופן כמותי בעזרת מעריכי ברירת מחדל או מעריכים מותאמים לבחירתכם. כדי להתחיל עם ה‑prompt flow sdk להערכת המערכת שלכם, תוכלו לעקוב אחרי ה[quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk). לאחר הרצת הערכה, תוכלו [ל視ואליזציה של התוצאות ב־Azure AI Studio](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).
