# Co-op Translator का उपयोग करके अपने प्रोजेक्ट का अनुवाद करें

**Co-op Translator** एक कमांड-लाइन इंटरफेस (CLI) टूल है जो आपके प्रोजेक्ट में मौजूद मार्कडाउन और इमेज फाइलों का अनुवाद कई भाषाओं में करने में मदद करता है। इस सेक्शन में टूल के उपयोग का तरीका, विभिन्न CLI विकल्प, और विभिन्न उपयोग मामलों के उदाहरण दिए गए हैं।

> [!NOTE]
> कमांड्स की पूरी सूची और उनकी विस्तृत जानकारी के लिए कृपया [Command reference](./command-reference.md) देखें।

---

## उदाहरण परिदृश्य और कमांड्स

यहाँ **Co-op Translator** के कुछ सामान्य उपयोग और उनके लिए उपयुक्त कमांड्स दिए गए हैं।

### 1. बेसिक अनुवाद (एक भाषा)

अपने पूरे प्रोजेक्ट (मार्कडाउन फाइलें और इमेज) को एक भाषा में, जैसे कोरियाई, अनुवादित करने के लिए निम्न कमांड का उपयोग करें:

```bash
translate -l "ko"
```

यह कमांड सभी मार्कडाउन और इमेज फाइलों को कोरियाई में अनुवादित करेगा, बिना किसी मौजूदा अनुवाद को हटाए नए अनुवाद जोड़ देगा।

> [!TIP]
>
> जानना चाहते हैं कि **Co-op Translator** में कौन-कौन से भाषा कोड उपलब्ध हैं? अधिक जानकारी के लिए रिपॉजिटरी में [Supported Languages](https://github.com/Azure/co-op-translator#supported-languages) सेक्शन देखें।

#### Phi-3 CookBook पर उदाहरण

**Phi-3 CookBook** में, मैंने मौजूदा मार्कडाउन फाइलों और इमेज के लिए कोरियाई अनुवाद जोड़ने के लिए निम्न तरीका अपनाया।

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko"
Translating images: 100%|███████████████████████████████████████████████████| 276/276 [1:09:56<00:00, 15.37s/it]
Translating markdown files: 100%|████████████████████████████████████████████████| 153/153 [1:43:07<00:00, 241.31s/it]
```

### 2. एक से अधिक भाषाओं में अनुवाद

अपने प्रोजेक्ट को कई भाषाओं (जैसे स्पेनिश, फ्रेंच, और जर्मन) में अनुवादित करने के लिए यह कमांड उपयोग करें:

```bash
translate -l "es fr de"
```

यह कमांड प्रोजेक्ट को स्पेनिश, फ्रेंच, और जर्मन में अनुवादित करेगा, बिना मौजूदा अनुवादों को ओवरराइट किए नए अनुवाद जोड़ते हुए।

#### Phi-3 CookBook पर उदाहरण

**Phi-3 CookBook** में, नवीनतम कमिट्स को प्रतिबिंबित करने के लिए लेटेस्ट चेंजेस पुल करने के बाद, मैंने नए जोड़े गए मार्कडाउन फाइलों और इमेज का अनुवाद करने के लिए निम्न तरीका अपनाया।

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko ja zh tw es fr" -a
Translating images: 100%|███████████████████████████████████████████████████| 273/273 [1:09:56<00:00, 15.37s/it]
Translating markdown files: 100%|████████████████████████████████████████████████| 6/6 [24:07<00:00, 241.31s/it]
```

> [!NOTE]
> आमतौर पर एक बार में एक भाषा का अनुवाद करना बेहतर होता है, लेकिन ऐसी स्थितियों में जहां विशिष्ट बदलाव जोड़ने हों, एक साथ कई भाषाओं का अनुवाद करना अधिक प्रभावी हो सकता है।

### 3. रूट डायरेक्टरी निर्दिष्ट करना

डिफ़ॉल्ट रूप से, ट्रांसलेटर वर्तमान वर्किंग डायरेक्टरी का उपयोग करता है। यदि आपका प्रोजेक्ट कहीं और स्थित है, तो -r विकल्प के साथ रूट डायरेक्टरी निर्दिष्ट करें:

```bash
translate -l "es fr de" -r "./my_project"
```

यह कमांड `./my_project` में मौजूद फाइलों का अनुवाद करेगा।

`-u` विकल्प का उपयोग मौजूदा अनुवादों को हटाकर फिर से अनुवाद करने के लिए किया जाता है।

```bash
translate -l "ko" -u
```

सावधान: यह कमांड मौजूदा अनुवादों को हटाने से पहले आपकी पुष्टि मांगेगा।

#### Phi-3 CookBook पर उदाहरण

**Phi-3 CookBook** में, मैंने स्पेनिश में सभी अनुवादित फाइलों को अपडेट करने के लिए निम्न तरीका अपनाया। जब मूल सामग्री में कई मार्कडाउन दस्तावेजों में महत्वपूर्ण बदलाव हों, तब यह तरीका उपयोगी होता है। यदि केवल कुछ अनुवादित फाइलें अपडेट करनी हों, तो उन फाइलों को मैन्युअली हटाना और फिर `-a` विकल्प से अपडेटेड अनुवाद जोड़ना अधिक प्रभावी होता है।

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l "es" -u

Do you want to continue? Type 'yes' to proceed: yes
Proceeding with update...
Translating images: 100%|████████████████████████████████████████████| 150/150 [43:46<00:00, 15.55s/it]
Translating markdown files: 100%|███████████████████████████████████| 95/95 [1:40:27<00:00, 125.62s/it]
```

### 6. केवल इमेज का अनुवाद

अपने प्रोजेक्ट में केवल इमेज फाइलों का अनुवाद करने के लिए `-img` विकल्प का उपयोग करें:

```bash
translate -l "ko" -img
```

यह कमांड केवल इमेज को कोरियाई में अनुवादित करेगा, बिना किसी मार्कडाउन फाइल को प्रभावित किए।

### 7. केवल मार्कडाउन फाइलों का अनुवाद

केवल मार्कडाउन फाइलों का अनुवाद करने के लिए `-md` विकल्प का उपयोग करें:

```bash
translate -l "ko" -md
```

### 8. अनुवादित फाइलों में त्रुटियों की जांच

यदि आप अनुवादित फाइलों में त्रुटियों की जांच करना चाहते हैं और आवश्यक होने पर अनुवाद को पुनः प्रयास करना चाहते हैं, तो `-chk` विकल्प का उपयोग करें:

```bash
translate -l "ko" -chk
```

यह कमांड अनुवादित मार्कडाउन फाइलों को स्कैन करेगा और त्रुटिपूर्ण फाइलों के लिए अनुवाद पुनः प्रयास करेगा।

#### Phi-3 CookBook पर उदाहरण

**Phi-3 CookBook** में, मैंने कोरियाई फाइलों में अनुवाद त्रुटियों की जांच करने और पाए गए मुद्दों वाली फाइलों के लिए स्वतः अनुवाद पुनः प्रयास करने के लिए निम्न तरीका अपनाया।

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko" -chk 
Checking translated files for errors in ko...
Checking files for ko: 100%|██████████████████████████████████████████████████| 95/95 [00:01<00:00, 65.47file/s]
Retrying vsc-extension-quickstart.md for ko:   0%|                                     | 0/17 [00:00<?, ?file/s] 
```

यह विकल्प अनुवाद त्रुटियों की जांच करता है। वर्तमान में, यदि मूल और अनुवादित फाइलों के बीच लाइन ब्रेक का अंतर छह से अधिक होता है, तो उस फाइल को अनुवाद त्रुटि के रूप में चिह्नित किया जाता है। भविष्य में इस मानदंड को और लचीला बनाने की योजना है।

उदाहरण के लिए, यह तरीका गायब हिस्सों या भ्रष्ट अनुवादों का पता लगाने में मदद करता है और उन फाइलों का अनुवाद स्वतः पुनः करता है।

हालांकि, यदि आप पहले से जानते हैं कि कौन सी फाइलें समस्या वाली हैं, तो उन फाइलों को मैन्युअली हटाना और `-a` option to re-translate them.

### 9. Debug Mode

To enable detailed logging for troubleshooting, use the `-d` विकल्प का उपयोग करना अधिक प्रभावी होगा:

```bash
translate -l "ko" -d
```

यह कमांड अनुवाद को डिबग मोड में चलाएगा, जिससे अतिरिक्त लॉगिंग जानकारी मिलेगी जो अनुवाद प्रक्रिया के दौरान समस्याओं की पहचान में मदद कर सकती है।

#### Phi-3 CookBook पर उदाहरण

**Phi-3 CookBook** में, मुझे एक समस्या का सामना करना पड़ा जहाँ मार्कडाउन फाइलों में कई लिंक वाले अनुवादों से फॉर्मेटिंग त्रुटियाँ होती थीं, जैसे टूटा हुआ अनुवाद और अनदेखी की गई लाइन ब्रेक। इस समस्या का पता लगाने के लिए मैंने `-d` विकल्प का उपयोग किया ताकि यह देखा जा सके कि अनुवाद प्रक्रिया कैसे काम करती है।

### 10. सभी भाषाओं में अनुवाद

यदि आप प्रोजेक्ट को सभी समर्थित भाषाओं में अनुवादित करना चाहते हैं, तो all कीवर्ड का उपयोग करें।

> [!WARNING]
> एक साथ सभी भाषाओं का अनुवाद करना प्रोजेक्ट के आकार के अनुसार काफी समय ले सकता है। उदाहरण के लिए, **Phi-3 CookBook** को स्पेनिश में अनुवादित करने में लगभग 2 घंटे लगे। इतने बड़े पैमाने पर 20 भाषाओं को एक व्यक्ति द्वारा संभालना व्यावहारिक नहीं है। इसलिए यह सुझाव दिया जाता है कि काम को कई योगदानकर्ताओं में बांटा जाए, हर कोई एक या दो भाषाओं का प्रबंधन करे, और अनुवाद धीरे-धीरे अपडेट करें।

```bash
translate -l "all"
```

यह कमांड प्रोजेक्ट को सभी उपलब्ध भाषाओं में अनुवादित करेगा। यदि आप आगे बढ़ते हैं, तो अनुवाद में प्रोजेक्ट के आकार के अनुसार काफी समय लग सकता है।

> [!TIP]
>
> ### अपडेट करने वाली फाइलें हटाना
> पुल रिक्वेस्ट में हाल ही में बदली गई फाइलों को अपडेट करने के लिए पहला कदम उन फाइलों के सभी मौजूदा संस्करणों को हटाना है, जो विभिन्न भाषा अनुवाद फोल्डरों में मौजूद हैं। आप निम्न कमांड का उपयोग करके एक साथ सभी फाइलों को हटा सकते हैं जो एक विशिष्ट नाम वाली होती हैं।
>
> ### विंडोज़ पर:
> 1. **कमांड प्रॉम्प्ट का उपयोग:**
>    - कमांड प्रॉम्प्ट खोलें।
>    - `cd` कमांड से उस फोल्डर में जाएं जहाँ फाइलें हैं।
>    - फाइलें हटाने के लिए निम्न कमांड का उपयोग करें:
>      ```
>      del /s *filename*
>      ```
>      `filename` with the specific part of the file name you're looking for. The `/s` विकल्प सबडायरेक्टरीज़ को भी खोजता है।
>
> 2. **पावरशेल का उपयोग:**
>    - पावरशेल खोलें।
>    - यह कमांड चलाएं:
>      ```powershell
>      Get-ChildItem -Path "C:\YourPath" -Filter "*filename*" -Recurse | Remove-Item -Force
>      ```
>      `"C:\YourPath"` with the folder path and `filename` with the specific name.
>
> ### On macOS/Linux:
> 1. **Using Terminal**:
>   - Open Terminal.
>   - Navigate to the directory with `cd`.
>   - Use the `find` कमांड:
>     ```bash
>     find . -type f -name "*filename*" -delete
>     ```
>     नवीनतम फाइल बदलाव अपडेट करने के लिए `filename` with the specific name.
>
> Always double-check the files before deleting to avoid accidental loss. 
>
> Once you have deleted the files which need to be replace simply rerun your `translate -l` कमांड का उपयोग करें।

**अस्वीकरण**:  
यह दस्तावेज़ AI अनुवाद सेवा [Co-op Translator](https://github.com/Azure/co-op-translator) का उपयोग करके अनुवादित किया गया है। जबकि हम सटीकता के लिए प्रयासरत हैं, कृपया ध्यान दें कि स्वचालित अनुवादों में त्रुटियाँ या गलतियाँ हो सकती हैं। मूल दस्तावेज़ अपनी मूल भाषा में प्रामाणिक स्रोत माना जाना चाहिए। महत्वपूर्ण जानकारी के लिए, पेशेवर मानव अनुवाद की सलाह दी जाती है। इस अनुवाद के उपयोग से उत्पन्न किसी भी गलतफहमी या गलत व्याख्या के लिए हम जिम्मेदार नहीं हैं।
