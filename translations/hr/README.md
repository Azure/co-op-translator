## Pregled

**Co-op Translator** pomaže vam lokalizirati vaš edukativni GitHub sadržaj na više jezika bez napora.
Kada ažurirate svoje Markdown datoteke, slike ili bilježnice, prijevodi ostaju automatski sinkronizirani, osiguravajući da vaš sadržaj ostane točan i ažuran za učenike diljem svijeta.

Koristite ga iz CLI-ja za prevođenje repozitorija, preko Python API-ja za automatizaciju ili putem MCP poslužitelja za tijekove rada agenata i urednika.

Primjer kako je prevedeni sadržaj organiziran:

![Primjer](../../imgs/translation-ex.png)

## Zašto Co-op Translator?

Prevođenje jedne datoteke je lako. Održavanje cijelog repozitorija dokumentacije
prevedenog, povezanog i ažurnog je teži dio.

| Problem | Kako Co-op Translator pomaže |
| --- | --- |
| Long docs are not one prompt | Velike Markdown datoteke se dijele na dijelove, tako da dugačak README ne ovisi o jednom krhkom odgovoru modela. Ako dio zakaže, Co-op Translator može pokušati ponovno i ponovo razdijeliti samo neuspjeli dio. |
| Incomplete translations should not be marked current | Skratki prijevod nikada se ne bi trebao označiti kao ažuran. Co-op Translator provjerava integritet prijevoda prije spremanja i može otkriti strukturno nepotpune postojeće prijevode. |
| Links should match the translated repo structure | Ručni prijevodi često ostavljaju relativne linkove koji vode natrag na izvorno stablo. Co-op Translator prepravlja Markdown, bilježnice, slike i README linkove da odgovaraju strukturi `translations/<lang>/...`. |
| Translation should work across an entire repo | Co-op Translator obrađuje README datoteke, dokumentaciju, bilježnice i tekst na slikama kao dio jednog radnog toka repozitorija, umjesto da prevodi datoteke jednu po jednu. |
| Maintaining translations matters more than creating them once | Izvorišni hashovi i metapodaci prijevoda omogućuju Co-op Translatoru da pronađe zastarjele datoteke, preskoči nepromijenjene datoteke i održi sinkroniziran prevedeni sadržaj kako se izvorni repozitorij razvija. |

## Kako se upravlja statusom prijevoda

Co-op Translator upravlja prevedenim sadržajem kao **verzioniranim softverskim artefaktima**,  
a ne kao statičnim datotekama.

Alat prati stanje prevedenog Markdowna, slika i bilježnica
koristeći **metapodatke ograničene na jezik**.

Ovaj dizajn omogućuje Co-op Translatoru da:

- Pouzdano otkriva zastarjele prijevode
- Postupa dosljedno s Markdownom, slikama i bilježnicama
- Sigurno skalira preko velikih, brzo-promjenjivih, višejzičnih repozitorija

Modeliranjem prijevoda kao upravljanih artefakata,
tijekovi rada prijevoda prirodno se usklađuju s modernim
praksama upravljanja softverskim ovisnostima i artefaktima.

→ [Kako se upravlja statusom prijevoda](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)

## Počnite

Co-op Translator se može koristiti iz CLI-ja, Python API-ja ili MCP poslužitelja. Započnite s vodičem za tijek rada ako birate između lokalnog prevođenja, automatizacije, CI-a i integracije agenata/urednika.

- [Odaberite svoj radni tijek](../../docs/workflows.md)
- [Konfigurirajte vjerodajnice](../../docs/configuration.md)
- [Prevedi iz CLI-ja](../../docs/cli.md)
- [Automatizirajte s Python API-jem](../../docs/api.md)
- [Povežite se s MCP poslužiteljem](../../docs/mcp.md)
- [Pokreni u GitHub Actions](../../docs/github-actions.md)

Minimalni CLI primjer nakon konfiguracije:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate

pip install co-op-translator
translate -l "ko" -md
co-op-review -l "ko"
```

Za prve pokrete na velikim repozitorijima, koristite `--dry-run` prije pisanja prevedenih datoteka. Pogledajte [CLI Reference](../../docs/cli.md) za zastavice tipova sadržaja, zapise, pregled i migraciju linkova.

Brzi pokret u kontejneru s Bash/Zsh:

```bash
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

Brzi pokret u kontejneru s PowerShellom:

```powershell
docker run --rm -it --env-file .env -v ${PWD}:/work ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

## Značajke

- Automatizirani prijevod za Markdown, bilježnice i slike
- Održava prijevode sinkroniziranima s promjenama izvora
- Radi lokalno (CLI) ili u CI-u (GitHub Actions)
- Izlaže alate za prijevod Markdowna, bilježnica, slika, recenzije i projekte putem MCP-a
- Koristi Azure OpenAI ili OpenAI za prijevode podržane od pružatelja
- Omogućuje MCP-u da domaćinski agenti prevode dijelove Markdowna i bilježnica bez Co-op Translator LLM vjerodajnica
- Koristi Azure AI Vision za izvlačenje teksta sa slika i prijevod
- Pregledava strukturu prijevoda i ažurnost pomoću determinističkih provjera
- Očuva oblikovanje i strukturu Markdowna

## Responsible AI

Microsoft je posvećen pomaganju našim korisnicima da odgovorno koriste naše AI proizvode, dijeljenju naših saznanja i izgradnji partnerstava temeljenih na povjerenju kroz alate poput Transparency Notes i Impact Assessments. Mnogi od tih resursa mogu se pronaći na [https://aka.ms/RAI](https://aka.ms/RAI).
Microsoftov pristup odgovornoj AI utemeljen je na našim AI načelima pravednosti, pouzdanosti i sigurnosti, privatnosti i sigurnosti, uključenosti, transparentnosti i odgovornosti.

Modeli za prirodni jezik, slike i govor velikih razmjera - poput onih korištenih u ovom primjeru - mogu se ponašati na načine koji su nepravedni, nepouzdani ili uvredljivi, što može uzrokovati štetu. Molimo konzultirajte [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) kako biste se informirali o rizicima i ograničenjima.

Preporučeni pristup ublažavanju tih rizika je uključiti sigurnosni sustav u vašu arhitekturu koji može otkrivati i sprječavati štetno ponašanje. [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) pruža neovisni sloj zaštite, sposoban otkriti štetni sadržaj generiran od strane korisnika i AI-a u aplikacijama i uslugama. Azure AI Content Safety uključuje tekstualne i slikovne API-je koji vam omogućuju otkrivanje štetnog materijala. Također imamo interaktivni Content Safety Studio koji vam omogućuje pregledavanje, istraživanje i isprobavanje primjera koda za otkrivanje štetnog sadržaja preko različitih modaliteta. Sljedeća [quickstart dokumentacija](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) vodi vas kroz slanje zahtjeva servisu.

Još jedan aspekt koji treba uzeti u obzir je ukupna učinkovitost aplikacije. Kod multimodalnih i aplikacija koje koriste više modela, učinkovitost smatramo time da sustav radi onako kako vi i vaši korisnici očekujete, uključujući i to da ne generira štetne izlaze. Važno je procijeniti performanse vaše cjelovite aplikacije koristeći [mjerne pokazatelje kvalitete generiranja te rizika i sigurnosti](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in).

Bilo da imate testni skup podataka ili cilj, generacije vaše generativne AI aplikacije kvantitativno se mjere ugrađenim evaluatorima ili prilagođenim evaluatorima po vašem izboru.
