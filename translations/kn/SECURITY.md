## ಭದ್ರತೆ

ಮೈಕ್ರೋಸಾಫ್ಟ್ ತನ್ನ ಸಾಫ್ಟ್‌ವೇರ್ ಉತ್ಪನ್ನಗಳು ಮತ್ತು ಸೇವೆಗಳ ಭದ್ರತೆಯನ್ನು ಗಂಭೀರವಾಗಿ ಪರಿಗಣಿಸುತ್ತದೆ, ಇದರಲ್ಲಿ ನಮ್ಮ GitHub ಸಂಸ್ಥೆಗಳ ಮೂಲಕ ನಿರ್ವಹಿಸಲ್ಪಡುವ ಎಲ್ಲಾ ಮೂಲ ಕೋಡ್ ರೆಪೊಸಿಟರಿಗಳು ಸೇರಿವೆ, ಉದಾಹರಣೆಗೆ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) ಮತ್ತು [Xamarin](https://github.com/xamarin).

ನೀವು ಮೈಕ್ರೋಸಾಫ್ಟ್-ಸ್ವಾಮ್ಯದ ಯಾವುದೇ ರೆಪೊಸಿಟರಿಯಲ್ಲಿ [ಮೈಕ್ರೋಸಾಫ್ಟ್‌ನ ಭದ್ರತಾ ದೋಷದ ವ್ಯಾಖ್ಯಾನ](https://aka.ms/security.md/definition)ಕ್ಕೆ ಹೊಂದುವ ಭದ್ರತಾ ದೋಷವನ್ನು ಕಂಡುಹಿಡಿದಿದ್ದರೆ, ದಯವಿಟ್ಟು ಕೆಳಗಿನಂತೆ ನಮಗೆ ವರದಿ ಮಾಡಿ.

## ಭದ್ರತಾ ಸಮಸ್ಯೆಗಳನ್ನು ವರದಿ ಮಾಡುವುದು

**ದಯವಿಟ್ಟು ಸಾರ್ವಜನಿಕ GitHub ಸಮಸ್ಯೆಗಳ ಮೂಲಕ ಭದ್ರತಾ ದೋಷಗಳನ್ನು ವರದಿ ಮಾಡಬೇಡಿ.**

ಅದಕ್ಕೆ ಬದಲಾಗಿ, ದಯವಿಟ್ಟು [Microsoft Security Response Center (MSRC)](https://aka.ms/security.md/msrc/create-report) ಗೆ ವರದಿ ಮಾಡಿ.

ನೀವು ಲಾಗಿನ್ ಮಾಡದೆ ಸಲ್ಲಿಸಲು ಇಚ್ಛಿಸುತ್ತಿದ್ದರೆ, [secure@microsoft.com](mailto:secure@microsoft.com) ಗೆ ಇಮೇಲ್ ಕಳುಹಿಸಿ. ಸಾಧ್ಯವಾದರೆ, ನಮ್ಮ PGP ಕೀ ಬಳಸಿ ನಿಮ್ಮ ಸಂದೇಶವನ್ನು ಎನ್‌ಕ್ರಿಪ್ಟ್ ಮಾಡಿ; ದಯವಿಟ್ಟು [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) ನಿಂದ ಅದನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ.

ನೀವು 24 ಗಂಟೆಗಳ ಒಳಗೆ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಪಡೆಯುತ್ತೀರಿ. ಏನಾದರೂ ಕಾರಣಕ್ಕಾಗಿ ನೀವು ಪ್ರತಿಕ್ರಿಯೆ ಪಡೆಯದಿದ್ದರೆ, ದಯವಿಟ್ಟು ಇಮೇಲ್ ಮೂಲಕ ಅನುಸರಿಸಿ, ನಾವು ನಿಮ್ಮ ಮೂಲ ಸಂದೇಶವನ್ನು ಸ್ವೀಕರಿಸಿದ್ದೇವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಹೆಚ್ಚಿನ ಮಾಹಿತಿಯನ್ನು [microsoft.com/msrc](https://www.microsoft.com/msrc) ನಲ್ಲಿ ಕಾಣಬಹುದು.

ದಯವಿಟ್ಟು ಕೆಳಗಿನ ಮಾಹಿತಿಯನ್ನು (ನೀವು ನೀಡಬಹುದಾದಷ್ಟು) ಸೇರಿಸಿ, ಸಾಧ್ಯವಾದ ಸಮಸ್ಯೆಯ ಸ್ವರೂಪ ಮತ್ತು ವ್ಯಾಪ್ತಿಯನ್ನು ಉತ್ತಮವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು:

  * ಸಮಸ್ಯೆಯ ಪ್ರಕಾರ (ಉದಾ. ಬಫರ್ ಓವರ್‌ಫ್ಲೋ, SQL ಇಂಜೆಕ್ಷನ್, ಕ್ರಾಸ್-ಸೈಟ್ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್, ಇತ್ಯಾದಿ)
  * ಸಮಸ್ಯೆಯ ಪ್ರಭಾವವನ್ನು ಹೊಂದಿರುವ ಮೂಲ ಕೋಡ್‌ನ ಸಂಪೂರ್ಣ ಪಾತ್‌ಗಳು
  * ಪ್ರಭಾವಿತ ಮೂಲ ಕೋಡ್‌ನ ಸ್ಥಳ (ಟ್ಯಾಗ್/ಶಾಖೆ/ಕಮಿಟ್ ಅಥವಾ ನೇರ URL)
  * ಸಮಸ್ಯೆಯನ್ನು ಪುನರಾವೃತ್ತಿ ಮಾಡಲು ಅಗತ್ಯವಿರುವ ಯಾವುದೇ ವಿಶೇಷ ಸಂರಚನೆ
  * ಸಮಸ್ಯೆಯನ್ನು ಪುನರಾವೃತ್ತಿ ಮಾಡಲು ಹಂತ-ಹಂತದ ಸೂಚನೆಗಳು
  * ಪ್ರೂಫ್-ಆಫ್-ಕಾನ್ಸೆಪ್ಟ್ ಅಥವಾ ಎಕ್ಸ್‌ಪ್ಲಾಯಿಟ್ ಕೋಡ್ (ಸಾಧ್ಯವಾದರೆ)
  * ಸಮಸ್ಯೆಯ ಪ್ರಭಾವ, ದೋಷವನ್ನು ಹೇಗೆ ದುರುಪಯೋಗಪಡಿಸಿಕೊಳ್ಳಬಹುದು ಎಂಬುದರ ವಿವರ

ಈ ಮಾಹಿತಿಯು ನಿಮ್ಮ ವರದಿಯನ್ನು ವೇಗವಾಗಿ ಪ್ರಕ್ರಿಯೆಗೊಳಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

ನೀವು ಬಗ್ ಬೌಂಟಿಗಾಗಿ ವರದಿ ಮಾಡುತ್ತಿದ್ದರೆ, ಹೆಚ್ಚು ಸಂಪೂರ್ಣ ವರದಿಗಳು ಹೆಚ್ಚಿನ ಬೌಂಟಿ ಬಹುಮಾನಕ್ಕೆ ಸಹಾಯ ಮಾಡಬಹುದು. ನಮ್ಮ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ಪುಟಕ್ಕೆ ಭೇಟಿ ನೀಡಿ, ನಮ್ಮ ಸಕ್ರಿಯ ಕಾರ್ಯಕ್ರಮಗಳ ಬಗ್ಗೆ ಹೆಚ್ಚಿನ ವಿವರಗಳನ್ನು ಪಡೆಯಲು.

## ಆದ್ಯತೆಯ ಭಾಷೆಗಳು

ನಾವು ಎಲ್ಲಾ ಸಂವಹನಗಳನ್ನು ಇಂಗ್ಲಿಷ್‌ನಲ್ಲಿ ಇಡಲು ಆದ್ಯತೆ ನೀಡುತ್ತೇವೆ.

## ನೀತಿ

ಮೈಕ್ರೋಸಾಫ್ಟ್ [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) ತತ್ವವನ್ನು ಅನುಸರಿಸುತ್ತದೆ.
