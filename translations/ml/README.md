> git sparse-checkout set --no-cone '/*' '!translations' '!translated_images'
> ```
>
> **CMD (Windows):**
> ```cmd
> git clone --filter=blob:none --sparse https://github.com/Azure/co-op-translator.git
> cd co-op-translator
> git sparse-checkout set --no-cone "/*" "!translations" "!translated_images"
> ```
>
> ഇതുവഴി കോഴ്‌സ് പൂർത്തിയാക്കുന്നതിനുള്ള ആവശ്യമായ എല്ലാം കൂടാതെ much faster ഡൗൺലോഡും നിങ്ങൾക്കുണ്ടാകും.

## അവലോകനം

**Co-op Translator** നിങ്ങളുടെ വിദ്യാഭ്യാസ GitHub ഉള്ളടക്കം പലഭാഷകളിലായി എളുപ്പത്തിൽ ലോക്കലൈസ് ചെയ്യാൻ സഹായിക്കുന്നു. Markdown ഫയലുകൾ, ചിത്രങ്ങൾ, അല്ലെങ്കിൽ നോട്ട്‌ബുക്കുകൾ അപ്‌ഡേറ്റ് ചെയ്താൽ, വിവർത്തനങ്ങൾ സ്വയം സിങ്ക് ആയി നിലനിൽക്കുകയും ലോകമാകെ പഠിക്കുന്നവർക്കായി നിങ്ങളുടെ ഉള്ളടക്കം ശരിയും പുതുക്കപ്പെട്ടതുമായിരിക്കുമെന്ന് ഉറപ്പാക്കുകയും ചെയ്യും.

റിപ്പോസിറ്ററി വിവർത്തനത്തിന് CLI-ൽനിന്നോ, ഓട്ടോമേഷൻ ആവശ്യങ്ങൾക്ക് Python API-ൽനിന്നോ, അല്ലെങ്കിൽ ഏജന്റ്/എഡിറ്റർ പ്രവാഹങ്ങൾക്കായി MCP സർവറിലൂടെ ഇത് ഉപയോഗിക്കുക.

വിവർത്തനം ചെയ്ത ഉള്ളടക്കം എങ്ങനെ ക്രമീകരിച്ചിരിക്കുന്നതിന്റെ ഉദാഹരണം:

## എന്തുകൊണ്ട് Co-op Translator?

ഒരു ഫയൽ വിവർത്തനം ചെയ്യുക എളുപ്പമാണ്. ഒരു മുഴുവൻ ഡോക്യുമെന്റേഷൻ റിപോസിറ്ററിയെ വിവർത്തനം ചെയ്ത നിലയിൽ, ലിങ്കുകൾ ചേർത്തു, അപ്രകാരം അപ് ടു ഡേറ്റ് ആക്കുന്നത് അതാണ് കടുത്തതായത്.

## വിവർത്തന അവസ്ഥ എങ്ങനെ കൈകാര്യം ചെയ്യപ്പെടുന്നു

Co-op Translator വിവർത്തനമായ ഉള്ളടക്കങ്ങളെ **versioned software artifacts** ആയി മാനേജുചെയ്യുന്നു,  
സ്ഥಿರമായ ഫയലുകളായി അല്ല.

ടൂൾ ഭാഷാ-പരിധി മെറ്റാഡാറ്റ ഉപയോഗിച്ച് വിവർത്തനമായ Markdown, ചിത്രം, നോട്ട്ബുക്കുകൾ എന്നിവയുടെ അവസ്ഥ ട്രാക്കുചെയ്‌യുന്നു.

ഈ രൂപകൽപ്പന Co-op Translator-ന് സാധ്യമാക്കുന്നത്:

- വിശ്വാസ്യമായി കാലഹരണപ്പെട്ട വിവർത്തനങ്ങൾ കണ്ടെത്തുക
- Markdown, ചിത്രങ്ങൾ, നോട്ട്ബുക്കുകൾ ഒരേ രീതിയിൽ കൈകാര്യം ചെയ്യുക
- വലിയ, വേഗത്തിൽ മാറുന്ന, ബഹുഭാഷാ റിപോസിറ്ററികളിൽ സുരക്ഷിതമായി സ്കെയിൽ ചെയ്യുക

വിവർത്തനങ്ങളെ മാനേജുചെയ്ത ആർട്ടിഫാക്റ്റുകളായി മോഡൽ ചെയ്യുന്നതിലൂടെ, വിവർത്തന പ്രവാഹങ്ങൾ ആധുനിക സോഫ്റ്റ്വെയർ ഡിപ്പൻഡൻസിയും ആർട്ടിഫാക്റ്റ് മാനേജ്മെന്റ് പ്രാക്ടീസുകളുമായി സ്വാഭാവികമായി മെച്ചഭാവം കൊണ്ടുവരുന്നു.

ക്രമീകരണത്തിന് ശേഷം ലഘു CLI ഉദാഹരണം:

```bash
python -m venv .venv
# വിൻഡോസ്
.venv\Scripts\activate
# മാക്‌ഒഎസ്/ലിനക്‌സ്
source .venv/bin/activate

pip install co-op-translator
translate -l "ko" -md
co-op-review -l "ko"
```

വലിയ റിപോസിറ്ററികളിലെ ആദ്യമായ ഓട്ടുകൾക്കായി, വിവർത്തനം ചെയ്ത ഫയലുകൾ എഴുതുന്നതിന് മുമ്പ് `--dry-run` ഉപയോഗിക്കുക. ഉള്ളടക്ക തരം ഫ്ലാഗുകൾ, ലോഗുകൾ, റിവ്യൂ, ലിങ്ക് മൈഗ്രേഷൻ എന്നിവക്കായി [CLI റഫറൻസ്](../../docs/cli.md) കാണുക.

കോണ്ടെയ്നർ പെട്ടന്ന് ഓടിക്കൽ Bash/Zsh ഉപയോഗിച്ച്:

```bash
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

കോണ്ടെയ്നർ പെട്ടെന്ന് ഓടിക്കൽ PowerShell ഉപയോഗിച്ച്:

```powershell
docker run --rm -it --env-file .env -v ${PWD}:/work ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

## ഫീച്ചറുകൾ

- Markdown, നോട്ട്‌ബുക്കുകൾ, ചിത്രങ്ങൾ എന്നിവയ്ക്ക് സ്വയംചालित വിവർത്തനം
- ഉറവിടമാറ്റങ്ങളോട് കൂടിയിട്ട് വിവർത്തനങ്ങൾ സമന്വയിപ്പിച്ച് സൂക്ഷിക്കുന്നു
- ലോക്കൽ (CLI) യിലും CI (GitHub Actions) യിലും പ്രവർത്തിക്കുന്നു
- MCP മുഖേന Markdown, നോട്ട്‌ബുക്ക്, ചിത്രം, റിവ്യൂ, പ്രോജക്ട് വിവർത്തന ഉപകരണങ്ങൾ ലഭ്യമാണ്
- പ്രൊവൈഡർ-ബാക്ക്ഡ് വിവർത്തനത്തിന് Azure OpenAI അല്ലെങ്കിൽ OpenAI ഉപയോഗിക്കുന്നു
- Co-op Translator LLM ക്രെഡൻഷ്യലുകൾ ഇല്ലാതെ MCP ഹോസ്റ്റുചെയ്യുന്ന ഏജന്റുകൾ Markdown, നോട്ട്‌ബുക്ക് ചങ്കുകൾ വിവർത്തനം ചെയ്യാൻ അനുവദിക്കുന്നു
- ചിത്രം ടെക്‌സ്‌റ്റ് എക്സ്ട്രാക്ഷനും വിവർത്തനത്തിനും Azure AI Vision ഉപയോഗിക്കുന്നു
- നിർണായക പരിശോധനകളിലൂടെ വിവർത്തനത്തിന്റെ ഘടനയും പുതുമയും പരിശോധന ചെയ്യുന്നു
- Markdown ഫോർമാറ്റിംഗ് மற்றும் ഘടന സംരക്ഷിക്കുന്നു

## ഉത്തരവാദിത്വമുള്ള AI

Microsoft ഞങ്ങളുടെ ഉപഭോക്താക്കളെ ഞങ്ങളുടെ AI ഉൽപ്പന്നങ്ങൾ ഉത്തരവാദിത്വത്തോടെ ഉപയോഗിക്കാൻ സഹായിക്കുന്നതിൽ, ഞങ്ങളുടെ പഠനങ്ങൾ പങ്കുവയ്‌ക്കുന്നതിൽ, Transparency Notes ۽ Impact Assessments പോലുള്ള ഉപകരണങ്ങളിലൂടെ വിശ്വാസത്തെ ആശ്രയിച്ച് പങ്കാളിത്തങ്ങൾ നിർമ്മിക്കുന്നതിൽ പ്രതിബദ്ധമാണ്. ഈ വിഭവങ്ങളുടെ പലതും [https://aka.ms/RAI](https://aka.ms/RAI) ൽ കണ്ടെത്താം.
ഉത്തരവാദിത്വമുള്ള AI-യിലേക്കുള്ള Microsoft-ന്റെ സമീപനം നീതിമേറൽ, വിശ്വാസ്യതയും സുരക്ഷയും, സ്വകാര്യതയും സുരക്ഷയും, ഉൾക്കൊള്ളലും, പരദർശിത്വവും ഉത്തരവാദിത്വവുമുള്ള ഞങ്ങളുടെ AI സിദ്ധാന്തങ്ങളിൽ നിൽക്കുന്നു.

വലിയ തോതിലുള്ള പ്രകൃതിഭാഷ, ചിത്രം, ശബ്ദ മോഡലുകൾ — ഈ സാമ്പിളിൽ ഉപയോഗിച്ച മോഡലുകൾ പോലെയുള്ളവ — ചിലപ്പോൾ അസമതുല്യമായ, വിശ്വാസഹീനമായ, അല്ലെങ്കിൽ അപമാനകരമായ രീതിയിൽ പ്രവർത്തിച്ച് നാശം ഉണ്ടാക്കാനുള്ള സാധ്യതയുണ്ട്. റിസ്‌കുകളും പരിമിതികളും സംബന്ധിച്ച് അറിയാൻ ദയവായി [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) സൌജന്യമായി പരിശോധിക്കുക.

ഈ റിസ്കുകൾ കുറയ്ക്കാൻ ശുപാർശ ചെയ്യപ്പെടുന്ന സമീപനം നിങ്ങളുടെ ആർക്കിടെക്ചറിൽ ഹാനികരമായ പെരുമാറ്റം കണ്ടെത്താനും തടയാനുമുള്ള ഒരു സുരക്ഷാ സിസ്റ്റം ഉൾപ്പെടുത്തുകയാണ്. [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) ആപ്ലിക്കേഷനുകളിലും സേവനങ്ങളിലുമായി ഹാനികരമായ ഉപയോക്തൃ-ഉൽപാദിതവും AI-ഉൽപാദിതവുമായ ഉള്ളടക്കം കണ്ടെത്താൻ കഴിയുന്ന സ്വതന്ത്ര സംരക്ഷണ ലെയർ ആണ്. Azure AI Content Safety ടെകസ്റ്റ് மற்றும் ഇമേജ് API-കൾ ഉൾക്കൊള്ളിക്കൊണ്ട് ഹാനികരമായ ഉള്ളടക്കം കണ്ടെത്താൻ സഹായിക്കുന്നു. ഹാനികരമായ ഉള്ളടക്കം വിവിധ മോഡാലിറ്റികളിൽ കണ്ടെത്തുന്നതിനുള്ള സാമ്പിൾ കോഡ് കാണാനും പരീക്ഷിക്കാനും kukho ഇന്ററാക്ടീവ് Content Safety Studio-യും ഞങ്ങളോട് ചേർന്നാണ്. സേവനത്തോട് അഭ്യർത്ഥനകൾ അയയ്ക്കുന്നത് കാണിക്കപ്പെടാൻ ദയവായി താഴെയുള്ള [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) സന്ദർശിക്കുക.

കൂടാതെ പരിഗണിക്കേണ്ട മറ്റൊരു മേഖല മൊത്തത്തിലുള്ള അപ്ലിക്കേഷൻ പ്രകടനമാണ്. മൾട്ടി-മൊഡൽയും മൾട്ടി-മോഡൽ ഉപയോഗിച്ചുള്ള അപ്ലിക്കേഷനുകളിൽ, പ്രകടനം എന്ന് പറയുന്നത് സിസ്റ്റം നിങ്ങൾക്കും നിങ്ങളുടെ ഉപയോക്താക്കൾക്കും പ്രതീക്ഷിക്കുന്നതുപോലെ പ്രവർത്തിക്കുന്നതായിരിക്കണം, അതിൽ ഹാനികരമായ ഔട്ട്പുട്ടുകൾ സൃഷ്ടിക്കാതിരിക്കുക ഉൾപ്പെടുന്നു. നിങ്ങളുടെ മൊത്തം അപ്ലിക്കേഷന്റെ പ്രകടനം വിലയിരുത്താൻ [generation quality and risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) ഉപയോഗിച്ച് മാപനം നടത്തുന്നത് പ്രധാനമാണ്.

നിങ്ങൾക്ക് വികസന പരിസ്ഥിതിയിൽ [prompt flow SDK](https://microsoft.github.io/promptflow/index.html) ഉപയോഗിച്ച് നിങ്ങളുടെ AI അപ്ലിക്കേഷൻ വിലയിരുത്താവുന്നതാണ്. ഒരു ടെസ്റ്റ് ഡാറ്റാസെറ്റോ ലക്ഷ്യവസ്തുവോ നൽകിയാൽ, നിങ്ങളുടെ ജനറേറ്റീവ് AI അപ്ലിക്കേഷനുകളുടെ ജനറേഷനുകൾ ഇൻബിൽട്ട് ഇവലുവേറ്ററുകൾ അല്ലെങ്കിൽ നിങ്ങൾ തിരഞ്ഞെടുക്കുന്ന കസ്റ്റം ഇവലുവേറ്ററുകൾ ഉപയോഗിച്ച് ഗണിതപരമായി അളക്കപ്പെടും. നിങ്ങളുടെ സിസ്റ്റം വിലയിരുത്താൻ prompt flow sdk ഉപയോഗിച്ച് ആരംഭിക്കാൻ, നിങ്ങൾക്ക് [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) പിന്തുടരാൻ കഴിയും. ഒരു വിലയിരുത്തൽ റൺ നിർവഹിച്ചശേഷം, നിങ്ങൾക്ക് [Azure AI Studio-ൽ ഫലങ്ങൾ ദൃശ്യമാക്കാം](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).

ഈ പ്രോജക്ടിന്റെ തിരുത്തിയ പതിപ്പുകളിൽ Microsoft ട്രേഡ്മാർക്കുകൾ അല്ലെങ്കിൽ ലോഗോകൾ ഉപയോഗിക്കുന്നത് ആശയക്കുഴപ്പം സൃഷ്ടിക്കരുത് അല്ലെങ്കിൽ Microsoft-ന്റെ സ്പോൺസർഷിപ്പ് സൂചിപ്പിക്കരുത്.
