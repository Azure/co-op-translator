## സുരക്ഷ

മൈക്രോസോഫ്റ്റ് അവരുടെ സോഫ്റ്റ്വെയർ ഉൽപ്പന്നങ്ങളും സേവനങ്ങളും, അതിൽ ഉൾപ്പെടുന്ന എല്ലാ GitHub ഓർഗനൈസേഷനുകൾ വഴി നിയന്ത്രിക്കുന്ന സോഴ്‌സ് കോഡ് റിപോസിറ്ററികളും, [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet), [Xamarin](https://github.com/xamarin) എന്നിവ ഉൾപ്പെടെ, സുരക്ഷയെ ഗൗരവത്തോടെ കാണുന്നു.

മൈക്രോസോഫ്റ്റ് ഉടമസ്ഥതയിലുള്ള ഏതെങ്കിലും റിപോസിറ്ററിയിൽ [മൈക്രോസോഫ്റ്റിന്റെ സുരക്ഷാ ദൗർബല്യത്തിന്റെ നിർവചനം](https://aka.ms/security.md/definition) പാലിക്കുന്ന ഒരു സുരക്ഷാ ദൗർബല്യം കണ്ടെത്തിയതായി നിങ്ങൾ വിശ്വസിക്കുന്നുവെങ്കിൽ, താഴെ വിശദീകരിച്ചിരിക്കുന്നതുപോലെ അത് ഞങ്ങൾക്ക് റിപ്പോർട്ട് ചെയ്യുക.

## സുരക്ഷാ പ്രശ്നങ്ങൾ റിപ്പോർട്ട് ചെയ്യൽ

**സുരക്ഷാ ദൗർബല്യങ്ങൾ പൊതുവായ GitHub ഇഷ്യൂസ് വഴി റിപ്പോർട്ട് ചെയ്യരുത്.**

അതിനുപകരം, ദയവായി അത് മൈക്രോസോഫ്റ്റ് സെക്യൂരിറ്റി റെസ്പോൺസ് സെന്ററിലേക്ക് (MSRC) [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) വഴി റിപ്പോർട്ട് ചെയ്യുക.

ലോഗിൻ ചെയ്യാതെ സമർപ്പിക്കാൻ നിങ്ങൾ ഇഷ്ടപ്പെടുന്നുവെങ്കിൽ, [secure@microsoft.com](mailto:secure@microsoft.com) എന്ന ഇമെയിലിലേക്ക് അയക്കുക. നിങ്ങളുടെ സന്ദേശം ഞങ്ങളുടെ PGP കീ ഉപയോഗിച്ച് എൻക്രിപ്റ്റ് ചെയ്യാൻ കഴിയുമെങ്കിൽ, ദയവായി അത് [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) ൽ നിന്ന് ഡൗൺലോഡ് ചെയ്യുക.

24 മണിക്കൂറിനുള്ളിൽ നിങ്ങൾക്ക് ഒരു പ്രതികരണം ലഭിക്കണം. എന്തെങ്കിലും കാരണത്താൽ നിങ്ങൾക്ക് ലഭിക്കാത്ത പക്ഷം, നിങ്ങളുടെ ആദ്യ സന്ദേശം ഞങ്ങൾ സ്വീകരിച്ചിട്ടുണ്ടെന്ന് ഉറപ്പാക്കാൻ ഇമെയിൽ വഴി പിന്തുടരുക. കൂടുതൽ വിവരങ്ങൾ [microsoft.com/msrc](https://www.microsoft.com/msrc) ൽ ലഭ്യമാണ്.

സാധ്യമായത്ര കൂടുതൽ വിവരങ്ങൾ (താങ്കൾക്ക് നൽകാൻ കഴിയുന്നത്ര) ഉൾപ്പെടുത്തുക, ഇത് പ്രശ്നത്തിന്റെ സ്വഭാവവും പരിധിയും നന്നായി മനസിലാക്കാൻ ഞങ്ങളെ സഹായിക്കും:

  * പ്രശ്നത്തിന്റെ തരം (ഉദാ. ബഫർ ഓവർഫ്ലോ, SQL ഇൻജക്ഷൻ, ക്രോസ്-സൈറ്റ് സ്ക്രിപ്റ്റിംഗ്, മുതലായവ)
  * പ്രശ്നം പ്രകടമാകുന്ന സോഴ്‌സ് ഫയലുകളുടെ പൂർണ്ണ പാതകൾ
  * ബാധിച്ച സോഴ്‌സ് കോഡിന്റെ സ്ഥാനം (ടാഗ്/ബ്രാഞ്ച്/കമിറ്റ് അല്ലെങ്കിൽ നേരിട്ടുള്ള URL)
  * പ്രശ്നം പുനരാവർത്തിക്കാൻ ആവശ്യമായ പ്രത്യേക കോൺഫിഗറേഷൻ
  * പ്രശ്നം പുനരാവർത്തിക്കാൻ ഘട്ടം ഘട്ടമായുള്ള നിർദ്ദേശങ്ങൾ
  * പ്രൂഫ്-ഓഫ്-കോൺസെപ്റ്റ് അല്ലെങ്കിൽ എക്സ്പ്ലോയിറ്റ് കോഡ് (സാധ്യമായെങ്കിൽ)
  * പ്രശ്നത്തിന്റെ സ്വാധീനം, ഒരു ആക്രമി പ്രശ്നം എങ്ങനെ ഉപയോഗപ്പെടുത്താൻ സാധ്യതയുണ്ട് എന്നതടക്കം

ഈ വിവരങ്ങൾ നിങ്ങളുടെ റിപ്പോർട്ട് വേഗത്തിൽ പരിഗണിക്കാൻ ഞങ്ങളെ സഹായിക്കും.

നിങ്ങൾ ബഗ് ബൗണ്ടിക്കായി റിപ്പോർട്ട് ചെയ്യുകയാണെങ്കിൽ, കൂടുതൽ പൂർണ്ണമായ റിപ്പോർട്ടുകൾ ഉയർന്ന ബൗണ്ടി അവാർഡിലേക്ക് സംഭാവന ചെയ്യാൻ കഴിയും. ഞങ്ങളുടെ സജീവ പ്രോഗ്രാമുകളെക്കുറിച്ചുള്ള കൂടുതൽ വിവരങ്ങൾക്കായി [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) പേജ് സന്ദർശിക്കുക.

## മുൻഗണനാ ഭാഷകൾ

എല്ലാ ആശയവിനിമയങ്ങളും ഇംഗ്ലീഷിൽ നടത്താൻ ഞങ്ങൾ മുൻഗണന നൽകുന്നു.

## നയം

മൈക്രോസോഫ്റ്റ് [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) എന്ന തത്വം പിന്തുടരുന്നു.
