## အနှစ်ချုပ်

**Co-op Translator** သည် သင်၏ပညာရေးဆိုင်ရာ GitHub အကြောင်းအရာများကို ဘာသာစကားအမျိုးမျိုးသို့ အလျင်အမြန် ဘာသာပြန်နိုင်စေပြီး ကမ္ဘာတစ်ဝှမ်းလုံးမှ ပရိသတ်များထံသို့ လွယ်ကူစွာ ရောက်ရှိစေနိုင်ပါသည်။ သင်၏ Markdown ဖိုင်များ၊ ပုံများ သို့မဟုတ် Jupyter notebooks များကို ပြင်ဆင်သည့်အခါ ဘာသာပြန်ချက်များကို အလိုအလျောက် တစ်ပြိုင်နက်တည်း အသစ်ပြင်ဆင်ပေးသဖြင့် သင်၏ပညာရေးဆိုင်ရာ GitHub အကြောင်းအရာများသည် နိုင်ငံတကာအသုံးပြုသူများအတွက် အမြဲတမ်း အသစ်နှင့် သက်ဆိုင်မှုရှိနေစေပါသည်။

Co-op Translator သည် ဘာသာပြန်ထားသော ပညာရေးဆိုင်ရာ GitHub အကြောင်းအရာများကို မည်သို့ စီမံခန့်ခွဲထားသည်ကို ကြည့်ပါ-

![ဥပမာ](../../translated_images/translation-ex.0c8aa6a7ee0aad2b35cddcc110c719baf0afc640e8c5a45540e6c166b9907d91.my.png)

## အလွယ်တကူ စတင်အသုံးပြုရန်

```bash
# Create and activate a virtual environment (recommended)
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
# Install the package
pip install co-op-translator
# Translate
translate -l "ko ja fr" -md
```

Docker:

```bash
# Pull the public image from GHCR
docker pull ghcr.io/azure/co-op-translator:latest
# Run with current folder mounted and .env provided (Bash/Zsh)
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko ja fr" -md
```

## အနည်းဆုံးလိုအပ်ချက်များ

- `.env` ကို ဒီ template ကို အသုံးပြု၍ ဖန်တီးပါ - [.env.template](../../.env.template)
- LLM provider တစ်ခု (Azure OpenAI သို့မဟုတ် OpenAI) ကို ပြင်ဆင်ပါ
- ပုံများကို ဘာသာပြန်လိုပါက (`-img`), Azure AI Vision ကိုလည်း ပြင်ဆင်ပါ
- အကြံပြုချက် - အခြား tool များဖြင့် ဘာသာပြန်ထားသော ဖိုင်များရှိပါက ပထမဦးဆုံး သန့်ရှင်းပါ (ဥပမာ- `translations/`)
- အကြံပြုချက် - README တွင် ဘာသာပြန်ချက်များအကြောင်း အပိုင်းတစ်ခု ထည့်ပါ ([README languages template](./README_languages_template.md) ကို အသုံးပြုနိုင်သည်)
- ကြည့်ရန် - [Azure AI ကို ပြင်ဆင်ခြင်း](./getting_started/set-up-azure-ai.md)

## အသုံးပြုနည်း

ထောက်ပံ့ထားသော အမျိုးအစားအားလုံးကို ဘာသာပြန်ရန်-

```bash
translate -l "ko ja"
```

Markdown သာ-

```bash
translate -l "de" -md
```

Markdown + ပုံများ-

```bash
translate -l "pt" -md -img
```

Notebook များသာ-

```bash
translate -l "zh" -nb
```

နောက်ထပ် flag များ - [Command reference](./getting_started/command-reference.md)

Microsoft သည် ကျွန်ုပ်တို့၏ AI ထုတ်ကုန်များကို တာဝန်ရှိစွာ အသုံးပြုနိုင်ရန် ကူညီပေးရန်၊ ကျွန်ုပ်တို့၏ အတွေ့အကြုံများကို မျှဝေရန်နှင့် Transparency Notes နှင့် Impact Assessments ကဲ့သို့သော ကိရိယာများဖြင့် ယုံကြည်မှုအပေါ် အခြေခံသော မိတ်ဖက်ဆက်ဆံရေးများ တည်ဆောက်ရန် တာဝန်ယူပါသည်။ ဤအရင်းအမြစ်များအနက် အများစုကို [https://aka.ms/RAI](https://aka.ms/RAI) တွင် ရှာဖွေနိုင်ပါသည်။
Microsoft ၏ တာဝန်ရှိသော AI သုံးသပ်ချက်သည် တရားမျှတမှု၊ ယုံကြည်စိတ်ချရမှုနှင့် လုံခြုံရေး၊ ကိုယ်ရေးအချက်အလက်နှင့် လုံခြုံရေး၊ ပါဝင်နိုင်မှု၊ ပွင့်လင်းမြင်သာမှုနှင့် တာဝန်ယူမှုတို့ကို အခြေခံထားပါသည်။

ဤနမူနာတွင် အသုံးပြုထားသည့် သဘာဝဘာသာစကား၊ ပုံနှင့် အသံအကြီးစား မော်ဒယ်များသည် တရားမျှတမှုမရှိခြင်း၊ ယုံကြည်စိတ်ချရမှုမရှိခြင်း သို့မဟုတ် စိတ်ထိခိုက်စေသော အပြုအမူများ ပြုလုပ်နိုင်သဖြင့် အန္တရာယ်များ ဖြစ်ပေါ်နိုင်ပါသည်။ အန္တရာယ်များနှင့် ကန့်သတ်ချက်များအကြောင်း သိရှိရန် [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ကို ဖတ်ရှုပါ။

ဤအန္တရာယ်များကို လျှော့ချရန် အကြံပြုချက်မှာ သင့်စနစ်အတွင်း အန္တရာယ်ရှိသော အပြုအမူများကို ရှာဖွေနိုင်ပြီး တားဆီးနိုင်သော လုံခြုံရေးစနစ်တစ်ခု ထည့်သွင်းပါ။ [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) သည် လွတ်လပ်သော ကာကွယ်မှုအလွှာတစ်ခုဖြစ်ပြီး အသုံးပြုသူများနှင့် AI မှ ထုတ်လုပ်သော အကြောင်းအရာအန္တရာယ်ရှိမှုကို ရှာဖွေနိုင်ပါသည်။ Azure AI Content Safety တွင် စာသားနှင့် ပုံ API များပါဝင်ပြီး အန္တရာယ်ရှိသော အကြောင်းအရာများကို ရှာဖွေနိုင်ပါသည်။ Content Safety Studio ကိုလည်း အသုံးပြု၍ modality များစွာအတွက် အန္တရာယ်ရှိသော အကြောင်းအရာများကို ရှာဖွေရန် နမူနာကုဒ်များကို ကြည့်ရှု၊ စမ်းသပ်နိုင်ပါသည်။ အောက်ပါ [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) တွင် ဝန်ဆောင်မှုသို့ တောင်းဆိုနည်းကို လမ်းညွှန်ပေးထားပါသည်။
တစ်ခုထပ်ပြီးစဉ်းစားရမယ့်အချက်ကတော့ အပလီကေးရှင်းရဲ့ စုစုပေါင်းလုပ်ဆောင်ရည်ပါ။ မူလတန်းအမျိုးမျိုးနဲ့ မော်ဒယ်အမျိုးမျိုးပါဝင်တဲ့ အပလီကေးရှင်းတွေမှာ လုပ်ဆောင်ရည်ဆိုတာ သင်နဲ့ သုံးစွဲသူတွေ မျှော်လင့်ထားသလို စနစ်က အလုပ်လုပ်ပေးတာ၊ အန္တရာယ်ရှိတဲ့ output မထုတ်ပေးတာပါ။ သင့်ရဲ့ အပလီကေးရှင်းတစ်ခုလုံးရဲ့ လုပ်ဆောင်ရည်ကို [generation quality နဲ့ risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) တွေသုံးပြီး တိုင်းတာသုံးသပ်ဖို့ အရေးကြီးပါတယ်။

သင့်ရဲ့ AI အပလီကေးရှင်းကို development environment မှာ [prompt flow SDK](https://microsoft.github.io/promptflow/index.html) သုံးပြီး တိုင်းတာနိုင်ပါတယ်။ Test dataset တစ်ခုဖြစ်စေ၊ target တစ်ခုဖြစ်စေ၊ သင့်ရဲ့ generative AI application output တွေကို built-in evaluators သို့မဟုတ် သင်ရွေးချယ်ထားတဲ့ custom evaluators တွေနဲ့ တိကျစွာ တိုင်းတာနိုင်ပါတယ်။ prompt flow sdk ကို သုံးပြီး သင့်စနစ်ကို တိုင်းတာချင်ရင် [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) ကို လိုက်နာနိုင်ပါတယ်။ Evaluation run တစ်ခုလုပ်ပြီးသွားရင် [Azure AI Studio မှာ ရလဒ်တွေကို မြင်နိုင်ပါတယ်](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results)။
