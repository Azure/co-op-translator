## Visão geral

**Co-op Translator** ajuda a localizar o seu conteúdo educativo do GitHub em múltiplos idiomas sem esforço.
Quando atualiza os seus ficheiros Markdown, imagens ou notebooks, as traduções mantêm-se automaticamente sincronizadas, assegurando que o seu conteúdo permanece preciso e atualizado para alunos em todo o mundo.

Use-o a partir da CLI para traduzir repositórios, a partir da API Python para automatização, ou através do servidor MCP para fluxos de trabalho com agentes e editores.

## Porque o Co-op Translator?

Traduzir um ficheiro é fácil. Manter um repositório inteiro de documentação
traduzido, ligado e atualizado é a parte difícil.

| Problema | Como o Co-op Translator ajuda |
| --- | --- |
| Long docs are not one prompt | Ficheiros Markdown grandes são divididos em blocos, de modo que um README longo não depende de uma única resposta frágil do modelo. Se um bloco falhar, o Co-op Translator pode tentar novamente e voltar a dividir apenas a parte falhada. |
| Incomplete translations should not be marked current | Uma tradução truncada nunca deve ser assinalada como atual. O Co-op Translator verifica a integridade da tradução antes de guardar e pode detetar traduções existentes estruturalmente incompletas. |
| Links should match the translated repo structure | Traduções manuais frequentemente deixam links relativos apontando de volta para a árvore de origem. O Co-op Translator reescreve ligações em Markdown, notebooks, imagens e README para corresponder à estrutura `translations/<lang>/...`. |
| Translation should work across an entire repo | O Co-op Translator trata ficheiros README, documentação, notebooks e texto de imagens como parte de um único fluxo de trabalho do repositório, em vez de traduzir ficheiros um a um. |
| Maintaining translations matters more than creating them once | Hashes de origem e metadados de tradução permitem ao Co-op Translator encontrar ficheiros desatualizados, saltar ficheiros não alterados e manter o conteúdo traduzido sincronizado à medida que o repositório fonte evolui. |

## Como é gerido o estado da tradução

O Co-op Translator gere o conteúdo traduzido como **artefactos de software versionados**,  
não como ficheiros estáticos.

A ferramenta rastreia o estado de Markdown, imagens e notebooks traduzidos
usando **metadados com âmbito por idioma**.

Este design permite ao Co-op Translator:

- Detetar fiavelmente traduções desatualizadas
- Tratar Markdown, imagens e notebooks de forma consistente
- Escalar com segurança em repositórios grandes, dinâmicos e multilíngues

Ao modelar traduções como artefactos geridos,
os fluxos de trabalho de tradução alinham-se naturalmente com as práticas modernas
de gestão de dependências e artefactos de software.

→ [Como é gerido o estado da tradução](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)

## Começar

O Co-op Translator pode ser usado a partir da CLI, da API Python, ou do servidor MCP. Comece pelo guia de fluxo de trabalho se estiver a escolher entre tradução local, automatização, CI, e integração com agentes/editors.

- [Escolha o seu fluxo de trabalho](../../docs/workflows.md)
- [Configurar credenciais](../../docs/configuration.md)
- [Traduzir a partir da CLI](../../docs/cli.md)
- [Automatizar com a API Python](../../docs/api.md)
- [Ligar ao servidor MCP](../../docs/mcp.md)
- [Executar no GitHub Actions](../../docs/github-actions.md)

Exemplo mínimo de CLI após a configuração:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate

pip install co-op-translator
translate -l "ko" -md
co-op-review -l "ko"
```

Para as primeiras execuções em repositórios grandes, use `--dry-run` antes de escrever ficheiros traduzidos. Veja a [Referência da CLI](../../docs/cli.md) para flags de tipo de conteúdo, logs, revisão e migração de links.

Execução rápida em contentor com Bash/Zsh:

```bash
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

Execução rápida em contentor com PowerShell:

```powershell
docker run --rm -it --env-file .env -v ${PWD}:/work ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

## Funcionalidades

- Tradução automatizada para Markdown, notebooks e imagens
- Mantém as traduções sincronizadas com as alterações de origem
- Funciona localmente (CLI) ou em CI (GitHub Actions)
- Expõe ferramentas de tradução de Markdown, notebooks, imagens, revisão e projetos através do MCP
- Usa Azure OpenAI ou OpenAI para tradução suportada por fornecedor
- Permite que o MCP hospede agentes que traduzem blocos de Markdown e notebooks sem credenciais LLM do Co-op Translator
- Usa Azure AI Vision para extração e tradução de texto em imagens
- Revê a estrutura e a atualidade das traduções com verificações determinísticas
- Preserva a formatação e estrutura do Markdown

## IA Responsável

A Microsoft compromete-se a ajudar os nossos clientes a utilizar os nossos produtos de IA de forma responsável, a partilhar os nossos aprendizados e a construir parcerias baseadas na confiança através de ferramentas como Notas de Transparência e Avaliações de Impacto. Muitos destes recursos podem ser encontrados em [https://aka.ms/RAI](https://aka.ms/RAI).
A abordagem da Microsoft à IA responsável está fundamentada nos nossos princípios de IA de equidade, fiabilidade e segurança, privacidade e segurança, inclusão, transparência e responsabilidade.

Modelos de linguagem natural, imagem e fala de grande escala — como os utilizados neste exemplo — podem potencialmente comportar-se de maneiras que sejam injustas, pouco fiáveis ou ofensivas, causando danos. Consulte a [Nota de Transparência do serviço Azure OpenAI](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) para se informar sobre riscos e limitações.

A abordagem recomendada para mitigar estes riscos é incluir um sistema de segurança na sua arquitetura que consiga detetar e prevenir comportamentos prejudiciais. [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) fornece uma camada independente de proteção, capaz de detetar conteúdo prejudicial gerado por utilizadores e por IA em aplicações e serviços. O Azure AI Content Safety inclui APIs de texto e imagem que lhe permitem detetar material que é prejudicial. Também temos um Content Safety Studio interativo que lhe permite ver, explorar e experimentar exemplos de código para detetar conteúdo prejudicial em diferentes modalidades. A seguinte [documentação de início rápido](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) orienta-o na realização de pedidos ao serviço.

Outro aspeto a ter em conta é o desempenho global da aplicação. Em aplicações multimodais e com múltiplos modelos, consideramos desempenho o facto de o sistema funcionar como espera, incluindo não gerar saídas prejudiciais. É importante avaliar o desempenho da sua aplicação global utilizando [métricas de qualidade de geração e de risco e segurança](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in).

Pode avaliar a sua aplicação de IA no seu ambiente de desenvolvimento usando o [prompt flow SDK](https://microsoft.github.io/promptflow/index.html). Com um conjunto de dados de teste ou um objetivo, as gerações da sua aplicação de IA generativa são medidas quantitativamente com avaliadores integrados ou avaliadores personalizados à sua escolha. Para começar com o prompt flow SDK para avaliar o seu sistema, pode seguir o [guia de início rápido](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk). Depois de executar uma execução de avaliação, pode [visualizar os resultados no Azure AI Studio](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).
