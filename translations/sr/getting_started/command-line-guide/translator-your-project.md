# Преведите свој пројекат користећи Co-op Translator

**Co-op Translator** је алат за командну линију (CLI) који вам помаже да преведете markdown и сликовне фајлове у вашем пројекту на више језика. Овај одељак објашњава како користити алат, обухвата различите CLI опције и пружа примере за различите случајеве употребе.

> [!NOTE]
> За комплетну листу команди и њихових детаљних описа, погледајте [Command reference](./command-reference.md).

---

## Примери сценарија и команде

Ево неколико уобичајених случајева употребе **Co-op Translator** заједно са одговарајућим командама.

### 1. Основни превод (један језик)

Да бисте превели цео пројекат (markdown фајлове и слике) на један језик, на пример корејски, користите следећу команду:

```bash
translate -l "ko"
```

Ова команда ће превести све markdown и сликовне фајлове на корејски, додајући нове преводе без брисања постојећих.

> [!TIP]
>
> Желите да видите који језички кодови су доступни у **Co-op Translator**? Посетите одељак [Supported Languages](https://github.com/Azure/co-op-translator#supported-languages) у репозиторијуму за више детаља.

#### Пример на Phi-3 CookBook

У **Phi-3 CookBook** користио сам следећи начин да додам корејски превод за постојеће markdown фајлове и слике.

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko"
Translating images: 100%|███████████████████████████████████████████████████| 276/276 [1:09:56<00:00, 15.37s/it]
Translating markdown files: 100%|████████████████████████████████████████████████| 153/153 [1:43:07<00:00, 241.31s/it]
```

### 2. Превођење на више језика

Да бисте превели пројекат на више језика (нпр. шпански, француски и немачки), користите ову команду:

```bash
translate -l "es fr de"
```

Ова команда ће превести пројекат на шпански, француски и немачки, додајући нове преводе без преписивања постојећих.

#### Пример на Phi-3 CookBook

У **Phi-3 CookBook**, након што сам повукао најновије измене да би одразио најсвежије комите, користио сам следећи начин да преведем ново додате markdown фајлове и слике.

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko ja zh tw es fr" -a
Translating images: 100%|███████████████████████████████████████████████████| 273/273 [1:09:56<00:00, 15.37s/it]
Translating markdown files: 100%|████████████████████████████████████████████████| 6/6 [24:07<00:00, 241.31s/it]
```

> [!NOTE]
> Иако је генерално препоручљиво преводити један језик по један, у оваквим ситуацијама где треба додати специфичне измене, превођење више језика одједном може бити ефикасно.

### 3. Ажурирање превода (брише постојеће преводе)

Да бисте ажурирали постојеће преводе (односно обрисали тренутне преводе и заменили их новима), користите опцију `-u`. Ово ће обрисати све постојеће преводе за назначене језике и поново их превести.

```bash
translate -l "ko" -u
```

Упозорење: Ова команда ће вас тражити потврду пре него што обрише постојеће преводе.

#### Пример на Phi-3 CookBook

У **Phi-3 CookBook** користио сам следећи начин да ажурирам све преведене фајлове на шпанском. Препоручујем ову методу када постоје значајне промене у оригиналном садржају у више markdown докумената. Ако треба ажурирати само неколико преведених markdown фајлова, ефикасније је ручно обрисати те фајлове, па онда користити метод `-a` за додавање ажурираног превода.

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l "es" -u

```

### 5. Превођење само слика

Да бисте превели само сликовне фајлове у пројекту, користите опцију `-img`:

```bash
translate -l "ko" -img
```

Ова команда ће превести само слике на корејски, без утицаја на markdown фајлове.

### 6. Превођење само markdown фајлова

Да бисте превели само markdown фајлове у пројекту, користите опцију `-md`:

```bash
translate -l "ko" -md
```

### 7. Провера грешака у преведеним фајловима

Ако желите да проверите преведене фајлове на грешке и по потреби поново покренете превод, користите опцију `-chk`:

```bash
translate -l "ko" -chk
```

Ова команда ће прегледати преведене markdown фајлове и поновити превод за све фајлове у којима се појаве грешке.

#### Пример на Phi-3 CookBook

У **Phi-3 CookBook** користио сам следећи начин да проверим грешке у преводу корејских фајлова и аутоматски поновим превод за све фајлове у којима су откривени проблеми.

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko" -chk 
Checking translated files for errors in ko...
Checking files for ko: 100%|██████████████████████████████████████████████████| 95/95 [00:01<00:00, 65.47file/s]
Retrying vsc-extension-quickstart.md for ko:   0%|                                     | 0/17 [00:00<?, ?file/s] 
```

Ова опција проверава грешке у преводу. Тренутно, ако је разлика у прекидима линија између оригиналног и преведеног фајла већа од шест, фајл се означава као да има грешку у преводу. Планирам да у будућности унапредим овај критеријум ради веће флексибилности.

На пример, овај метод је користан за откривање недостајућих делова или оштећених превода, и аутоматски ће поновити превод за те фајлове.

Међутим, ако већ знате који су фајлови проблематични, ефикасније је ручно обрисати те фајлове и користити опцију `-a` option to re-translate them.

### 8. Debug Mode

To enable detailed logging for troubleshooting, use the `-d`:

```bash
translate -l "ko" -d
```

Ова команда ће покренути превод у debug режиму, пружајући додатне информације у логовима које вам могу помоћи да идентификујете проблеме током превода.

### 9. Превођење на све језике

Ако желите да преведете пројекат на све подржане језике, користите кључну реч all.

> [!WARNING]
> Превођење свих језика одједном може трајати знатно времена у зависности од величине пројекта. На пример, превођење **Phi-3 CookBook** на шпански је трајало око 2 сата. С обзиром на обим, није практично да једна особа ради на 20 језика. Препоручује се да се посао подели између више сарадника, од којих сваки покрива један или два језика, и да се преводи постепено ажурирају.

```bash
translate -l "all"
```

Ова команда ће превести пројекат на све доступне језике. Ако наставите, превођење може трајати значајно времена у зависности од величине пројекта.

> [!TIP]
>
> ### Ручно брисање преведених фајлова (опционо)
> Преведени фајлови се сада аутоматски детектују и чисте када се ажурира изворни фајл.
>
> Међутим, ако желите ручно да ажурирате превод - на пример, да поново урадите одређени фајл или да препишете понашање система - можете користити следећу команду да обришете све верзије тог фајла у свим језичким фолдерима.
>
> ### На Windows-у:
> 1. **Коришћење Command Prompt-а**:
>    - Отворите Command Prompt.
>    - Идите у фасциклу где се налазе фајлови помоћу команде `cd`.
>    - Користите следећу команду за брисање фајлова:
>      ```
>      del /s *filename*
>      ```
>      Опција `/s` претражује и поддиректоријуме.
>
> 2. **Коришћење PowerShell-а**:
>    - Отворите PowerShell.
>    - Покрените ову команду:
>      ```powershell
>      Get-ChildItem -Path "C:\YourPath" -Filter "*filename*" -Recurse | Remove-Item -Force
>      ```
>      Команда `"C:\YourPath"` with the folder path and `filename` with the specific name.
>
> ### On macOS/Linux:
> 1. **Using Terminal**:
>   - Open Terminal.
>   - Navigate to the directory with `cd`.
>   - Use the `find`:
>     ```bash
>     find . -type f -name "*filename*" -delete
>     ```
>     Команда `filename` with the specific name.
>
> Always double-check the files before deleting to avoid accidental loss. 
>
> Once you have deleted the files which need to be replace simply rerun your `translate -l` служи за ажурирање најновијих промена у фајловима.
