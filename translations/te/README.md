## అవలోకనం

**Co-op Translator** మీ విద్యా సంబంధ GitHub కంటెంట్‌ను అనేక భాషల్లో స్థానికీకరించడానికి సులభంగా సహాయపడుతుంది.
మీ Markdown ఫైళ్లు, చిత్రాలు, లేదా నోట్‌బుక్స్‌ను నవీకరించినప్పుడు, అనువాదాలు ఆటోమేటిక్‌గా సమకాలీకృతంగా ఉంటాయి, తద్వారా ప్రపంచంలోని అభ్యాసులకు మీ కంటెంట్ ఖచ్చితంగా మరియు తాజాగా ఉంటుంది.

Repository అనువాదం కోసం CLI ద్వారా, ఆటోమేషన్ కోసం Python API ద్వారా, లేదా ఏజెంట్ మరియు ఎడిటర్ వర్క్‌ఫ్లోల కోసం MCP సర్వర్ ద్వారా దీన్ని ఉపయోగించండి.

అనువాద స్థానికీకరించిన కంటెంట్ ఎలా వ్యవస్థీకృతమవుతుందో ఉదాహరణ:

## Co-op Translator ఎందుకు?

ఒక ఫైల్‌ను అనువదించడం సులభమే. మొత్తం డాక్యుమెంటేషన్ రిపోజిటరీని
అనువదించిన, లింక్ చేసిన, మరియు తాజాగా ఉంచుకోవడం కష్టం.

| Problem | How Co-op Translator helps |
| --- | --- |
| Long docs are not one prompt | పెద్ద Markdown ఫైళ్లు చిన్న భాగాలుగా విడగొడతారు, తద్వారా ఒక పొడవైన README ఒకే ఒక అస్థిర మోడల్ రెస్పాన్స్ మీద ఆధారపడదు. ఒక భాగం విఫలమైనట్లయితే, Co-op Translator ఆ విఫలమైన భాగాన్ని మాత్రమే తిరిగి ప్రయత్నించగలదు మరియు మళ్లీ ఛంక్ చేయగలదు. |
| Incomplete translations should not be marked current | ఒక అనువాదం మధ్యలో ఉన్నప్పుడే దానిని తాజాగా అయ్యిందని గుర్తించకూడదు. Co-op Translator సేవ్ చేసేముందు అనువాద సమగ్రతను తనిఖీ చేస్తుంది మరియు నిర్మాణాత్మకంగా అసంపూర్ణంగా ఉన్న అనువాదాలను గుర్తించగలదు. |
| Links should match the translated repo structure | మాన్యువల్ అనువాదాలు తరచుగా సంబంధిత లింక్స్‌ను మూల సంకేతనకు పాయింట్ చేసినియ్యవచ్చు. Co-op Translator `translations/<lang>/...` నిర్మాణంతో అనుకూలంగా Markdown, నోట్‌బుక్, చిత్రం మరియు README లింక్స్‌ను రీరైట్రైట్ చేస్తుంది. |
| Translation should work across an entire repo | Co-op Translator README ఫైళ్లు, డాక్స్, నోట్‌బుక్స్, మరియు చిత్రం టెక్స్ట్‌లను ఒక్క రిపాజిటరీ వర్క్‌ఫ్లో భాగంగా హ్యాండిల్ చేస్తుంది, ఫైళులను ఒక్కొక్కటిగా అనువదించే బదులుగా. |
| Maintaining translations matters more than creating them once | సోర్స్ హాష్‌లు మరియు అనువాద మెటాడేటా Co-op Translator‌కు పాత అనువాద ఫైళ్ళను కనబడడానికి, మార్పులేని ఫైళ్ళను స్కిప్ చేయడానికి, మరియు సోర్స్ రిపో మారినప్పుడు అనువదించిన కంటెంట్‌ను సమకాలీకరించడానికి సహాయపడతాయి. |

## అనువాద స్థితి ఎలా నిర్వహించబడుతుంది

Co-op Translator అనువాద పదార్థాలను స్థిర ఫైళ్లుగా కాకుండా **వర్షన్లు ఉన్న సాఫ్ట్‌వేర్ ఆర్టిఫాక్ట్స్**గా నిర్వహిస్తుంది,  
స్థిర ఫైళ్లు కాదన్నారు.

ఈ టూల్ అనువాదించిన Markdown, చిత్రాలు, మరియు నోట్‌బుక్స్ యొక్క స్థితిని ట్రాక్ చేయడానికి **భాష పరిమిత మెటాడేటా** ఉపయోగిస్తుంది.

ఈ రూపకల్పన Co-op Translator కు అవకాశం ఇస్తుంది:

- పాత అనువాదాలను విశ్వసనీయంగా గుర్తించడానికి
- Markdown, చిత్రాలు, మరియు నోట్‌బుక్స్‌ను సुसంగతంగా నిర్వహించడానికి
- పెద్ద, వేగంగా మారే బహుభాషా రిపోజిటరీలలో భద్రంగా స్కేలు చేయడానికి

అనువాదాలను నిర్వహించబడిన ఆర్టిఫాక్ట్స్‌గా మోడలింగ్ చేయడం ద్వారా,
అనువాద వర్క్‌ఫ్లోలు ఆధునిక
సాఫ్ట్‌వేర్ డిపెండెన్సీ మరియు ఆర్టిఫాక్ట్ నిర్వహణ పద్ధతులతో సహజంగా సర్దుబాటు అవుతాయి.

→ [అనువాద స్థితి ఎలా నిర్వహించబడుతుంది](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)

## ప్రారంభించండి

Co-op Translator ను CLI, Python API, లేదా MCP సర్వర్ నుండి ఉపయోగించవచ్చు. మీరు స్థానిక అనువాదం, ఆటోమేషన్, CI, మరియు ఏజెంట్/ఎడిటర్ ఇంటిగ్రేషన్ మధ్య ఎంచుకుంటున్నట్లయితే వర్క్‌ఫ్లో గైడ్ నుండి ప్రారంభించండి.

- [Choose your workflow](../../docs/workflows.md)
- [Configure credentials](../../docs/configuration.md)
- [Translate from the CLI](../../docs/cli.md)
- [Automate with the Python API](../../docs/api.md)
- [Connect with the MCP Server](../../docs/mcp.md)
- [Run in GitHub Actions](../../docs/github-actions.md)

కాన్ఫిగరేషన్ అనంతరం కనిష్ట CLI ఉదాహరణ:

```bash
python -m venv .venv
# విండోస్
.venv\Scripts\activate
# మ్యాక్ ఓఎస్/లినక్స్
source .venv/bin/activate

pip install co-op-translator
translate -l "ko" -md
co-op-review -l "ko"
```

పెద్ద రిపోజిటరీలపై మొదటి రన్స్ సమయంలో, అనువాద ఫైళ్ళను రాయడానికి ముందు `--dry-run` ఉపయోగించండి. కంటెంట్ రకం ఫ్లాగ్‌లు, లాగ్స్, సమీక్ష, మరియు లింక్ మైగ్రేషన్ కోసం [CLI Reference](../../docs/cli.md) చూడండి.

Bash/Zsh తో కంటెయినర్ క్విక్ రన్:

```bash
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

PowerShell తో కంటెయినర్ క్విక్ రన్:

```powershell
docker run --rm -it --env-file .env -v ${PWD}:/work ghcr.io/azure/co-op-translator:latest -l "ko" -md
```

## ఫీచర్లు

- Markdown, నోట్‌బుక్స్, మరియు చిత్రాల కోసం ఆటోమేటెడ్ అనువాదం
- మూల మార్పులతో అనువాదాలను సమకాలీకరిస్తుంది
- స్థానికంగా (CLI) లేదా CI (GitHub Actions) లో పనిచేస్తుంది
- MCP ద్వారా Markdown, నోట్‌బుక్, చిత్రం, సమీక్ష, మరియు ప్రాజెక్ట్ అనువాద టూల్స్ ను ఎక్స్‌పోజ్ చేస్తుంది
- provider-బ్యాక్ చేసిన అనువాదానికి Azure OpenAI లేదా OpenAI ను ఉపయోగిస్తుంది
- MCP ఏజెంట్లు Co-op Translator LLM క్రెడెంజియల్స్ లేకుండానే Markdown మరియు నోట్‌బుక్ ఛంక్‌లను అనువదించేందుకు హోస్ట్ చేయగలదు

## బాధ్యతాయుత AI

Microsoft తన వినియోగదారులు మా AI ఉత్పత్తులను బాధ్యతాయుతంగా ఉపయోగించడంలో సహాయపడటానికి, మా అనుభవాలను పంచుకోవడానికి, Transparency Notes మరియు Impact Assessments వంటి సాధనాల ద్వారా నమ్మకంపై ఆధారపడిన భాగస్వామ్యాలను నిర్మించడానికి బద్ధలైంది. ఈ వనరులలో చాలా వాటిని [https://aka.ms/RAI](https://aka.ms/RAI) వద్ద పొందవచ్చు.
Microsoft బాధ్యతాయుత AI పైన పాటించే దృష్టికోణం మా AI సూత్రాలైన సమానత్వం, విశ్వసనీయత మరియు భద్రత, గోప్యత మరియు భద్రత, సమావేశత, పారదర్శకత మరియు జవాబుదారీతనంపై ఆధారపడినది.

ఈ నమూనాలో ఉపయోగించినట్లుగానే పెద్ద సంఖ్యలో సహజ భాషా, చిత్రం మరియు మాట్లాడే మోడల్‌లు అనవసరంగా అసమానమైన, అవిశ్వసనీయమైన లేదా ఆగ్రహకరమైన ప్రవర్తన చూపవచ్చు, ఫలితంగా హానులు జరగవచ్చు. ప్రమాదాలు మరియు పరిమితుల గురించి సమాచారానికి దయచేసి [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ని చూడండి.

ఈ రిస్క్‌లను తగ్గించడానికి సిఫార్సు చేయబడిన పద్ధతి మీ ఆర్కిటెక్చర్లో హానికర ప్రవర్తనను గుర్తించి నివారించగల సేఫ్టీ వ్యవస్థను చేర్చడం. [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) అప్లికేషన్లు మరియు సేవలలో హానికరమైన వినియోగదారు-సృష్టించిన మరియు AI-సృష్టించిన కంటెంట్‌ను గుర్తించగల ఒక స్వతంత్ర రక్షణ పొరను అందిస్తుంది. Azure AI Content Safety టెక్స్ట్ మరియు ఇమేజ్ APIలను కలిగి ఉండి, హానికరమైన పదార్ధాలను గుర్తించడానికి వీలు కల్పిస్తుంది. మనం ఇంటరాక్టివ్ Content Safety Studio ని కూడా కలిగి ఉన్నాము, ఇది వివిధ మోడాలిటీల్లో హానికర కంటెంట్‌ను గుర్తించడానికి నమూనా కోడ్‌ను వీక్షించడానికి, అన్వేషించడానికి మరియు ప్రయోగించడానికి అనుమతిస్తుంది. క్రింది [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) మీరు సేవకు అభ్యర్థనలు చేయడాన్ని మీకు మార్గనిర్దేశం చేస్తుంది.

మరొక విషయంగా పరిగణలోకి తీసుకోవాల్సింది మొత్తం అప్లికేషన్ పనితీరు. బహు-మోడల్ మరియు బహు-మొడ్యూల్ అప్లికేషన్లలో, పనితీరు అంటే వ్యవస్థ మీరు మరియు మీ వినియోగదారులు ఆశించేలా పని చేయడం, హానికర అవుట్పుట్‌లను ఉత్పత్తి కాకపోవడం అనే అర్థంలోనూ ఉంటుంది. మీ మొత్తం అప్లికేషన్ పనితీరును అంచనా వేయడానికి [generation quality and risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) ఉపయోగించడం ముఖ్యం.

మీ డెవలప్‌మెంట్ పరిసరాలలో [prompt flow SDK](https://microsoft.github.io/promptflow/index.html) ఉపయోగించి మీ AI అప్లికేషన్‌ను మూల్యాంకనం చేయవచ్చు. పరీక్షా డేటాసెట్ లేదా లక్ష్యం ఇచ్చినప్పుడు, మీ జనరేటివ్ AI అప్లికేషన్ ఉత్పత్తులను బిల్ట్-ఇన్ ఎవాల్యువేటర్లు లేదా మీకు అవసరమైన కస్టమ్ ఎవాల్యువేటర్ల ద్వారా పరిమాణాత్మకంగా కొలవవచ్చు. మీ వ్యవస్థను మూల్యాంకనం చేయడానికి prompt flow sdk తో ప్రారంభించడానికి, మీరు [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) ను అనుసరించవచ్చు. ఒక మూల్యాంకన రన్‌ను అమలు చేసిన తరువాత, మీరు [visualize the results in Azure AI Studio](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results) ద్వారా ఫలితాలను దృశ్యరూపంలో చూడవచ్చు.
