# Перекладіть свій проект за допомогою Co-op Translator

**Co-op Translator** — це інструмент командного рядка (CLI), який допомагає перекладати markdown-файли та зображення у вашому проєкті на кілька мов. У цьому розділі пояснюється, як користуватися інструментом, розглядаються різні опції CLI та наведено приклади для різних сценаріїв використання.

> [!NOTE]
> Для повного списку команд і детальних описів зверніться до [Command reference](./command-reference.md).

---

## Приклади сценаріїв і команд

Ось кілька поширених випадків використання **Co-op Translator** із відповідними командами.

### 1. Базовий переклад (одна мова)

Щоб перекласти весь проєкт (markdown-файли та зображення) на одну мову, наприклад корейську, використайте таку команду:

```bash
translate -l "ko"
```

Ця команда перекладе всі markdown-файли та зображення на корейську, додаючи нові переклади без видалення існуючих.

> [!TIP]
>
> Хочете дізнатися, які коди мов підтримуються в **Co-op Translator**? Перейдіть до розділу [Supported Languages](https://github.com/Azure/co-op-translator#supported-languages) у репозиторії для детальнішої інформації.

#### Приклад на Phi-3 CookBook

У **Phi-3 CookBook** я використовував наступний спосіб, щоб додати корейський переклад для наявних markdown-файлів та зображень.

```bash

```

### 2. Переклад кількох мов

Щоб перекласти проєкт на кілька мов (наприклад, іспанську, французьку та німецьку), використайте цю команду:

```bash
translate -l "es fr de"
```

Ця команда перекладе проєкт іспанською, французькою та німецькою, додаючи нові переклади без перезапису існуючих.

#### Приклад на Phi-3 CookBook

У **Phi-3 CookBook**, після отримання останніх змін для відображення найновіших комітів, я використав наступний спосіб для перекладу нових markdown-файлів та зображень.

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko ja zh tw es fr" -a
Translating images: 100%|███████████████████████████████████████████████████| 273/273 [1:09:56<00:00, 15.37s/it]
Translating markdown files: 100%|████████████████████████████████████████████████| 6/6 [24:07<00:00, 241.31s/it]
```

> [!NOTE]
> Хоча зазвичай рекомендується перекладати по одній мові за раз, у випадках, коли потрібно додати конкретні зміни, переклад кількох мов одночасно може бути ефективним.

### 3. Оновлення перекладів (видаляє існуючі переклади)

Щоб оновити існуючі переклади (тобто видалити поточні переклади та замінити їх новими), використайте опцію `-u`. Вона видалить усі наявні переклади для вказаних мов і виконає переклад заново.

```bash
translate -l "ko" -u
```

Увага: ця команда запросить підтвердження перед тим, як видалити існуючі переклади.

#### Приклад на Phi-3 CookBook

У **Phi-3 CookBook** я використовував цей спосіб для оновлення всіх перекладених файлів іспанською. Рекомендую застосовувати цей метод, коли в оригінальному вмісті відбулися значні зміни у кількох markdown-документах. Якщо потрібно оновити лише кілька перекладених markdown-файлів, ефективніше вручну видалити ці конкретні файли, а потім додати оновлені переклади за допомогою методу `-a`.

### 5. Переклад лише зображень

Щоб перекласти тільки зображення у вашому проєкті, використайте опцію `-img`:

```bash
translate -l "ko" -img
```

Ця команда перекладе лише зображення на корейську, не зачіпаючи markdown-файли.

### 6. Переклад лише markdown-файлів

Щоб перекласти лише markdown-файли у вашому проєкті, використайте опцію `-md`:

```bash
translate -l "ko" -md
```

### 7. Перевірка перекладених файлів на помилки

Якщо ви хочете перевірити перекладені файли на помилки та за потреби повторити переклад, використайте опцію `-chk`:

```bash
translate -l "ko" -chk
```

Ця команда просканує перекладені markdown-файли і повторить переклад для файлів з помилками.

#### Приклад на Phi-3 CookBook

У **Phi-3 CookBook** я застосував цей метод для перевірки помилок перекладу в корейських файлах і автоматичного повторного перекладу для файлів із виявленими проблемами.

```bash
(.venv) C:\Users\sms79\dev\Phi-3CookBook>translate -l"ko" -chk

Checking files for ko: 100%|██████████████████████████████████████████████████| 95/95 [00:01<00:00, 65.47file/s]

```

Ця опція перевіряє помилки перекладу. Наразі, якщо різниця у розбитті рядків між оригіналом і перекладом перевищує шість, файл вважається таким, що має помилку перекладу. У майбутньому планую покращити цей критерій для більшої гнучкості.

Наприклад, цей метод корисний для виявлення відсутніх частин або пошкоджених перекладів, і автоматично повторить переклад таких файлів.

Однак, якщо ви вже знаєте, які файли проблемні, ефективніше вручну видалити ці файли та використати опцію `-a` option to re-translate them.

### 8. Debug Mode

To enable detailed logging for troubleshooting, use the `-d`:

```bash
translate -l "ko" -d
```

Ця команда запустить переклад у режимі налагодження, надаючи додаткову інформацію в логах, що допоможе ідентифікувати проблеми під час перекладу.

### 9. Переклад усіх мов

Якщо ви хочете перекласти проєкт на всі підтримувані мови, використайте ключове слово all.

> [!WARNING]
> Переклад усіх мов одночасно може зайняти значний час залежно від розміру проєкту. Наприклад, переклад **Phi-3 CookBook** іспанською тривав близько 2 годин. З огляду на обсяг, для однієї людини не практично опрацьовувати 20 мов. Рекомендується розподілити роботу між кількома учасниками, кожен з яких опікуватиметься однією-двома мовами, та оновлювати переклади поступово.

```bash
translate -l "all"
```

Ця команда перекладе проєкт на всі доступні мови. Якщо ви продовжите, переклад може зайняти значний час залежно від розміру проєкту.

> [!TIP]
>
> ### Ручне видалення перекладених файлів (опціонально)
> Перекладені файли тепер автоматично визначаються і очищуються при оновленні вихідного файлу.
>
> Однак, якщо ви хочете вручну оновити переклад — наприклад, переробити конкретний файл або перевизначити поведінку системи — ви можете використати наступну команду для видалення всіх версій файлу у папках мов.
>
> ### У Windows:
> 1. **Використання Command Prompt**:
>    - Відкрийте Command Prompt.
>    - Перейдіть у папку з файлами за допомогою команди `cd`.
>    - Виконайте таку команду для видалення файлів:
>      ```
>      del /s *filename*
>      ```
>      Опція `/s` шукає також у підпапках.
>
> 2. **Використання PowerShell**:
>    - Відкрийте PowerShell.
>    - Виконайте цю команду:
>      ```powershell
>      Get-ChildItem -Path "C:\YourPath" -Filter "*filename*" -Recurse | Remove-Item -Force
>      ```
>      Замініть `"C:\YourPath"` на шлях до вашої папки.
>
> ### Використання команди `cd` та `find`:
>     ```bash
>     find . -type f -name "*filename*" -delete
>     ```
